<h3 class="md-end-block md-heading"><span class="md-plain">1. 什么是 JWT？它由哪几部分组成？</span></h3>
<p class="md-end-block md-p md-focus"><span class="md-plain">JWT（JSON Web Token）是一种基于 JSON 的开放标准（RFC 7519），用于在网络应用间安全传递声明。它由三部分组成，用 </span><span class="md-pair-s" spellcheck="false"><code>.</code></span><span class="md-plain"> 分隔：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Header（头部）</span></strong></span><span class="md-plain">：通常包含令牌类型（如 </span><span class="md-pair-s" spellcheck="false"><code>JWT</code></span><span class="md-plain">）和签名算法（如 </span><span class="md-pair-s" spellcheck="false"><code>HS256</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Payload（负载）</span></strong></span><span class="md-plain">：包含声明（Claims），如用户身份、权限、过期时间等。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Signature（签名）</span></strong></span><span class="md-plain">：使用 Header 中指定的算法，对 Header 和 Payload 签名，确保数据未被篡改。</span></p>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">结构示例</span></strong></span><span class="md-plain">：</span></p>



```plaintext
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.  // Header
eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.  // Payload
SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c  // Signature
```


<h3 class="md-end-block md-heading"><span class="md-plain">2. JWT 的三部分分别存储哪些信息？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Header</span></strong></span><span class="md-plain">：</span></p>



```json
{
  "alg": "HS256",  // 签名算法
  "typ": "JWT"     // 令牌类型
}
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Payload</span></strong></span><span class="md-plain">：</span></p>



```json
{
  "sub": "1234567890",  // 主题（通常是用户 ID）
  "name": "John Doe",   // 自定义声明
  "iat": 1516239022,    // 签发时间（Issued At）
  "exp": 1516325422     // 过期时间（Expiration Time）
}
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Signature</span></strong></span><span class="md-plain">：</span></p>



```bash
HMACSHA256(
  base64UrlEncode(header) + "." +
  base64UrlEncode(payload),
  your-256-bit-secret
)
```


</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">3. JWT 的工作原理是什么？</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">用户登录：输入凭证发送到服务器</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">生成 JWT：服务器验证通过后生成包含用户信息的 JWT</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">发送 JWT：将 JWT 返回给客户端</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">存储 JWT：客户端本地存储（如 localStorage、cookie）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">发送请求：每次请求携带 JWT（通常在 Authorization 头部）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">验证 JWT：服务器提取并验证 JWT 的有效性</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">处理请求：根据 JWT 信息进行身份验证和授权</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">6. JWT 的优点是什么？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">无状态</span></strong></span><span class="md-plain">：无需服务器存储 Session，适合微服务和分布式系统。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">跨域支持</span></strong></span><span class="md-plain">：可通过 </span><span class="md-pair-s" spellcheck="false"><code>localStorage</code></span><span class="md-plain"> 存储，支持跨域请求。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">扩展性</span></strong></span><span class="md-plain">：Payload 可自定义声明，灵活携带用户信息。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">安全性</span></strong></span><span class="md-plain">：签名机制确保数据完整性，支持多种加密算法。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">移动端友好</span></strong></span><span class="md-plain">：易于在移动应用中实现。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">7. JWT 的常见算法有哪些？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">对称加密</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>HS256</code></span><span class="md-plain">（HMAC-SHA256）：使用相同密钥签名和验证。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>HS384</code></span><span class="md-plain">（HMAC-SHA384）、</span><span class="md-pair-s" spellcheck="false"><code>HS512</code></span><span class="md-plain">（HMAC-SHA512）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">非对称加密</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>RS256</code></span><span class="md-plain">（RSA-SHA256）：使用私钥签名，公钥验证。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>ES256</code></span><span class="md-plain">（ECDSA-SHA256）：椭圆曲线加密，性能更高。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">8. JWT 中重要的注册声明（Claims）有哪些？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">标准声明（非强制）</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>iss</code></span><span class="md-plain">（Issuer）：签发者。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>sub</code></span><span class="md-plain">（Subject）：主题（通常是用户 ID）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>aud</code></span><span class="md-plain">（Audience）：受众。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>exp</code></span><span class="md-plain">（Expiration Time）：过期时间（Unix 时间戳）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>nbf</code></span><span class="md-plain">（Not Before）：生效时间（在此之前令牌无效）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>iat</code></span><span class="md-plain">（Issued At）：签发时间。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>jti</code></span><span class="md-plain">（JWT ID）：唯一标识符（用于防重放攻击）。</span></p>
</li>
</ul>
</li>
</ul>

<h3 class="md-end-block md-heading"><span class="md-plain">10. </span><span class="md-pair-s "><strong><span class="md-plain">什么是密钥混淆攻击？如何防止？</span></strong></span></h3>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">攻击原理</span></strong></span><span class="md-plain">：</span> <span class="md-plain">攻击者通过猜测或泄露的密钥，伪造合法 JWT 签名。例如：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用公共密钥（如 </span><span class="md-pair-s" spellcheck="false"><code>RS256</code></span><span class="md-plain"> 的公钥）尝试签名。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">利用弱密钥（如短字符串、默认值）进行暴力破解。</span></p>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">防范措施</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用足够复杂的密钥（如 256 位随机字符串）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">定期轮换密钥。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">避免在客户端或公开代码中暴露密钥。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">优先使用非对称加密（如 </span><span class="md-pair-s" spellcheck="false"><code>RS256</code></span><span class="md-plain">），私钥仅保存在服务器。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">11. </span><span class="md-pair-s "><strong><span class="md-plain">JWT 的安全风险有哪些？如何降低？</span></strong></span></h3>
<h5 class="md-end-block md-heading"><span class="md-plain">1. 签名未校验攻击</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞原理：开发人员混淆 verify () 和 decode () 方法，仅解码而不验证签名</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击步骤</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">登录获取 JWT</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">修改 Payload 中的用户身份（如 sub 改为 administrator）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">重新生成 JWT 并发送请求</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">绕过验证访问管理员功能</span></p>
</li>
</ol>
</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">2. 签名用 None 攻击</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞原理：JWT Header 中 alg 设为 "none" 表示不签名，任何 token 都有效</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击步骤</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">登录获取 JWT</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">修改 Header 的 alg 为 "none"</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">修改 Payload 中的用户身份</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">移除 Signature 部分（保留点分隔符）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">发送请求冒充管理员</span></p>
</li>
</ol>
</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">3. JWT 字典爆破攻击</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞原理：使用默认或弱密钥签名 JWT</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击步骤</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">获取有效的 JWT</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用 hashcat 等工具结合字典暴力破解密钥</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">命令示例：</span><span class="md-pair-s" spellcheck="false"><code>hashcat -a 0 -m 16500 &lt;jwt&gt; &lt;wordlist&gt;</code></span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用破解的密钥生成管理员 JWT</span></p>
</li>
</ol>
</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">4. JWK Header 注入攻击</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞原理：服务器错误使用 jwk 参数中嵌入的密钥</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击步骤</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">生成新的 RSA 密钥对</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">修改 JWT 的 Payload 为管理员身份</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">在 Header 中注入 JWK 参数（包含生成的公钥）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用私钥签名 JWT</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">发送请求绕过验证</span></p>
</li>
</ol>
</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">5. 敏感信息泄露攻击</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞原理：JWT 中包含敏感信息且未加密</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击步骤</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">获取 JWT 令牌</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">解码 Payload 获取敏感信息（如密码、用户身份）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用获取的信息进行登录或进一步攻击</span></p>
</li>
</ol>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">11. </span><span class="md-pair-s "><strong><span class="md-plain">JWT 如何实现刷新机制？</span></strong></span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">双令牌方案</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">访问令牌（Access Token）</span></strong></span><span class="md-plain">：短期有效（如 15 分钟），用于 API 请求。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">刷新令牌（Refresh Token）</span></strong></span><span class="md-plain">：长期有效（如 7 天），存储在安全位置（如 </span><span class="md-pair-s" spellcheck="false"><code>HttpOnly</code></span><span class="md-plain"> Cookie）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">流程</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">访问令牌过期时，客户端使用刷新令牌向 </span><span class="md-pair-s" spellcheck="false"><code>/refresh</code></span><span class="md-plain"> 端点请求新的访问令牌。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">服务器验证刷新令牌，生成新的访问令牌并返回。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">安全措施</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">刷新令牌仅能使用一次，使用后立即失效。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">存储刷新令牌时使用 </span><span class="md-pair-s" spellcheck="false"><code>Secure</code></span><span class="md-plain"> 和 </span><span class="md-pair-s" spellcheck="false"><code>HttpOnly</code></span><span class="md-plain"> 属性。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">记录令牌使用日志，检测异常行为。</span></p>
</li>
</ul>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">12. </span><span class="md-pair-s "><strong><span class="md-plain">JWT 是如何实现签名和验证的？加密和签名的区别是什么？</span></strong></span></h3>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">签名流程</span></strong></span><span class="md-plain">：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">服务器使用 Header 中指定的算法（如 </span><span class="md-pair-s" spellcheck="false"><code>HS256</code></span><span class="md-plain">），对 Header 和 Payload 进行哈希。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用密钥对哈希值签名，生成 Signature。</span></p>
</li>
</ol>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">验证流程</span></strong></span><span class="md-plain">：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">客户端接收到 JWT，分离三部分。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">重新计算 Header 和 Payload 的签名（使用相同算法和密钥）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">比较计算结果与接收到的 Signature 是否一致。</span></p>
</li>
</ol>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">加密 vs 签名</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">加密</span></strong></span><span class="md-plain">：将数据转换为密文，防止未授权读取（如 AES 加密）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">签名</span></strong></span><span class="md-plain">：验证数据来源和完整性，不加密数据（如 HMAC）。</span></p>
</li>
</ul>

<h4 class="md-end-block md-heading"><span class="md-plain">JWT 的应用场景</span></h4>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">分布式系统认证</span></strong></span><span class="md-plain">：跨服务、跨域名的统一身份验证（如微服务架构中，无需共享 Session）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">API 接口授权</span></strong></span><span class="md-plain">：第三方应用调用 API 时，通过 JWT 传递用户权限，避免频繁登录。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">单点登录（SSO）</span></strong></span><span class="md-plain">：用户一次登录后，通过 JWT 在多个关联系统中实现免密访问。</span></p>
</li>
 	<li class="md-list-item md-focus-container">
<p class="md-end-block md-p md-focus"><span class="md-pair-s "><strong><span class="md-plain">短期授权</span></strong></span><span class="md-plain md-expand">：如临时链接（文件下载、邮件验证），通过设置过期时间控制权限时效。</span></p>
</li>
</ul>
