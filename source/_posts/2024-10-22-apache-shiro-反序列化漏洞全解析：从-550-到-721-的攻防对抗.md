<h2 class="header-vfC6AV auto-hide-last-sibling-br">一、Apache Shiro 漏洞概述</h2>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">Apache Shiro 是一款流行的 Java 安全框架，提供身份认证、授权、会话管理等功能。但其历史版本中存在两个高危反序列化漏洞 ——<strong>Shiro 550（CVE-2016-4437）</strong> 和<strong>Shiro 721</strong>，攻击者可利用漏洞实现远程代码执行，对 Java 应用造成严重威胁。</div>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">二、Shiro 550 反序列化漏洞（CVE-2016-4437）</h2>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. 漏洞核心信息</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>影响版本</strong>：Apache Shiro < 1.2.4</li>
 	<li><strong>漏洞类型</strong>：反序列化远程代码执行</li>
 	<li><strong>触发条件</strong>：目标启用 “RememberMe” 功能，且使用默认 AES 加密密钥</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">2. 漏洞原理：RememberMe 机制的致命缺陷</h3>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">Shiro 的 “RememberMe” 功能用于实现自动登录，其流程为：</div>
<ol class="auto-hide-last-sibling-br">
 	<li>用户勾选 “记住我” 登录后，服务端生成包含用户信息的<strong>序列化对象</strong>，经 AES 加密和 Base64 编码后，存入<code>rememberMe</code> Cookie 返回客户端；</li>
 	<li>后续请求中，服务端读取<code>rememberMe</code> Cookie，按 “Base64 解码→AES 解密→反序列化” 流程还原对象，实现自动登录。</li>
</ol>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space"><strong>漏洞根源</strong>：早期 Shiro 版本硬编码 AES 密钥为固定值<code>kPH+blxkD2deZikcaaA==</code>。攻击者可利用该密钥构造恶意序列化对象，加密编码后放入<code>rememberMe</code> Cookie，服务端反序列化时触发代码执行。</div>

<h2 class="header-vfC6AV auto-hide-last-sibling-br">三、Shiro 721 反序列化漏洞</h2>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. 漏洞核心信息</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>影响版本</strong>：Apache Shiro 1.2.5 ~ 1.4.1</li>
 	<li><strong>漏洞类型</strong>：Padding Oracle 攻击导致的反序列化</li>
 	<li><strong>触发条件</strong>：获取合法用户的<code>rememberMe</code> Cookie（无需知道加密密钥）</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">2. 漏洞原理：AES-CBC 加密模式的缺陷</h3>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">Shiro 1.2.5 及以上版本修复了默认密钥问题，但仍存在加密模式缺陷：</div>
<ul class="auto-hide-last-sibling-br">
 	<li>采用<strong>AES-128-CBC</strong>模式加密<code>rememberMe</code> Cookie，该模式依赖填充机制（Padding）；</li>
 	<li>服务器对 “填充错误” 和 “解密成功” 的响应存在差异（如响应时间、状态码），攻击者可利用这一差异实施<strong>Padding Oracle 攻击</strong>。</li>
</ul>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space"><strong>攻击流程</strong>：</div>
<ol class="auto-hide-last-sibling-br">
 	<li>攻击者获取合法用户的<code>rememberMe</code> Cookie；</li>
 	<li>通过 Padding Oracle 攻击逐字节推测密文结构，构造含恶意序列化对象的伪造 Cookie；</li>
 	<li>服务端解密伪造 Cookie 时触发反序列化，执行恶意代码（无需知晓加密密钥）。</li>
</ol>

<h2 class="header-vfC6AV auto-hide-last-sibling-br">五、如何识别 Shiro 反序列化漏洞？</h2>
<ol class="auto-hide-last-sibling-br">
 	<li><strong>检测 RememberMe 特征</strong>：访问登录页面，查看响应 Cookie 是否含<code>rememberMe=deleteMe</code>—— 存在则启用了 RememberMe 功能，可能存在漏洞；</li>
 	<li><strong>密钥爆破</strong>：使用 ShiroAttack2 工具扫描，若爆破出默认密钥（如<code>kPH+blxkD2deZikcaaA==</code>），可能存在 Shiro 550；</li>
 	<li><strong>合法 Cookie 测试</strong>：获取合法<code>rememberMe</code> Cookie 后，使用 Shiro 721 攻击脚本测试，若能构造恶意 Cookie 触发反序列化，可能存在 Shiro 721；</li>
 	<li><strong>版本核查</strong>：确认目标 Shiro 版本，<1.2.4 可能存在 550 漏洞，1.2.5~1.4.1 可能存在 721 漏洞。</li>
</ol>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">六、全面防御建议</h2>
<ol class="auto-hide-last-sibling-br">
 	<li><strong>版本管理</strong>：统一升级至 Shiro 1.10.0+（修复所有已知反序列化漏洞）；</li>
 	<li><strong>密钥安全</strong>：
<ul class="auto-hide-last-sibling-br">
 	<li>禁止使用默认密钥，通过官方 API 生成随机密钥（如<code>generateNewKey()</code>）；</li>
 	<li>密钥存储在安全配置中心（如 Vault），避免硬编码；</li>
</ul>
</li>
 	<li><strong>功能管控</strong>：非必要场景禁用 RememberMe 功能，减少攻击面；</li>
 	<li><strong>流量防护</strong>：
<ul class="auto-hide-last-sibling-br">
 	<li>部署 WAF 拦截含序列化特征（如<code>ac ed 00 05</code>魔数）的<code>rememberMe</code> Cookie；</li>
 	<li>限制<code>rememberMe</code>字段长度（建议≤1024 字节），过滤异常值；</li>
</ul>
</li>
 	<li><strong>监控审计</strong>：
<ul class="auto-hide-last-sibling-br">
 	<li>日志中监控反序列化失败记录（如<code>InvalidRememberMeTokenException</code>）；</li>
 	<li>定期用 OWASP Dependency-Check 扫描依赖，排查旧版本 Shiro。</li>
</ul>
</li>
</ol>
