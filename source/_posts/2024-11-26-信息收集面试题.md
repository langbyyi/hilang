<h3 class="md-end-block md-heading"><span class="md-plain">1. 什么是信息收集？主动收集与被动收集的核心区别是什么？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">信息收集</span></strong></span><span class="md-plain">：指通过技术或非技术手段，系统性收集目标对象（如企业、网站、服务器等）的各类信息（网络架构、资产范围、人员信息、技术栈等），为后续渗透测试提供基础。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">主动收集与被动收集的核心区别</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">主动收集：直接与目标系统交互（如发送请求、扫描端口），可能被目标检测到（有 “痕迹”）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">被动收集：通过第三方渠道获取信息（如搜索引擎、公开数据库），不直接与目标交互（无 “痕迹”）。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">2. 给一个目标的信息收集步骤？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">以 “目标为某企业官网（</span><span class="md-meta-i-c md-link"><a href="https://example.com/"><span class="md-plain">example.com</span></a></span><span class="md-plain">）” 为例：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">基础信息：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">WHOIS 查询（域名注册人、邮箱、过期时间）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">DNS 记录（A 记录→IP，CNAME→CDN / 子域名，MX→邮件服务器）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">判断 CDN 存在及获取真实 IP</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">网络资产：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">子域名枚举（工具 + 搜索引擎：site:example.com -www）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">IP 反查（旁站扫描，通过 Censys 查该 IP 绑定的其他域名）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">端口扫描（Nmap -sV -p- [IP]，识别开放端口及服务版本）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">C 段嗅探（同网段扫描）</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">应用层信息：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">指纹识别（Wappalyzer：如 “PHP+Nginx+WordPress”）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">目录扫描（Dirsearch -u </span><span class="md-meta-i-c md-link"><a href="https://example.com/"><span class="md-plain">https://example.com</span></a></span><span class="md-plain"> -e php,txt）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">查找后台（常见路径爆破 + 查看页面源码注释）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">API 接口</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">敏感信息：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">检查 robots.txt、sitemap.xml；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">搜索 GitHub 是否有</span><span class="md-meta-i-c md-link"><a href="https://example.com/"><span class="md-plain">example.com</span></a></span><span class="md-plain">相关代码泄露；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">查看 Wayback Machine 历史快照，寻找旧版本漏洞（如过时的 API 接口）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">人员信息：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">招聘网站（如拉勾网）查该企业招聘岗位，获取技术栈和邮箱格式（如 </span><span class="md-link md-pair-s" spellcheck="false"><a href="mailto:name@example.com">name@example.com</a></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">领英查员工姓名，尝试猜测账号（如 </span><span class="md-link md-pair-s" spellcheck="false"><a href="mailto:zhangsan@example.com">zhangsan@example.com</a></span><span class="md-plain">）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">信息整合与漏洞分析：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">整理关键信息；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞利用方向</span></p>
</li>
</ul>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">3. 子域名收集方式？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">爆破工具</span></strong></span><span class="md-plain">：基于字典枚举（如 Layer 子域名挖掘机、OneForAll，支持多线程 + DNS 验证）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">搜索引擎</span></strong></span><span class="md-plain">：Google/Baidu 语法（site:example.com -www，排除主域名）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">SSL 证书</span></strong></span><span class="md-plain">：通过 Censys、crt.sh 查询目标域名的 SSL 证书，证书中常包含子域名。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">DNS 记录查询</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">枚举 DNS 服务器的 AXFR 记录（若允许区域传输，可获取所有子域名）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">解析历史（如</span><span class="md-meta-i-c md-link"><a href="https://viewdns.info/"><span class="md-plain">ViewDNS.info</span></a></span><span class="md-plain">的 DNS 历史记录）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">第三方服务</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">威胁情报平台（如微步在线、360 威胁情报）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">子域名聚合工具（如 dnsdb.io、Sublist3r 集成多源数据）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">关联信息挖掘</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">企业备案信息中的关联域名；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">邮件后缀（如 </span><span class="md-link md-pair-s" spellcheck="false"><a href="mailto:support@mail.example.com">support@mail.example.com</a></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">网站内部链接（如 “关于我们” 页面的子域名跳转）。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">4. 怎么获取目标的文件目录？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">目录扫描工具</span></strong></span><span class="md-plain">：Dirsearch、Gobuster、Burp Suite Intruder（爆破常见目录名，如 /admin、/backup）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">网站爬虫</span></strong></span><span class="md-plain">：Burp Spider、OWASP ZAP 爬取网站链接，提取目录结构（如 /static/css、/api/v1）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">敏感文件泄露</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">robots.txt（可能包含禁止爬取的目录，如 “Disallow: /admin/”）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">.DS_Store（Mac 系统生成，泄露目录结构）、.git/（Git 版本控制文件，可恢复目录和文件）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">错误信息</span></strong></span><span class="md-plain">：访问深层路径（如 /example/a/b/c），若返回 403/404，可能泄露上层目录（如 “/example/a/b/not found”）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">漏洞利用</span></strong></span><span class="md-plain">：文件遍历漏洞（如 “../” 跳转）、任意文件读取漏洞（直接读取目录列表或配置文件）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">公开文档</span></strong></span><span class="md-plain">：目标企业的技术文档、帮助中心可能提及内部目录结构（如 “上传文件存储在 /uploads/”）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">5. 网站查找后台的方法有哪些？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">常见路径爆破</span></strong></span><span class="md-plain">：使用字典（如 admin、manage、login、system、backend）+ 常见后缀（.php、.asp、.jsp），工具如 Dirsearch。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">网站内部链接</span></strong></span><span class="md-plain">：检查页面源码、导航栏、底部链接（如 “管理员入口”“登录后台”）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">robots.txt</span></strong></span><span class="md-plain">：可能包含后台路径（如 “Disallow: /admin/login.php”）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">CMS 默认后台</span></strong></span><span class="md-plain">：如 WordPress（/wp-admin）、Dedecms（/dede）、Discuz（/admin.php）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">搜索引擎语法</span></strong></span><span class="md-plain">：site:example.com inurl:admin（查找 URL 含 “admin” 的页面）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">错误信息泄露</span></strong></span><span class="md-plain">：输入错误账号密码，跳转的 URL 可能为后台路径（如 “/admin/login?error=1”）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">源码泄露</span></strong></span><span class="md-plain">：通过 GitHub、备份文件（如 wwwroot.rar）查找后台路径注释。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">暴力猜测</span></strong></span><span class="md-plain">：结合企业名称缩写（如企业名为 “ABC”，尝试 /abcadmin、/abc_login）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">6. Phpinfo 你会关注哪些信息？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">服务器信息</span></strong></span><span class="md-plain">：操作系统（OS）、Web 服务器版本（如 Apache/2.4.29）、PHP 版本（判断是否有已知漏洞）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">配置项</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>allow_url_fopen</code></span><span class="md-plain">/</span><span class="md-pair-s" spellcheck="false"><code>allow_url_include</code></span><span class="md-plain">（是否允许远程文件包含）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>register_globals</code></span><span class="md-plain">（是否开启全局变量注册，可能导致变量覆盖）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>magic_quotes_gpc</code></span><span class="md-plain">（是否过滤特殊字符，影响 SQL 注入利用）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>open_basedir</code></span><span class="md-plain">（限制文件访问范围，判断文件包含漏洞利用难度）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">文件路径</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>DOCUMENT_ROOT</code></span><span class="md-plain">（网站根目录）、</span><span class="md-pair-s" spellcheck="false"><code>SCRIPT_FILENAME</code></span><span class="md-plain">（当前 phpinfo 文件路径）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">环境变量</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>$_SERVER</code></span><span class="md-plain">中的 IP、域名、用户信息（如</span><span class="md-pair-s" spellcheck="false"><code>REMOTE_ADDR</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>USER</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">扩展模块</span></strong></span><span class="md-plain">：是否加载危险模块（如</span><span class="md-pair-s" spellcheck="false"><code>php_curl</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>php_exec</code></span><span class="md-plain">，可能被用于命令执行）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Session 配置</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>session.save_path</code></span><span class="md-plain">（Session 文件存储路径，可能用于 Session 劫持）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">SSL 信息</span></strong></span><span class="md-plain">：若启用 HTTPS，查看 SSL 协议版本、证书信息（判断是否有 SSL 漏洞）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">7. 常用端口有哪些？（含对应服务，举例 15 个）</span></h3>
<p class="md-end-block md-p"><span class="md-plain">常用端口及对应服务如下：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">21/tcp：FTP（文件传输协议，用于文件上传下载）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">22/tcp：SSH（安全外壳协议，用于远程登录服务器）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">23/tcp：Telnet（远程登录协议，明文传输，安全性低）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">25/tcp：SMTP（简单邮件传输协议，用于发送邮件）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">53/udp/tcp：DNS（域名系统，用于域名解析）</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">9. 常见的端口扫描技术有哪些？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">全连接扫描（TCP Connect Scan）</span></strong></span><span class="md-plain">：通过调用</span><span class="md-pair-s" spellcheck="false"><code>connect()</code></span><span class="md-plain">函数与目标端口建立完整 TCP 连接，成功则端口开放（特征明显，易被防火墙检测）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">半连接扫描（SYN Scan）</span></strong></span><span class="md-plain">：发送 SYN 包，若收到 SYN+ACK 则端口开放（不建立完整连接，隐蔽性较好）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">FIN 扫描</span></strong></span><span class="md-plain">：发送 FIN 包，若收到 RST 则端口关闭，无响应则可能开放（适用于对 FIN 包不严格处理的系统，如 Unix）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Xmas 扫描</span></strong></span><span class="md-plain">：发送带 FIN、URG、PSH 标志的包，规则同 FIN 扫描（利用系统对异常标志包的处理差异）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">NULL 扫描</span></strong></span><span class="md-plain">：发送无任何标志的包，逻辑同 FIN 扫描（仅对遵循 RFC 的系统有效）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">ACK 扫描</span></strong></span><span class="md-plain">：发送 ACK 包，通过返回的 RST 包 TTL 值判断端口是否被防火墙过滤（用于探测防火墙规则）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">UDP 扫描</span></strong></span><span class="md-plain">：发送 UDP 包，若收到 ICMP 端口不可达则端口关闭，无响应可能开放（速度慢，因 UDP 无连接特性）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">10. 3 种端口扫描工具及核心功能；nmap 如何识别开放端口及服务版本？</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">端口扫描工具：</span></h4>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Nmap</span></strong></span><span class="md-plain">：核心功能是端口扫描、服务版本探测、操作系统识别、脚本扫描（如漏洞检测），支持多种扫描技术，灵活性极强。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Masscan</span></strong></span><span class="md-plain">：超高速端口扫描工具，可同时扫描全网端口，核心功能是大规模网段快速探测，速度远快于 Nmap（但精度略低）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Zmap</span></strong></span><span class="md-plain">：专注于网络普查的高速扫描工具，核心功能是对大量 IP 进行特定端口扫描（如 443 端口），支持自定义探针，适合批量探测。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">Nmap 识别开放端口及服务版本的方法：</span></h4>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">执行命令：</span><span class="md-pair-s" spellcheck="false"><code>nmap -sV [目标IP]</code></span><span class="md-plain">（</span><span class="md-pair-s" spellcheck="false"><code>-sV</code></span><span class="md-plain">参数用于服务版本探测）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">原理：先通过端口扫描确定开放端口，再向开放端口发送特定探针（如 HTTP 的</span><span class="md-pair-s" spellcheck="false"><code>GET /</code></span><span class="md-plain">、SSH 的版本请求），根据返回的 banner 信息（如 “Apache/2.4.41”）匹配内置指纹库，识别服务名称及版本。</span></p>
</li>
</ul>

<h3 class="md-end-block md-heading"><span class="md-plain">12. T3 协议（含 Weblogic T3 反序列化漏洞原理及防御）</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">T3 协议</span></strong></span><span class="md-plain">：Weblogic 特有的通信协议（基于 TCP），用于 Weblogic 服务器与客户端（如 EJB 客户端）的远程通信，默认端口 7001。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">Weblogic T3 反序列化漏洞原理：</span></h4>
<p class="md-end-block md-p"><span class="md-plain">攻击者向 Weblogic 的 T3 端口发送恶意序列化数据（如精心构造的 Java 对象），Weblogic 在解析 T3 协议数据时，会对序列化对象进行反序列化操作，若对象中包含恶意代码（如</span><span class="md-pair-s" spellcheck="false"><code>CommonsCollections</code></span><span class="md-plain">链），则会触发代码执行，获取服务器权限。</span></p>

<h4 class="md-end-block md-heading"><span class="md-plain">防御措施：</span></h4>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">升级 Weblogic 至官方补丁版本（如 CPU 补丁）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">禁用 T3 协议：在</span><span class="md-pair-s" spellcheck="false"><code>weblogic.xml</code></span><span class="md-plain">中配置</span><span class="md-pair-s" spellcheck="false"><code>&lt;enforce-valid-basic-auth-credentials&gt;true&lt;/enforce-valid-basic-auth-credentials&gt;</code></span><span class="md-plain">，或通过控制台关闭 T3 监听。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">限制 T3 端口访问：仅允许信任 IP 连接 7001 端口（如防火墙 ACL 规则）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">13. RMI 协议（含工作流程、核心组件、格式）</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">RMI（Remote Method Invocation）</span></strong></span><span class="md-plain">：远程方法调用协议，允许 Java 程序调用远程主机上的对象方法。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">工作流程：</span></h4>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">服务端注册</span></strong></span><span class="md-plain">：远程对象通过</span><span class="md-pair-s" spellcheck="false"><code>LocateRegistry.createRegistry()</code></span><span class="md-plain">在指定端口（默认 1099）注册服务。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">客户端查找</span></strong></span><span class="md-plain">：客户端通过</span><span class="md-pair-s" spellcheck="false"><code>Naming.lookup()</code></span><span class="md-plain">从注册中心（Registry）获取远程对象引用。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">方法调用</span></strong></span><span class="md-plain">：客户端调用远程对象方法，参数通过序列化传输到服务端，服务端执行后返回结果（反序列化）。</span></p>
</li>
</ol>
<h4 class="md-end-block md-heading"><span class="md-plain">核心组件：</span></h4>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Registry</span></strong></span><span class="md-plain">：注册中心，负责存储远程对象的引用（绑定名称与对象）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Server</span></strong></span><span class="md-plain">：提供远程对象实现类，注册到 Registry。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Client</span></strong></span><span class="md-plain">：查找远程对象并调用其方法。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">格式：</span></h4>
<p class="md-end-block md-p"><span class="md-plain">基于</span><span class="md-pair-s "><strong><span class="md-plain">JRMP（Java Remote Method Protocol）</span></strong></span><span class="md-plain">，数据传输采用 Java 序列化格式（</span><span class="md-pair-s" spellcheck="false"><code>ObjectOutputStream</code></span><span class="md-plain">），协议头包含 JRMP 版本等信息。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">14. LDAP 协议</span></h3>

<h3 class="md-end-block md-heading"><span class="md-plain">16. 跨域；Jsonp 与 Cors 是什么？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">跨域</span></strong></span><span class="md-plain">：浏览器的同源策略限制下，不同源的网页之间无法互相访问资源（如 AJAX 请求），“不同源” 指协议、域名、端口任意一项不同。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Jsonp（JSON with Padding）</span></strong></span><span class="md-plain">：跨域解决方案，利用</span><span class="md-pair-s" spellcheck="false"><code>&lt;script&gt;</code></span><span class="md-plain">标签不受同源策略限制的特性，通过回调函数获取远程数据。流程：客户端定义回调函数，请求 URL 携带回调名（如</span><span class="md-pair-s" spellcheck="false"><code>http://api.com/data?callback=func</code></span><span class="md-plain">），服务端返回</span><span class="md-pair-s" spellcheck="false"><code>func(data)</code></span><span class="md-plain">，客户端脚本执行</span><span class="md-pair-s" spellcheck="false"><code>func()</code></span><span class="md-plain">处理数据。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">CORS（Cross-Origin Resource Sharing）</span></strong></span><span class="md-plain">：服务器端配置的跨域机制，通过响应头（如</span><span class="md-pair-s" spellcheck="false"><code>Access-Control-Allow-Origin: https://example.com</code></span><span class="md-plain">）明确允许特定源访问资源，支持多种 HTTP 方法（如 POST、PUT），比 Jsonp 更规范、安全。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">17. 同源策略中的同源条件；作用与限制？</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">同源条件：</span></h4>
<p class="md-end-block md-p"><span class="md-plain">需同时满足</span><span class="md-pair-s "><strong><span class="md-plain">协议相同</span></strong></span><span class="md-plain">（如均为</span><span class="md-pair-s" spellcheck="false"><code>http</code></span><span class="md-plain">或</span><span class="md-pair-s" spellcheck="false"><code>https</code></span><span class="md-plain">）、</span><span class="md-pair-s "><strong><span class="md-plain">域名相同</span></strong></span><span class="md-plain">（如均为</span><span class="md-pair-s" spellcheck="false"><code>example.com</code></span><span class="md-plain">，</span><span class="md-pair-s" spellcheck="false"><code>a.example.com</code></span><span class="md-plain">与</span><span class="md-pair-s" spellcheck="false"><code>b.example.com</code></span><span class="md-plain">不同源）、</span><span class="md-pair-s "><strong><span class="md-plain">端口相同</span></strong></span><span class="md-plain">（如均为 80 或 443）。</span></p>

<h4 class="md-end-block md-heading"><span class="md-plain">作用：</span></h4>
<p class="md-end-block md-p"><span class="md-plain">防止恶意网站通过脚本窃取用户数据（如 Cookie、LocalStorage）或操作 DOM，是浏览器的核心安全机制。</span></p>

<h4 class="md-end-block md-heading"><span class="md-plain">限制：</span></h4>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">不能读取不同源网页的 Cookie、LocalStorage、SessionStorage。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">不能访问不同源网页的 DOM（如</span><span class="md-pair-s" spellcheck="false"><code>iframe</code></span><span class="md-plain">中的内容）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">不能发送跨域 AJAX/ Fetch 请求（或需服务器允许，如 CORS）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">不能调用</span><span class="md-pair-s" spellcheck="false"><code>window.open()</code></span><span class="md-plain">打开的不同源窗口的</span><span class="md-pair-s" spellcheck="false"><code>window</code></span><span class="md-plain">对象属性。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">18. 什么是 CDN？CDN 的作用是什么？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">CDN（Content Delivery Network，内容分发网络）</span></strong></span><span class="md-plain">：是一组分布在不同地理位置的服务器集群，通过负载均衡、内容缓存等技术，将源站内容分发到离用户最近的节点，实现内容快速传输。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">作用</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">加速内容访问（减少用户与源站的物理距离）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">减轻源服务器压力（流量分散到边缘节点）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">提高可用性（某节点故障时自动切换到其他节点）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">增强安全性（抵御 DDoS 攻击，隐藏源站 IP）。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">19. 如何判断目标是否存在 CDN？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">判断目标是否使用 CDN 的核心是检测其 IP 地址是否 “多节点分布式”，常用方法：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">多节点 ping 测试</span></strong></span><span class="md-plain">：从不同地区（如国内不同省份、国外）ping 目标域名，若返回的 IP 地址不同，大概率存在 CDN；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">DNS 解析查看</span></strong></span><span class="md-plain">：通过 </span><span class="md-pair-s" spellcheck="false"><code>nslookup</code></span><span class="md-plain"> 或 </span><span class="md-pair-s" spellcheck="false"><code>dig</code></span><span class="md-plain"> 命令查询域名的 DNS 记录，若返回多个 IP 且来自不同运营商（如阿里云、腾讯云、Cloudflare 等），可能存在 CDN；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">在线工具检测</span></strong></span><span class="md-plain">：使用 WhatIsMyIP、CDN Finder 等工具，直接查询域名是否使用 CDN；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">子域名排查</span></strong></span><span class="md-plain">：主域名可能用 CDN，但部分子域名（如 </span><span class="md-pair-s" spellcheck="false"><code>mail.xxx.com</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>ftp.xxx.com</code></span><span class="md-plain">）可能未使用，若子域名 IP 固定，可能为源站 IP。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">20. 如何绕过 CDN 找真实 IP 地址？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">绕过 CDN 的核心是 “找到与源站直接交互的入口”，五种方法：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">子域名遍历</span></strong></span><span class="md-plain">：主域名用 CDN 但子域名可能未用（如 </span><span class="md-pair-s" spellcheck="false"><code>admin.xxx.com</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>test.xxx.com</code></span><span class="md-plain">），解析子域名获取 IP 后，验证是否为源站（如访问该 IP 能否打开主站内容）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">邮件头分析</span></strong></span><span class="md-plain">：目标网站发送的邮件（如注册通知、营销邮件）中，</span><span class="md-pair-s" spellcheck="false"><code>Received</code></span><span class="md-plain"> 字段会记录邮件服务器的 IP，该 IP 可能与源站同网段；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">历史 DNS 记录查询</span></strong></span><span class="md-plain">：通过工具（如 DNSlytics、SecurityTrails）获取目标域名过去解析的 IP，尝试访问这些 IP 验证是否为源站；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">利用服务器主动连接</span></strong></span><span class="md-plain">：若目标存在 SSRF 漏洞，构造 payload 让服务器访问自己的公网监控地址（如 </span><span class="md-pair-s" spellcheck="false"><code>http://xxx.xxx.xxx.xxx:port</code></span><span class="md-plain">），监控日志中会记录目标的真实 IP；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">边缘节点漏洞</span></strong></span><span class="md-plain">：部分 CDN 配置不当（如未对源站 IP 做访问控制），可通过扫描 CDN 节点的反向代理配置（如 Nginx 的 </span><span class="md-pair-s" spellcheck="false"><code>X-Forwarded-For</code></span><span class="md-plain"> 泄露源站 IP）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">ICMP 隧道 / 特殊协议</span></strong></span><span class="md-plain">：部分 CDN 仅加速 HTTP/HTTPS 流量，对 ICMP（ping）、FTP 等协议未代理，可通过 ping 或 FTP 连接尝试获取真实 IP。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">21. 什么是目录遍历？如何通过工具（如 Dirbuster）发现隐藏文件或敏感目录？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">目录遍历</span></strong></span><span class="md-plain">：攻击者通过构造特殊请求（如 </span><span class="md-pair-s" spellcheck="false"><code>../</code></span><span class="md-plain"> 路径穿越）或爆破路径，访问网站未公开的目录（如 </span><span class="md-pair-s" spellcheck="false"><code>admin/</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>backup/</code></span><span class="md-plain">）或文件（如 </span><span class="md-pair-s" spellcheck="false"><code>config.php</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>database.sql</code></span><span class="md-plain">），获取敏感信息。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Dirbuster 用法</span></strong></span><span class="md-plain">：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输入目标 URL（如 </span><span class="md-pair-s" spellcheck="false"><code>http://xxx.com</code></span><span class="md-plain">），选择扫描模式（“List based brute force”）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">加载字典（如 </span><span class="md-pair-s" spellcheck="false"><code>directory-list-2.3-medium.txt</code></span><span class="md-plain">），设置文件扩展名（如 </span><span class="md-pair-s" spellcheck="false"><code>.php</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>.html</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">配置线程数（默认 10，视目标抗压力调整）、超时时间，启动扫描；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">扫描结果中，状态码为 200（存在）、301/302（跳转）的路径即为潜在敏感目录 / 文件。</span></p>
</li>
</ol>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">22. 简述通过 Robots.txt、Sitemap.xml 收集信息的技巧（如隐藏目录、备份文件）</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Robots.txt</span></strong></span><span class="md-plain">：网站用于告知搜索引擎 “哪些目录 / 文件不允许爬取”，但可能泄露敏感路径：</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">直接访问 </span><span class="md-pair-s" spellcheck="false"><code>http://xxx.com/robots.txt</code></span><span class="md-plain">，若存在 </span><span class="md-pair-s" spellcheck="false"><code>Disallow: /admin/</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>Disallow: /backup/</code></span><span class="md-plain"> 等记录，可直接尝试访问这些目录；</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若存在 </span><span class="md-pair-s" spellcheck="false"><code>Disallow: /config.php</code></span><span class="md-plain">，可能是数据库配置文件，尝试下载。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Sitemap.xml</span></strong></span><span class="md-plain">：用于告知搜索引擎网站的结构（如页面 URL、更新时间），可能暴露：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">隐藏页面（如 </span><span class="md-pair-s" spellcheck="false"><code>http://xxx.com/internal/report.html</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">备份文件（如 </span><span class="md-pair-s" spellcheck="false"><code>http://xxx.com/backup/20231001.sql</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">动态参数页面（如 </span><span class="md-pair-s" spellcheck="false"><code>http://xxx.com/user?id=123</code></span><span class="md-plain">），可用于后续参数注入测试。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">23. 什么是 Whois 查询？它能获取哪些关键信息（如注册人、DNS 服务器等）？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Whois 查询</span></strong></span><span class="md-plain">：通过域名注册商的数据库，查询域名的注册信息（公开可查）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">关键信息</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">注册人（姓名 / 企业名）、联系邮箱、联系电话；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">域名注册商、注册时间、过期时间；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">DNS 服务器（如 </span><span class="md-pair-s" spellcheck="false"><code>ns1.aliyun.com</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>ns2.cloudflare.com</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">注册地址（可能关联企业真实地址）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">用途</span></strong></span><span class="md-plain">：用于社会工程学攻击（如通过邮箱 / 电话钓鱼）、挖掘关联域名（同 DNS 服务器可能属于同一企业）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">24. DNS 枚举的常用工具</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>nslookup/dig</code></span><span class="md-plain">：基础 DNS 解析工具，用于查询 A 记录（IP）、CNAME 记录（别名）、MX 记录（邮件服务器）等；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>dnsenum</code></span><span class="md-plain">：自动化枚举工具，支持爆破子域名、查询 MX 记录、反向查询 IP 对应的域名；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>Amass</code></span><span class="md-plain">（OWASP 工具）：结合被动收集（如 DNS 存档、证书透明度日志）和主动爆破，高效发现子域名；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>Sublist3r</code></span><span class="md-plain">：专注子域名枚举，集成多个搜索引擎 API（如 Google、Bing）和字典爆破。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">25. 如何快速识别目标网站使用的 CMS（如 Wordpress、Drupal）？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">查看网页源码</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">检查 </span><span class="md-pair-s" spellcheck="false"><code>&lt;meta&gt;</code></span><span class="md-plain"> 标签（如 Wordpress 可能包含 </span><span class="md-pair-s" spellcheck="false"><code>&lt;meta name=&quot;generator&quot; content=&quot;WordPress 6.0&quot;&gt;</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">查看链接或脚本路径（如 </span><span class="md-pair-s" spellcheck="false"><code>wp-content/</code></span><span class="md-plain"> 为 Wordpress 特征，</span><span class="md-pair-s" spellcheck="false"><code>sites/default/files/</code></span><span class="md-plain"> 为 Drupal 特征）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">响应头分析</span></strong></span><span class="md-plain">：通过浏览器开发者工具（F12）的 “Network” 面板，查看 </span><span class="md-pair-s" spellcheck="false"><code>X-Powered-By</code></span><span class="md-plain"> 或自定义头（如 </span><span class="md-pair-s" spellcheck="false"><code>X-Drupal-Cache</code></span><span class="md-plain"> 表明 Drupal）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">特定文件检测</span></strong></span><span class="md-plain">：访问 </span><span class="md-pair-s" spellcheck="false"><code>readme.txt</code></span><span class="md-plain">（Wordpress 会显示版本）、</span><span class="md-pair-s" spellcheck="false"><code>CHANGELOG.txt</code></span><span class="md-plain">（Drupal 版本信息）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">工具识别</span></strong></span><span class="md-plain">：使用 </span><span class="md-pair-s" spellcheck="false"><code>WhatWeb</code></span><span class="md-plain">（命令行工具，如 </span><span class="md-pair-s" spellcheck="false"><code>whatweb http://xxx.com</code></span><span class="md-plain">）、</span><span class="md-pair-s" spellcheck="false"><code>Wappalyzer</code></span><span class="md-plain">（浏览器插件）直接识别 CMS 类型及版本。</span></p>
</li>
</ul>

<h3 class="md-end-block md-heading"><span class="md-plain">27. 如何手工快速判断目标站是 Windows 还是 Linux 服务器？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">可通过以下差异特征判断：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">文件路径分隔符</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Windows 使用反斜杠 </span><span class="md-pair-s" spellcheck="false"><code>\</code></span><span class="md-plain">（如 </span><span class="md-pair-s" spellcheck="false"><code>C:\inetpub\wwwroot</code></span><span class="md-plain">），Linux 使用正斜杠 </span><span class="md-pair-s" spellcheck="false"><code>/</code></span><span class="md-plain">（如 </span><span class="md-pair-s" spellcheck="false"><code>/var/www/html</code></span><span class="md-plain">）。若页面报错信息中出现路径，可直接区分。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">默认目录与文件</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Windows 服务器常见默认路径：</span><span class="md-pair-s" spellcheck="false"><code>/iisstart.htm</code></span><span class="md-plain">（IIS 默认页）、</span><span class="md-pair-s" spellcheck="false"><code>/aspnet_client/</code></span><span class="md-plain">（</span><span class="md-meta-i-c md-link"><a href="https://asp.net/"><span class="md-plain">ASP.NET</span></a></span><span class="md-plain"> 相关）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Linux 服务器常见默认路径：</span><span class="md-pair-s" spellcheck="false"><code>/index.php</code></span><span class="md-plain">（PHP 站点）、</span><span class="md-pair-s" spellcheck="false"><code>/robots.txt</code></span><span class="md-plain"> 权限差异（Linux 对文件权限更敏感）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">大小写敏感性</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Linux 系统对文件名 / 路径大小写敏感（如 </span><span class="md-pair-s" spellcheck="false"><code>Index.html</code></span><span class="md-plain"> 与 </span><span class="md-pair-s" spellcheck="false"><code>index.html</code></span><span class="md-plain"> 视为不同文件），Windows 不敏感。可通过访问不同大小写的 URL 测试（如 </span><span class="md-pair-s" spellcheck="false"><code>http://目标/Test</code></span><span class="md-plain"> 与 </span><span class="md-pair-s" spellcheck="false"><code>http://目标/test</code></span><span class="md-plain">）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">服务与端口</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Windows 常用 IIS 服务器（端口 80/443）、远程桌面（3389）；Linux 常用 Apache/Nginx（80/443）、SSH（22）。若目标开放 22 端口，大概率是 Linux；开放 3389 则大概率是 Windows。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">错误信息泄露</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">IIS 服务器（Windows）报错可能包含 </span><span class="md-pair-s" spellcheck="false"><code>Microsoft-IIS/版本</code></span><span class="md-plain"> 标识；Apache/Nginx（Linux）报错会显示对应服务器版本（如 </span><span class="md-pair-s" spellcheck="false"><code>Apache/2.4.6</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">PHP 环境中，</span><span class="md-pair-s" spellcheck="false"><code>phpinfo()</code></span><span class="md-plain"> 页面的 </span><span class="md-pair-s" spellcheck="false"><code>SERVER_SOFTWARE</code></span><span class="md-plain"> 字段会显示服务器系统（如 </span><span class="md-pair-s" spellcheck="false"><code>Windows NT</code></span><span class="md-plain"> 或 </span><span class="md-pair-s" spellcheck="false"><code>Linux</code></span><span class="md-plain">）。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading md-focus"><span class="md-plain">28. 如何使用 nmap 进行渗透测试？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">Nmap 是渗透测试的核心工具，主要用于信息收集与初步漏洞探测，流程如下：</span></p>

<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">存活主机探测</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>nmap -sn 192.168.1.0/24</code></span><span class="md-plain">（仅 Ping 扫描，不扫描端口，避免触发防火墙）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">端口扫描</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">全端口扫描：</span><span class="md-pair-s" spellcheck="false"><code>nmap -p- 192.168.1.1</code></span><span class="md-plain">（扫描所有 65535 个端口）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">常用端口扫描：</span><span class="md-pair-s" spellcheck="false"><code>nmap -p 1-1000 192.168.1.1</code></span><span class="md-plain">。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">服务与版本探测</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>nmap -sV 192.168.1.1</code></span><span class="md-plain">（识别端口对应服务及版本，如 “Apache httpd 2.4.29”）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">操作系统识别</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>nmap -O 192.168.1.1</code></span><span class="md-plain">（通过 TTL、端口特征推测操作系统）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">漏洞扫描</span></strong></span><span class="md-plain">：结合 NSE（Nmap 脚本引擎），如</span><span class="md-pair-s" spellcheck="false"><code>nmap --script vuln 192.168.1.1</code></span><span class="md-plain">（检测常见漏洞，如 Heartbleed）。</span></p>
</li>
 	<li class="md-list-item md-focus-container">
<p class="md-end-block md-p md-focus"><span class="md-pair-s "><strong><span class="md-plain">特定漏洞检测</span></strong></span><span class="md-plain">：使用专用脚本，如</span><span class="md-pair-s" spellcheck="false"><code>nmap --script redis-info 192.168.1.1 -p 6379</code></span><span class="md-plain md-expand">（检测 Redis 配置信息）。</span></p>
</li>
</ol>
