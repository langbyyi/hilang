<h2 class="md-end-block md-heading"><span class="md-plain">一、前言</span></h2>
<p class="md-end-block md-p"><span class="md-plain">如果你是一名安全从业者，Burp Suite 几乎是你每天都要打开的工具。而随着大语言模型（LLM）的爆发式发展，"AI + 安全测试"已经成为不可忽视的趋势。PortSwigger 敏锐地捕捉到了这一点，官方推出了 </span><span class="md-pair-s "><strong><span class="md-plain">MCP Server for Burp Suite</span></strong></span><span class="md-plain"> 扩展，通过 MCP（Model Context Protocol，模型上下文协议）将 Burp Suite 的能力暴露给 AI 客户端，让 AI 可以像人一样操控 Burp。</span></p>
<p class="md-end-block md-p"><span class="md-plain">本文将从安装配置、架构原理到实战场景，带你全面了解和使用这个扩展。</span></p>

<h2 class="md-end-block md-heading"><span class="md-plain">二、什么是 MCP？</span></h2>
<p class="md-end-block md-p"><span class="md-plain">MCP（Model Context Protocol）是由 Anthropic 提出的一种开放协议，旨在标准化 AI 模型与外部工具/数据源之间的交互方式。简单来说，它定义了一套统一的接口规范，让 AI 助手（如 Claude）能够调用外部应用程序的功能。</span></p>
<p class="md-end-block md-p"><span class="md-plain">在安全测试的语境下，MCP 让 AI 能够：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">读取 Burp Suite 的代理历史记录</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">通过 Burp 发送 HTTP 请求</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">查看扫描器发现的问题</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">操作 Repeater、Intruder 等模块</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">甚至修改 Burp 的配置</span></p>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-plain">这意味着，你只需要用自然语言描述你的需求，AI 就能自动完成对应的 Burp 操作。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">3.4 连接 Claude Code</span></h3>
<p class="md-end-block md-p"><span class="md-plain">Burp MCP 扩展使用 SSE 协议通信，而 Claude Code CLI 默认使用 stdio 方式连接 MCP，因此需要一个代理桥接。配置步骤如下：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Burp Suite 安装并启用 MCP 扩展，使其 SSE 服务监听在 </span><span class="md-pair-s" spellcheck="false"><code>127.0.0.1:9876</code></span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">从 Burp MCP 选项卡底部的 </span><span class="md-pair-s "><strong><span class="md-plain">Installation</span></strong></span><span class="md-plain"> 面板中提取 </span><span class="md-pair-s" spellcheck="false"><code>mcp-proxy.jar</code></span><span class="md-plain">，放置到 </span><span class="md-pair-s" spellcheck="false"><code>/home/kali/</code></span><span class="md-plain"> 目录</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">在 </span><span class="md-pair-s" spellcheck="false"><code>~/.claude.json</code></span><span class="md-plain"> 的 </span><span class="md-pair-s" spellcheck="false"><code>mcpServers</code></span><span class="md-plain"> 节点中添加以下配置：</span></p>
</li>
</ol>


```json
"burp": {
  "command": "java",
  "args": [
    "-jar",
    "/home/kali/mcp-proxy.jar",
    "--sse-url",
    "http://127.0.0.1:9876"
  ]
}
```


<ol class="ol-list" start="4">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">重启 Claude Code，MCP 即可自动完成连接</span></p>
</li>
</ol>
<h2 class="md-end-block md-heading"><span class="md-plain">四、架构原理</span></h2>
<p class="md-end-block md-p"><span class="md-plain">了解架构有助于你更好地理解数据流向和安全边界：</span></p>



```
┌─────────────────┐     stdio      ┌──────────────────┐     SSE/HTTP     ┌──────────────────┐
│  AI 客户端       │ ────────────── │  mcp-proxy.jar   │ ──────────────── │  Burp Suite 扩展  │
│  (Claude等)      │                │  (协议桥接代理)    │                  │  (Ktor/Netty)    │
└─────────────────┘                └──────────────────┘                  └────────┬─────────┘
                                                                                  │
                                                                                  ▼
                                                                       ┌──────────────────┐
                                                                       │  Burp Montoya API │
                                                                       │  (操控Burp核心)    │
                                                                       └──────────────────┘
```


<p class="md-end-block md-p"><span class="md-plain">核心组件：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">MCP Server（Ktor/Netty）</span></strong></span><span class="md-plain">：嵌入在 Burp 扩展中的 HTTP 服务器，监听 </span><span class="md-pair-s" spellcheck="false"><code>localhost:9876</code></span><span class="md-plain">，提供 SSE 端点</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">mcp-proxy.jar</span></strong></span><span class="md-plain">：协议桥接代理，将 stdio 协议转换为 SSE 协议（因为 Claude Code CLI 通过 stdio 方式连接 MCP）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">MCP SDK</span></strong></span><span class="md-plain">：使用官方 </span><span class="md-pair-s" spellcheck="false"><code>io.modelcontextprotocol:kotlin-sdk</code></span><span class="md-plain">，确保协议兼容性</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Montoya API</span></strong></span><span class="md-plain">：Burp Suite 的现代扩展 API，提供与 Burp 核心的交互能力</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">4.1 安全机制</span></h3>
<p class="md-end-block md-p"><span class="md-plain">扩展内置了多层安全防护：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">HTTP 请求审批</span></strong></span><span class="md-plain">：AI 发出的每一个 HTTP 请求都会弹出交互式确认对话框，你可以选择「允许一次 / 始终允许该主机 / 始终允许该主机:端口 / 拒绝」</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">历史记录访问审批</span></strong></span><span class="md-plain">：AI 读取代理历史时同样需要确认</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">DNS Rebinding 防护</span></strong></span><span class="md-plain">：服务器会验证 </span><span class="md-pair-s" spellcheck="false"><code>Origin</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>Host</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>Referer</code></span><span class="md-plain"> 头，阻止跨站攻击</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">自动放行列表</span></strong></span><span class="md-plain">：支持通配符配置信任的目标（如 </span><span class="md-pair-s" spellcheck="false"><code>*.your-target.com</code></span><span class="md-plain">），减少重复确认</span></p>
</li>
</ol>
