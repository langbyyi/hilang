<h4 class="md-end-block md-heading"><span class="md-plain">一、问题现象</span></h4>
<p class="md-end-block md-p"><span class="md-plain">在 CentOS 7 系统中执行 yum install、yum update 等命令时，出现以下典型错误：</span></p>



```
Loading mirror speeds from cached hostfile
Could not retrieve mirrorlist http://mirrorlist.centos.org/?release=7&arch=x86_64&repo=os&infra=stock error was
14: curl#6 - "Could not resolve host: mirrorlist.centos.org; 未知的错误"
```


<span role="presentation"><img class="alignnone size-full wp-image-574" src="/wp-content/uploads/2024/img-48ea97c5e2ba.png" alt="" width="1347" height="898" /></span>
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">核心特征</span></strong></span><span class="md-plain">：无法解析官方镜像列表域名，导致 yum 无法获取软件源元数据。</span></p>

<h4 class="md-end-block md-heading"><span class="md-plain">二、问题根源</span></h4>
<p class="md-end-block md-p"><span class="md-plain">CentOS 7 官方生命周期于 </span><span class="md-pair-s "><strong><span class="md-plain">2024 年 6 月 30 日</span></strong></span><span class="md-plain"> 结束，官方停止维护后：</span></p>

<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">原镜像列表服务（</span><span class="md-meta-i-c md-link"><a href="http://mirrorlist.centos.org/"><span class="md-plain">mirrorlist.centos.org</span></a></span><span class="md-plain">）停止解析，导致域名无法访问；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">官方软件仓库（</span><span class="md-link md-pair-s" spellcheck="false"><a href="http://mirror.centos.org/">http://mirror.centos.org</a></span><span class="md-plain">）被归档至历史仓库（</span><span class="md-meta-i-c md-link"><a href="http://vault.centos.org/"><span class="md-plain">vault.centos.org</span></a></span><span class="md-plain">），旧配置文件中的地址失效。</span></p>
</li>
</ol>
<h4 class="md-end-block md-heading"><span class="md-plain">三、解决方法（推荐使用root账户操作，两种方案任选）</span></h4>
<h5 class="md-end-block md-heading md-focus"><span class="md-plain">方案一：切换至官方归档仓库（适合需要原汁原味官方源的场景）</span></h5>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">步骤详解</span></strong></span><span class="md-plain">：</span></p>

<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">进入仓库配置目录</span></strong></span></p>
</li>
</ol>


```
cd /etc/yum.repos.d
```


<ol class="ol-list" start="2">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">备份原始配置（关键！防止操作失误）</span></strong></span></p>
</li>
</ol>


```
cp CentOS-Base.repo CentOS-Base.repo.bak
```


<ol class="ol-list" start="3">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">编辑配置文件</span></strong></span></p>
</li>
</ol>
<p class="md-end-block md-p"><span class="md-plain">使用 vi 或 nano 打开文件：</span></p>



```
vi CentOS-Base.repo
```


<ol class="ol-list" start="4">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">替换仓库地址</span></strong></span></p>
</li>
</ol>
<p class="md-end-block md-p"><span class="md-plain">（若系统版本不是 7.9，需将 URL 中的 7.9.2009 替换为对应版本，如 7.8.2003）</span></p>



```bash
# CentOS-Base.repo
#
# The mirror system uses the connecting IP address of the client and the
# update status of each mirror to pick mirrors that are updated to and
# geographically close to the client.  You should use this for CentOS updates
# unless you are manually picking other mirrors.
#
# If the mirrorlist= does not work for you, as a fall back you can try the 
# remarked out baseurl= line instead.
#
#
[base]
name=CentOS-$releasever - Base
#mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=os&infra=$infra
#baseurl=http://mirror.centos.org/centos/$releasever/os/$basearch/
#baseurl=http://vault.centos.org/7.9.2009/x86_64/os/
baseurl=http://vault.centos.org/7.9.2009/os/$basearch/
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-7
#released updates 
[updates]
name=CentOS-$releasever - Updates
#mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=updates&infra=$infra
#baseurl=http://mirror.centos.org/centos/$releasever/updates/$basearch/
#baseurl=http://vault.centos.org/7.9.2009/x86_64/os/
baseurl=http://vault.centos.org/7.9.2009/updates/$basearch/
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-7
#additional packages that may be useful
[extras]
name=CentOS-$releasever - Extras
#mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=extras&infra=$infra
#$baseurl=http://mirror.centos.org/centos/$releasever/extras/$basearch/
#baseurl=http://vault.centos.org/7.9.2009/x86_64/os/
baseurl=http://vault.centos.org/7.9.2009/extras/$basearch/
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-7
#additional packages that extend functionality of existing packages
[centosplus]
name=CentOS-$releasever - Plus
#mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=centosplus&infra=$infra
#baseurl=http://mirror.centos.org/centos/$releasever/centosplus/$basearch/
#baseurl=http://vault.centos.org/7.9.2009/x86_64/os/
baseurl=http://vault.centos.org/7.9.2009/centosplus/$basearch/
gpgcheck=1
enabled=0
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-7
```


<ol class="ol-list" start="5">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">生效配置</span></strong></span></p>
</li>
</ol>
<p class="md-end-block md-p"><span class="md-plain">清理旧缓存并生成新缓存：</span></p>



```
yum clean all && yum makecache
```


<h5 class="md-end-block md-heading"><span class="md-plain">方案二：使用阿里云镜像源（推荐，速度更快、稳定性更高）</span></h5>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">步骤详解</span></strong></span><span class="md-plain">：</span></p>

<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">下载阿里云配置文件</span></strong></span></p>
</li>
</ol>
<p class="md-end-block md-p"><span class="md-plain">若系统安装了 curl 或 wget，直接执行：</span></p>



```
# 方法1：通过 curl 下载  
curl -o /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo  
# 方法2：通过 wget 下载  
wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo
```


<p class="md-end-block md-p"><span class="md-plain">（若上述工具未安装，可手动访问 URL 下载文件，上传至 /etc/yum.repos.d/ 目录）</span></p>

<ol class="ol-list" start="2">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">验证配置有效性</span></strong></span></p>
</li>
</ol>
<p class="md-end-block md-p"><span class="md-plain">查看文件内容，确认包含阿里云镜像地址：</span></p>



```
grep "mirrors.aliyun.com" /etc/yum.repos.d/CentOS-Base.repo
```


<ol class="ol-list" start="3">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">生效配置</span></strong></span></p>
</li>
</ol>
<p class="md-end-block md-p"><span class="md-plain">清理缓存并重建：</span></p>



```
yum clean all && yum makecache
```


<h4 class="md-end-block md-heading"><span class="md-plain">四、验证解决效果</span></h4>
<p class="md-end-block md-p"><span class="md-plain">执行以下命令，若能正常列出仓库信息或安装软件，说明问题解决：</span></p>



```
# 查看仓库列表  
yum repolist  
# 测试安装一个基础工具（如 tree）  
yum install -y tree
```
