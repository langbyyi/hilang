<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">在 Web 安全领域，PHP 代码审计是识别和修复应用程序安全隐患的关键环节。由于 PHP 语言的灵活性和易用性，其代码中常存在各类可被攻击者利用的漏洞。本文将系统梳理 PHP 开发中常见的高风险漏洞类型，剖析其原理、示例及审计要点，为安全审计人员和开发人员提供参考。</div>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">一、代码执行漏洞</h2>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">代码执行漏洞源于对可执行代码的函数滥用，攻击者可通过可控参数注入恶意逻辑。以下为需重点审计的函数及细节：</div>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. eval()</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：将字符串作为 PHP 代码执行，语法为 <code>eval(string $code) : mixed</code>。</li>
 	<li><strong>关键特性</strong>：
<ul class="auto-hide-last-sibling-br">
 	<li>返回值：默认返回<code>NULL</code>，若代码中有<code>return</code>则返回对应值。</li>
 	<li>版本差异：PHP7 中解析错误会抛出<code>ParseError</code>异常；PHP5 中返回<code>FALSE</code>，但后续代码继续执行。</li>
</ul>
</li>
 	<li><strong>漏洞示例</strong>：</li>
</ul>

<h2 class="header-vfC6AV auto-hide-last-sibling-br">二、命令执行漏洞</h2>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">直接调用系统命令的函数，参数可控时可导致命令注入，需逐个审计：</div>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. system()</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：执行命令并输出结果，语法 <code>system(string $command [, int &amp;$return_var ]) : string</code>。</li>
 	<li><strong>示例</strong>：<code>system($_GET[&#039;cmd&#039;]);</code>（传入<code>cmd=whoami</code>执行）。</li>
 	<li><strong>特点</strong>：直接输出命令结果，易被发现。</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">2. exec()</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：执行命令，返回最后一行输出，语法 <code>exec(string $command [, array &amp;$output [, int &amp;$return_var ]]) : string</code>。</li>
 	<li><strong>示例</strong>：<code>exec($_GET[&#039;cmd&#039;], $output); var_dump($output);</code>（结果存于数组）。</li>
 	<li><strong>特点</strong>：不直接输出，需通过<code>$output</code>获取结果。</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">3. shell_exec()</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：通过 shell 执行命令，返回完整输出字符串，语法 <code>shell_exec(string $cmd) : string</code>。</li>
 	<li><strong>示例</strong>：<code>echo shell_exec($_GET[&#039;cmd&#039;]);</code>（执行并输出完整结果）。</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">4. passthru()</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：执行命令并显示原始输出（适合二进制数据），语法 <code>passthru(string $command [, int &amp;$return_var ]) : void</code>。</li>
 	<li><strong>示例</strong>：<code>passthru($_GET[&#039;cmd&#039;]);</code>（直接输出原始结果）。</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">5. pcntl_exec()</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：在当前进程空间执行程序，语法 <code>pcntl_exec(string $path [, array $args [, array $envs ]]) : void</code>。</li>
 	<li><strong>示例</strong>：<code>pcntl_exec(&#039;/bin/sh&#039;, [&#039;-c&#039;, $_GET[&#039;cmd&#039;]]);</code>（执行 shell 命令）。</li>
 	<li><strong>特点</strong>：需<code>pcntl</code>扩展，常用于子进程命令执行。</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">6. popen () 与 proc_open ()</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：
<ul class="auto-hide-last-sibling-br">
 	<li><code>popen(string $command, string $mode) : resource</code>：打开进程文件指针。</li>
 	<li><code>proc_open(string $cmd, array $descriptorspec, array &amp;$pipes [, string $cwd [, array $env [, array $other_options ]]])</code>：更复杂的进程控制。</li>
</ul>
</li>
 	<li><strong>示例</strong>：</li>
</ul>

<h2 class="header-vfC6AV auto-hide-last-sibling-br">三、文件包含漏洞</h2>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">通过包含文件执行代码，所有文件包含函数均需审计：</div>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. include () 与 include_once ()</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：
<ul class="auto-hide-last-sibling-br">
 	<li><code>include(string $filename)</code>：包含并执行文件，出错仅警告。</li>
 	<li><code>include_once(string $filename)</code>：确保文件只被包含一次。</li>
</ul>
</li>
 	<li><strong>漏洞示例</strong>：</li>
</ul>
<div class="relative w-fit custom-code-block-canvas-wrapper w-full">
<div class="code-block-element-YDlfvc light custom-code-block-container--php hide-indicator disable-theme-style custom-code-block-container">

```php
?&gt;</code>
```


</div>

<ul class="auto-hide-last-sibling-br">
 	<li><strong>审计要点</strong>：<code>$filename</code>是否可控，是否限制协议（如禁止<code>php://</code>、<code>file://</code>）。</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">2. require () 与 require_once ()</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：
<ul class="auto-hide-last-sibling-br">
 	<li><code>require(string $filename)</code>：包含文件，出错终止脚本。</li>
 	<li><code>require_once(string $filename)</code>：确保文件只被包含一次。</li>
</ul>
</li>
 	<li><strong>风险与<code>include</code>类似</strong>，但因出错终止特性，漏洞利用更易被发现。</li>
</ul>
