<h3 class="md-end-block md-heading"><span class="md-plain">一、Windows 日志清理</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">1. 系统核心日志</span></h4>
<p class="md-end-block md-p"><span class="md-plain">Windows 系统的日志详细记录了系统运行过程中的各种事件，是追踪入侵行为的重要依据。核心日志主要包括应用程序日志、系统日志和安全日志，它们分别存储在特定路径下：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">应用程序日志</span></strong></span><span class="md-plain">：位于</span><span class="md-pair-s" spellcheck="false"><code>%SystemRoot%\System32\Winevt\Logs\Application.evtx</code></span><span class="md-plain">，记录应用程序运行相关事件，可能包含登录关联的程序活动，如某些恶意程序在登录时自动启动并记录在该日志中。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">系统日志</span></strong></span><span class="md-plain">：路径为</span><span class="md-pair-s" spellcheck="false"><code>%SystemRoot%\System32\Winevt\Logs\System.evtx</code></span><span class="md-plain">，记录系统组件运行事件，例如服务启动、驱动加载等，入侵时对系统服务的操作可能会在此日志中留下痕迹，如异常的服务停止或启动。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">安全日志</span></strong></span><span class="md-plain">：存于</span><span class="md-pair-s" spellcheck="false"><code>%SystemRoot%\System32\Winevt\Logs\Security.evtx</code></span><span class="md-plain">，是记录用户登录 / 注销、权限变更、账户管理等安全事件的核心日志，成功登录、登录失败、特权使用等入侵相关操作都可能被记录其中。</span></p>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-plain">清理这些核心日志可采用多种方法：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">图形界面操作</span></strong></span><span class="md-plain">：通过运行</span><span class="md-pair-s" spellcheck="false"><code>eventvwr.msc</code></span><span class="md-plain">打开事件查看器，在其中找到对应的应用程序日志、系统日志和安全日志，右键选择 “清除日志” 选项。这种方式直观简单，但可能会在系统中留下操作记录。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">命令行清理：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用 PowerShell 命令</span><span class="md-pair-s" spellcheck="false"><code>Clear - EventLog -LogName Application, System, Security -ErrorAction SilentlyContinue</code></span><span class="md-plain">，能够一次性清除应用程序、系统和安全日志。其中，</span><span class="md-pair-s" spellcheck="false"><code>-ErrorAction SilentlyContinue</code></span><span class="md-plain">参数用于忽略可能出现的错误，确保清理过程顺利进行。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">借助</span><span class="md-pair-s" spellcheck="false"><code>wevtutil</code></span><span class="md-plain">工具，它功能强大，可实现日志的查看、检索、导出和清除等多种操作。例如，在命令提示符（CMD）中执行</span><span class="md-pair-s" spellcheck="false"><code>wevtutil cl Application</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>wevtutil cl System</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>wevtutil cl Security</code></span><span class="md-plain">命令，分别对应清除应用程序日志、系统日志和安全日志。若要同时清除这三类日志，可输入</span><span class="md-pair-s" spellcheck="false"><code>wevtutil cl security system application</code></span><span class="md-plain">。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">利用脚本停止日志记录</span></strong></span><span class="md-plain">：通过编写脚本遍历事件日志服务进程（专用</span><span class="md-pair-s" spellcheck="false"><code>svchost.exe</code></span><span class="md-plain">）的线程堆栈，标识并杀死事件日志线程，从而使系统无法收集日志。但需注意，在完成痕迹清理后，应使用</span><span class="md-pair-s" spellcheck="false"><code>net start eventlog</code></span><span class="md-plain">命令恢复日志服务，以避免引起管理员怀疑。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">2. 远程桌面登录记录</span></h4>
<p class="md-end-block md-p"><span class="md-plain">当攻击者通过远程桌面连接目标系统后，会在本地留下相关连接记录，这些记录可能成为被追踪的线索，因此需要及时清理：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">注册表中的连接历史记录：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">位置 1：</span><span class="md-pair-s" spellcheck="false"><code>HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client\Default</code></span><span class="md-plain">，存储默认连接记录。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">位置 2：</span><span class="md-pair-s" spellcheck="false"><code>HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client\Servers</code></span><span class="md-plain">，保存详细连接历史。</span> <span class="md-plain">可通过命令行进行清理，在 CMD 中输入：</span></p>
</li>
</ul>


```cmd
reg delete "HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client\Default" /va /f
reg delete "HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client\Servers" /f
reg add "HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client\Servers"
```


<p class="md-end-block md-p"><span class="md-plain">其中，</span><span class="md-pair-s" spellcheck="false"><code>reg delete</code></span><span class="md-plain">命令用于删除指定注册表项，</span><span class="md-pair-s" spellcheck="false"><code>/va</code></span><span class="md-plain">参数表示删除所有值，</span><span class="md-pair-s" spellcheck="false"><code>/f</code></span><span class="md-plain">参数强制删除，不提示确认。最后一行</span><span class="md-pair-s" spellcheck="false"><code>reg add</code></span><span class="md-plain">命令用于重建空键，避免因注册表项缺失导致系统异常。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">RDP 文件痕迹：用户文档目录下的</span><span class="md-pair-s" spellcheck="false"><code>Default.rdp</code></span><span class="md-plain">文件（</span><span class="md-pair-s" spellcheck="false"><code>%USERPROFILE%\Documents\Default.rdp</code></span><span class="md-plain">）记录了远程桌面连接配置。使用以下命令可将其删除：</span></p>



```cmd
del "%USERPROFILE%\Documents\Default.rdp" /f /q
```


<p class="md-end-block md-p"><span class="md-plain">这里</span><span class="md-pair-s" spellcheck="false"><code>/f</code></span><span class="md-plain">参数强制删除只读文件，</span><span class="md-pair-s" spellcheck="false"><code>/q</code></span><span class="md-plain">参数表示安静模式，不显示确认信息。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">3. 日志伪造与混淆</span></h4>
<p class="md-end-block md-p"><span class="md-plain">单纯清理日志可能会引起管理员的警觉，为了更好地掩盖入侵痕迹，可采用日志伪造与混淆的手段：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">利用</span><span class="md-pair-s" spellcheck="false"><code>eventcreate</code></span><span class="md-plain">命令伪造日志</span></strong></span><span class="md-plain">：该命令可用于创建自定义事件并将其写入事件日志。例如，以管理员权限在 CMD 中执行</span><span class="md-pair-s" spellcheck="false"><code>eventcreate /l Security /t INFORMATION /id 4624 /d &quot;伪造的正常登录事件&quot;</code></span><span class="md-plain">，此命令创建了一条 “成功登录” 的伪造安全日志，其中</span><span class="md-pair-s" spellcheck="false"><code>/l</code></span><span class="md-plain">参数指定日志类型为安全日志，</span><span class="md-pair-s" spellcheck="false"><code>/t</code></span><span class="md-plain">参数设置事件类型为信息，</span><span class="md-pair-s" spellcheck="false"><code>/id</code></span><span class="md-plain">参数指定事件 ID，</span><span class="md-pair-s" spellcheck="false"><code>/d</code></span><span class="md-plain">参数添加事件描述。通过伪造正常的日志事件，可降低真实入侵痕迹的辨识度。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">批量生成无关日志填充</span></strong></span><span class="md-plain">：编写脚本或使用工具批量生成大量无关紧要的日志信息，填充到日志文件中，使真实的入侵痕迹淹没在海量的日志数据中。但在生成这些日志时，需注意模拟正常系统活动的频率和类型，避免因日志内容异常而被察觉。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">二、服务相关日志清理</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">1. Web 服务日志（以 IIS 为例）</span></h4>
<p class="md-end-block md-p"><span class="md-plain">IIS（互联网信息服务）是 Windows 系统中常用的 Web 服务器，其日志记录了大量与网站访问相关的信息，对于入侵痕迹的清理至关重要：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">日志存储位置</span></strong></span><span class="md-plain">：默认路径为</span><span class="md-pair-s" spellcheck="false"><code>%SystemDrive%\inetpub\logs\LogFiles</code></span><span class="md-plain">，按照站点和日期分目录存储，例如</span><span class="md-pair-s" spellcheck="false"><code>W3SVC1</code></span><span class="md-plain">目录对应某个具体网站的日志。日志文件中包含访问 IP、请求路径、状态码等信息，这些信息可能暴露攻击者的访问行为。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">清理方法：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">直接删除日志文件</span></strong></span><span class="md-plain">：由于日志文件在 IIS 服务运行时可能被锁定，因此需要先停止 IIS 服务。在 CMD 中执行</span><span class="md-pair-s" spellcheck="false"><code>net stop w3svc</code></span><span class="md-plain">停止服务，然后使用</span><span class="md-pair-s" spellcheck="false"><code>del &quot;%SystemDrive%\inetpub\logs\LogFiles\W3SVC1\*.*&quot; /f /q</code></span><span class="md-plain">命令删除指定目录下的所有日志文件，最后执行</span><span class="md-pair-s" spellcheck="false"><code>net start w3svc</code></span><span class="md-plain">重新启动 IIS 服务。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">使用工具定向删除</span></strong></span><span class="md-plain">：利用</span><span class="md-pair-s" spellcheck="false"><code>CleanIISLog</code></span><span class="md-plain">工具可定向删除含特定 IP 的记录。在 CMD 中执行</span><span class="md-pair-s" spellcheck="false"><code>CleanIISLog.exe &quot;C:\inetpub\logs\LogFiles\W3SVC1&quot; 192.168.1.100</code></span><span class="md-plain">，该命令可清理指定路径中包含</span><span class="md-pair-s" spellcheck="false"><code>192.168.1.100</code></span><span class="md-plain">这个 IP 地址的日志记录，有效消除攻击者 IP 在日志中的痕迹。</span></p>
</li>
</ul>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">2. 数据库日志（以 SQL Server 为例）</span></h4>
<p class="md-end-block md-p"><span class="md-plain">数据库系统同样会记录大量操作日志，对于保护数据库安全和追踪异常操作具有重要意义。以 SQL Server 为例，其日志主要包括错误日志和事务日志：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">错误日志</span></strong></span><span class="md-plain">：默认路径为</span><span class="md-pair-s" spellcheck="false"><code>%ProgramFiles%\Microsoft SQL Server\MSSQLXX.MSSQLSERVER\MSSQL\Log\ERRORLOG</code></span><span class="md-plain">，记录数据库启动、连接错误等信息。若攻击者在入侵过程中对数据库进行了非法连接或操作，可能会在此日志中留下错误记录。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">事务日志</span></strong></span><span class="md-plain">：默认与数据库文件存于同一目录（如</span><span class="md-pair-s" spellcheck="false"><code>.ldf</code></span><span class="md-plain">文件），记录数据库操作事务，例如数据的插入、更新、删除等操作。入侵时对数据库数据的篡改等操作可能会被记录在事务日志中。</span></p>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-plain">清理方法如下：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">收缩并截断事务日志（需数据库权限）：在 SQL Server 查询分析器中执行以下 SQL 语句：</span></p>



```sql
USE master;
ALTER DATABASE 数据库名 SET RECOVERY SIMPLE;
DBCC SHRINKFILE (事务日志文件名, 1);
ALTER DATABASE 数据库名 SET RECOVERY FULL;
```


<p class="md-end-block md-p"><span class="md-plain">上述语句先将数据库恢复模式设置为简单模式，然后使用</span><span class="md-pair-s" spellcheck="false"><code>DBCC SHRINKFILE</code></span><span class="md-plain">命令收缩事务日志文件，最后再将恢复模式设置回完整模式。通过这种方式，可在一定程度上清理事务日志中的部分历史记录，但不会完全删除所有日志信息。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">直接删除错误日志（需停止 SQL Server 服务）</span></strong></span><span class="md-plain">：在 CMD 中执行</span><span class="md-pair-s" spellcheck="false"><code>net stop MSSQLSERVER</code></span><span class="md-plain">停止 SQL Server 服务，然后使用</span><span class="md-pair-s" spellcheck="false"><code>del &quot;C:\Program Files\Microsoft SQL Server\MSSQLXX.MSSQLSERVER\MSSQL\Log\ERRORLOG&quot; /f /q</code></span><span class="md-plain">命令删除错误日志文件，最后执行</span><span class="md-pair-s" spellcheck="false"><code>net start MSSQLSERVER</code></span><span class="md-plain">重新启动服务。但直接删除错误日志可能会影响数据库的故障排查和审计功能，需谨慎操作。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">3. 计划任务日志</span></h4>
<p class="md-end-block md-p"><span class="md-plain">计划任务日志记录了计划任务的创建、执行、失败等事件，若攻击者利用计划任务维持权限或执行恶意操作，相关痕迹可能会记录在该日志中：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">日志存储位置</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>%SystemRoot%\System32\Winevt\Logs\Microsoft - Windows - TaskScheduler%4Operational.evtx</code></span><span class="md-plain">。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">清理方法</span></strong></span><span class="md-plain">：在 CMD 中执行</span><span class="md-pair-s" spellcheck="false"><code>wevtutil cl &quot;Microsoft - Windows - TaskScheduler/Operational&quot;</code></span><span class="md-plain">命令，即可清理计划任务操作日志，有效消除利用计划任务进行入侵的痕迹。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">三、命令行历史记录清理</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">1. PowerShell 历史记录</span></h4>
<p class="md-end-block md-p"><span class="md-plain">PowerShell 是 Windows 系统中强大的命令行工具，其历史记录可能包含攻击者执行的敏感命令，需要及时清理：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">历史文件位置</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>%USERPROFILE%\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt</code></span><span class="md-plain">，该文件存储了用户在 PowerShell 中执行过的命令历史。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">清理方法：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用 PowerShell 命令</span><span class="md-pair-s" spellcheck="false"><code>Remove - Item &quot;%USERPROFILE%\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt&quot; -Force</code></span><span class="md-plain">，可直接删除历史文件。</span><span class="md-pair-s" spellcheck="false"><code>-Force</code></span><span class="md-plain">参数用于强制删除，即使文件处于只读状态也能删除。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">在当前 PowerShell 会话中，执行</span><span class="md-pair-s" spellcheck="false"><code>Clear - History -Force</code></span><span class="md-plain">命令可清除内存中的命令历史，防止这些命令在会话结束时被写入历史文件。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若要临时禁用 PowerShell 历史记录，可执行</span><span class="md-pair-s" spellcheck="false"><code>$env:PSModulePath = $null</code></span><span class="md-plain">命令，该命令通过间接禁用</span><span class="md-pair-s" spellcheck="false"><code>PSReadLine</code></span><span class="md-plain">模块，阻止命令历史记录功能。但需注意，此操作仅在当前会话有效，重新启动 PowerShell 后历史记录功能将恢复。</span></p>
</li>
</ul>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">2. CMD 历史记录</span></h4>
<p class="md-end-block md-p"><span class="md-plain">CMD 作为 Windows 传统的命令行工具，虽然默认不将命令历史写入文件，但在某些情况下（如使用</span><span class="md-pair-s" spellcheck="false"><code>doskey</code></span><span class="md-plain">命令）可能会记录命令历史，并且一些第三方工具也可能记录 CMD 命令：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">内存缓存中的历史记录</span></strong></span><span class="md-plain">：CMD 的命令历史临时存储在内存缓存中，可通过关闭 CMD 窗口来清除当前会话的命令历史。但如果使用了</span><span class="md-pair-s" spellcheck="false"><code>doskey</code></span><span class="md-plain">命令，其记录的命令历史可能不会因关闭窗口而清除。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">使用</span><span class="md-pair-s" spellcheck="false"><code>doskey</code></span><span class="md-plain">命令时的历史记录清理</span></strong></span><span class="md-plain">：若使用了</span><span class="md-pair-s" spellcheck="false"><code>doskey</code></span><span class="md-plain">命令记录历史，可通过执行</span><span class="md-pair-s" spellcheck="false"><code>doskey /reinstall</code></span><span class="md-plain">命令重新安装</span><span class="md-pair-s" spellcheck="false"><code>doskey</code></span><span class="md-plain">，从而清除已记录的命令历史。此外，也可通过修改环境变量来限制</span><span class="md-pair-s" spellcheck="false"><code>doskey</code></span><span class="md-plain">记录的命令数量，在系统环境变量中找到</span><span class="md-pair-s" spellcheck="false"><code>ComSpec</code></span><span class="md-plain">变量，将其值修改为</span><span class="md-pair-s" spellcheck="false"><code>%SystemRoot%\system32\cmd.exe /K doskey /history:50</code></span><span class="md-plain">（这里</span><span class="md-pair-s" spellcheck="false"><code>/history:50</code></span><span class="md-plain">表示最多记录 50 条命令历史），这样可减少历史记录的数量，降低敏感命令被留存的风险。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">四、避免使用易留痕迹的工具</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">1. 记事本（notepad.exe）替代方案</span></h4>
<p class="md-end-block md-p"><span class="md-plain">记事本是 Windows 系统中常用的文本编辑工具，但使用它可能会在</span><span class="md-pair-s" spellcheck="false"><code>%USERPROFILE%\AppData\Roaming\Microsoft\Windows\Recent</code></span><span class="md-plain">目录下留下最近打开文件的记录，为攻击者带来风险。可采用以下替代工具：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-pair-s" spellcheck="false"><code>type</code></span><span class="md-plain">命令</span></strong></span><span class="md-plain">：用于查看文本文件内容。例如，在 CMD 中执行</span><span class="md-pair-s" spellcheck="false"><code>type C:\test.txt</code></span><span class="md-plain">，即可在命令行中显示</span><span class="md-pair-s" spellcheck="false"><code>C:\test.txt</code></span><span class="md-plain">文件的内容，与使用记事本打开文件查看内容的效果类似，但不会留下文件打开记录。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-pair-s" spellcheck="false"><code>echo</code></span><span class="md-plain">命令</span></strong></span><span class="md-plain">：不仅可用于在命令行中输出文本信息，还可用于简单写入文件。例如，执行</span><span class="md-pair-s" spellcheck="false"><code>echo &quot;这是要写入文件的内容&quot; &gt; C:\new.txt</code></span><span class="md-plain">，该命令将双引号内的内容写入到</span><span class="md-pair-s" spellcheck="false"><code>C:\new.txt</code></span><span class="md-plain">文件中，实现了简单的文件写入功能，避免了使用记事本带来的痕迹问题。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">2. 资源管理器替代方案</span></h4>
<p class="md-end-block md-p"><span class="md-plain">资源管理器在使用过程中会在注册表</span><span class="md-pair-s" spellcheck="false"><code>HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\RecentDocs</code></span><span class="md-plain">中留下文件访问记录，为防止这些记录暴露攻击者的操作，可使用 CMD 命令行进行文件和目录操作：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">目录操作</span></strong></span><span class="md-plain">：使用</span><span class="md-pair-s" spellcheck="false"><code>cd</code></span><span class="md-plain">命令切换目录，例如</span><span class="md-pair-s" spellcheck="false"><code>cd C:\Program Files</code></span><span class="md-plain">可进入</span><span class="md-pair-s" spellcheck="false"><code>C:\Program Files</code></span><span class="md-plain">目录；使用</span><span class="md-pair-s" spellcheck="false"><code>dir</code></span><span class="md-plain">命令查看目录内容，</span><span class="md-pair-s" spellcheck="false"><code>dir C:\</code></span><span class="md-plain">可列出 C 盘根目录下的所有文件和子目录。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">文件操作</span></strong></span><span class="md-plain">：使用</span><span class="md-pair-s" spellcheck="false"><code>del</code></span><span class="md-plain">命令删除文件，如</span><span class="md-pair-s" spellcheck="false"><code>del C:\test.txt</code></span><span class="md-plain">可删除</span><span class="md-pair-s" spellcheck="false"><code>C:\test.txt</code></span><span class="md-plain">文件；使用</span><span class="md-pair-s" spellcheck="false"><code>copy</code></span><span class="md-plain">命令复制文件，</span><span class="md-pair-s" spellcheck="false"><code>copy C:\source.txt C:\destination.txt</code></span><span class="md-plain">可将</span><span class="md-pair-s" spellcheck="false"><code>C:\source.txt</code></span><span class="md-plain">文件复制为</span><span class="md-pair-s" spellcheck="false"><code>C:\destination.txt</code></span><span class="md-plain">。通过这些 CMD 命令行操作，可避免在注册表中留下文件访问记录，降低被追踪的风险。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">五、文件时间属性修改</span></h3>
<p class="md-end-block md-p"><span class="md-plain">文件的创建时间（CreationTime）、修改时间（LastWriteTime）和访问时间（LastAccessTime）可能会暴露文件被篡改的痕迹，因此需要对这些时间属性进行同步调整：</span></p>

<h4 class="md-end-block md-heading"><span class="md-plain">1. 时间属性查看</span></h4>
<p class="md-end-block md-p"><span class="md-plain">在 Windows 系统中，可通过多种方式查看文件的时间属性：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">文件属性窗口</span></strong></span><span class="md-plain">：右键单击文件，选择 “属性”，在弹出的属性窗口中，“详细信息” 选项卡下可查看文件的创建时间、修改时间和访问时间。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">命令行查看</span></strong></span><span class="md-plain">：在 CMD 中使用</span><span class="md-pair-s" spellcheck="false"><code>dir</code></span><span class="md-plain">命令，例如</span><span class="md-pair-s" spellcheck="false"><code>dir C:\test.txt</code></span><span class="md-plain">，命令输出结果中会显示文件的创建时间、修改时间和访问时间。在 PowerShell 中，使用</span><span class="md-pair-s" spellcheck="false"><code>Get - Item</code></span><span class="md-plain">命令，如</span><span class="md-pair-s" spellcheck="false"><code>Get - Item C:\test.txt</code></span><span class="md-plain">，也可获取文件的详细信息，包括时间属性。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">2. 时间属性修改技巧（PowerShell）</span></h4>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">同步目标文件与参照文件的时间：假设要将</span><span class="md-pair-s" spellcheck="false"><code>C:\temp\malware.exe</code></span><span class="md-plain">文件的时间属性修改为与</span><span class="md-pair-s" spellcheck="false"><code>C:\Windows\System32\notepad.exe</code></span><span class="md-plain">文件一致，可在 PowerShell 中执行以下命令：</span></p>



```powershell
$refFile = Get - Item "C:\Windows\System32\notepad.exe"
$targetFile = Get - Item "C:\temp\malware.exe"
$targetFile.CreationTime = $refFile.CreationTime
$targetFile.LastWriteTime = $refFile.LastWriteTime
$targetFile.LastAccessTime = $refFile.LastAccessTime
```


<p class="md-end-block md-p"><span class="md-plain">上述命令先获取参照文件和目标文件的对象，然后分别将目标文件的创建时间、修改时间和访问时间设置为与参照文件相同。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">批量修改目录下文件时间：若要批量修改</span><span class="md-pair-s" spellcheck="false"><code>C:\temp</code></span><span class="md-plain">目录下所有文件的时间属性，使其与参照文件一致，可执行以下命令：</span></p>



```powershell
$refFile = Get - Item "C:\Windows\System32\notepad.exe"
Get - ChildItem "C:\temp\*" -File | ForEach - Object {
    $_.CreationTime = $refFile.CreationTime
    $_.LastWriteTime = $refFile.LastWriteTime
    $_.LastAccessTime = $refFile.LastAccessTime
}
```


<p class="md-end-block md-p"><span class="md-plain">该命令首先获取参照文件对象，然后使用</span><span class="md-pair-s" spellcheck="false"><code>Get - ChildItem</code></span><span class="md-plain">命令获取</span><span class="md-pair-s" spellcheck="false"><code>C:\temp</code></span><span class="md-plain">目录下的所有文件，通过</span><span class="md-pair-s" spellcheck="false"><code>ForEach - Object</code></span></p>
<p class="md-end-block md-p"><span class="md-plain">循环遍历每个文件，并将其时间属性设置为与参照文件相同，从而实现批量修改文件时间属性的目的。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">六、安全删除文件</span></h3>
<p class="md-end-block md-p"><span class="md-plain">普通的</span><span class="md-pair-s" spellcheck="false"><code>del</code></span><span class="md-plain">命令删除文件只是将文件标记为删除，文件数据实际上仍存储在磁盘上，可通过数据恢复工具还原。为了彻底删除文件，防止数据被恢复，需采用专业的安全删除方法：</span></p>

<h4 class="md-end-block md-heading"><span class="md-plain">1. </span><span class="md-pair-s" spellcheck="false"><code>cipher</code></span><span class="md-plain">命令（系统自带）</span></h4>
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>cipher</code></span><span class="md-plain">命令是 Windows 系统自带的加密工具，可用于安全删除文件：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">功能原理</span></strong></span><span class="md-plain">：使用</span><span class="md-pair-s" spellcheck="false"><code>cipher</code></span><span class="md-plain">命令的</span><span class="md-pair-s" spellcheck="false"><code>/W</code></span><span class="md-plain">参数，它会用随机数据多次覆盖已删除文件的磁盘空间，从而防止数据被恢复。覆盖过程通常包括用</span><span class="md-pair-s" spellcheck="false"><code>0x00</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>0xFF</code></span><span class="md-plain">以及随机数依次覆盖，增加数据恢复的难度。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">示例</span></strong></span><span class="md-plain">：在 CMD 中以管理员权限执行</span><span class="md-pair-s" spellcheck="false"><code>cipher /w:D:\</code></span><span class="md-plain">，该命令会对 D 盘上的未使用空间进行覆盖操作，包括已删除文件所占用的空间。执行后，D 盘上已删除文件的磁盘空间将被覆盖三次，有效防止这些文件被恢复软件还原。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading md-focus"><span class="md-plain">2. </span><span class="md-pair-s" spellcheck="false"><code>sdelete</code></span><span class="md-plain">工具（SysinternalsSuite）</span></h4>
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>sdelete</code></span><span class="md-plain">是 SysinternalsSuite 工具包中的一款文件安全删除工具，功能强大：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item md-focus-container">
<p class="md-end-block md-p md-focus"><span class="md-pair-s "><strong><span class="md-plain">功能特点</span></strong></span><span class="md-plain md-expand">：它能够多次覆盖文件内容后再进行删除操作，支持对单个文件、目录以及磁盘空闲空间的清理。可通过设置不同的参数来调整覆盖次数和操作</span></p>
</li>
</ul>
