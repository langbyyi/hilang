<h2 class="wp-block-heading">表存储位置与文件</h2>



<p>MySQL 表默认存储在<code>{MySQL安装目录}/data/{数据库名}/</code>，不同存储引擎的文件不同：</p>



<h3 class="wp-block-heading">MyISAM 引擎（已较少使用）</h3>



<ul class="wp-block-list">
<li><code>表名.frm</code>：存储表结构定义；</li>



<li><code>表名.MYD</code>：存储表数据；</li>



<li><code>表名.MYI</code>：存储表索引。</li>
</ul>



<h3 class="wp-block-heading">InnoDB 引擎（默认推荐）</h3>



<ul class="wp-block-list">
<li>数据和索引通常存储在共享表空间（<code>ibdata1</code>文件）或独立表空间（<code>表名.ibd</code>）；</li>



<li>可通过以下命令查看表空间设置：</li>
</ul>





```
<code>SHOW CREATE TABLE 表名;
</code>
```





<h2 class="wp-block-heading">表引擎管理（选择合适的引擎）</h2>



<p>存储引擎决定了表的存储方式和功能，MySQL 支持多种引擎，常用的有：</p>



<ul class="wp-block-list">
<li><strong>InnoDB</strong>：支持事务、外键，安全性高，适合核心业务（如订单表）；</li>



<li><strong>MyISAM</strong>：不支持事务，查询速度快，适合只读场景（如日志表）。</li>
</ul>



<h3 class="wp-block-heading">查看表引擎</h3>





```
<code>SHOW TABLE STATUS LIKE &#039;表名&#039; \G;  -- \G使结果竖排显示，更易读
</code>
```





<h3 class="wp-block-heading">创建表时指定引擎</h3>





```
<code>CREATE TABLE 表名 (
  id INT PRIMARY KEY,
  name VARCHAR(50)
) ENGINE=InnoDB;  -- 指定InnoDB引擎
</code>
```





<h3 class="wp-block-heading">修改表引擎</h3>





```
<code>ALTER TABLE 表名 ENGINE=MyISAM;  -- 将表引擎改为MyISAM
</code>
```





<h2 class="wp-block-heading">表字符集设置（避免乱码）</h2>



<p>字符集用于存储文字（如中文、英文），推荐使用<code>utf8mb4</code>（支持所有字符，包括 emoji）。</p>



<h3 class="wp-block-heading">查看表字符集</h3>





```
<code>SHOW TABLE STATUS LIKE &#039;表名&#039; \G;
</code>
```





<h3 class="wp-block-heading">创建表时指定字符集</h3>





```
<code>CREATE TABLE 表名 (
  id INT PRIMARY KEY,
  name VARCHAR(50)
) DEFAULT CHARSET=utf8mb4;
</code>
```





<h3 class="wp-block-heading">修改表字符集</h3>





```
<code>-- 修改表整体字符集
ALTER TABLE 表名 CONVERT TO CHARACTER SET utf8mb4;

-- 修改单个字段字符集
ALTER TABLE 表名 MODIFY COLUMN name VARCHAR(50) CHARACTER SET utf8mb4;
</code>
```





<h2 class="wp-block-heading">元数据查询（查询数据库 / 表的信息）</h2>



<p>元数据是描述数据库、表、字段的数据，可通过<code>information_schema</code>系统库查询。</p>



<h3 class="wp-block-heading">查询所有数据库</h3>





```
<code>SELECT SCHEMA_NAME FROM information_schema.SCHEMATA;
-- 等价于SHOW DATABASES;，但更符合SQL标准
</code>
```





<h3 class="wp-block-heading">查询指定数据库的所有表</h3>





```
<code>SELECT TABLE_NAME FROM information_schema.TABLES
WHERE TABLE_SCHEMA = &#039;目标数据库名&#039;;

-- 示例：查询test_db的所有表
SELECT TABLE_NAME FROM information_schema.TABLES
WHERE TABLE_SCHEMA = &#039;test_db&#039;;
</code>
```





<h3 class="wp-block-heading">查询指定表的所有字段</h3>





```
<code>SELECT COLUMN_NAME, DATA_TYPE, IS_NULLABLE, COLUMN_DEFAULT
FROM information_schema.COLUMNS
WHERE TABLE_SCHEMA = &#039;目标数据库名&#039; 
  AND TABLE_NAME = &#039;目标表名&#039;
ORDER BY ORDINAL_POSITION;  -- 按字段顺序排序

-- 示例：查询test_db中users表的字段信息
SELECT COLUMN_NAME, DATA_TYPE, IS_NULLABLE, COLUMN_DEFAULT
FROM information_schema.COLUMNS
WHERE TABLE_SCHEMA = &#039;test_db&#039; 
  AND TABLE_NAME = &#039;users&#039;
ORDER BY ORDINAL_POSITION;</code>
```
