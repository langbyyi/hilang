<h3 class="md-end-block md-heading"><span class="md-plain">1. 文件包含具体指什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">文件包含是编程语言中一种代码复用机制，允许在一个文件中引入另一个文件的内容并执行，常用于模块化开发（如引入公共函数、配置文件等）。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">2. 文件包含分为哪两类？如何区分？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">本地文件包含（LFI）</span></strong></span><span class="md-plain">：只能包含服务器本地存在的文件（如</span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>./config.php</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">远程文件包含（RFI）</span></strong></span><span class="md-plain">：可以包含远程服务器上的文件（如</span><span class="md-pair-s" spellcheck="false"><code>http://attacker.com/evil.php</code></span><span class="md-plain">）。</span></p>
</li>
</ul>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">区分核心</span></strong></span><span class="md-plain">：是否能加载外部服务器的文件，取决于语言配置和漏洞场景。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">4. 文件包含漏洞的原理？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">当文件包含函数的</span><span class="md-pair-s "><strong><span class="md-plain">参数可控</span></strong></span><span class="md-plain">（如通过用户输入动态拼接文件路径），且未对参数进行严格过滤时，攻击者可构造恶意路径，导致包含非预期文件（如本地敏感文件、远程恶意脚本），从而执行恶意代码或读取敏感信息。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">5. 文件包含的危害有哪些？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">读取服务器敏感文件（如配置文件、系统密码文件</span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">执行恶意代码（如包含远程攻击脚本，获取服务器权限）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">触发其他漏洞（如通过包含日志文件注入代码，间接利用漏洞）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">绕过访问控制（如包含受保护的脚本，直接执行内部逻辑）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">6. 文件包含的利用思路是什么？</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">判断类型</span></strong></span><span class="md-plain">：通过构造本地 / 远程路径，确认是 LFI 还是 RFI。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">寻找可包含文件</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">本地：尝试包含敏感文件（</span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>/proc/self/environ</code></span><span class="md-plain">）、日志文件（Apache/Nginx 日志）、临时文件等。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">远程：构造恶意脚本（如 php 一句话木马），通过 URL 包含执行。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">绕过过滤</span></strong></span><span class="md-plain">：若路径被过滤，使用编码（</span><span class="md-pair-s" spellcheck="false"><code>../</code></span><span class="md-plain"> 编码为 </span><span class="md-pair-s" spellcheck="false"><code>%2e%2e%2f</code></span><span class="md-plain">）、截断（</span><span class="md-pair-s" spellcheck="false"><code>%00</code></span><span class="md-plain"> 截断）、路径拼接等方式绕过。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">7. 远程文件包含的必须条件是什么？（含 php 配置开启的两个条件）</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">编程语言支持远程文件包含（如 php 默认允许，Java 需特殊配置）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">php 需开启以下两个配置：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>allow_url_fopen = On</code></span><span class="md-plain">：允许通过 URL 打开远程文件。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>allow_url_include = On</code></span><span class="md-plain">：允许通过 </span><span class="md-pair-s" spellcheck="false"><code>include</code></span><span class="md-plain"> 等函数包含远程文件（php 5.2.0 后默认关闭）。</span></p>
</li>
</ul>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">8. 远程文件包含中，包含 php 文件和包含 txt 文件有什么区别？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">包含 php 文件</span></strong></span><span class="md-plain">：远程服务器的 php 文件解析后返回到当前服务器。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">包含 TXT 文件</span></strong></span><span class="md-plain">：若 TXT 文件中包含 php 代码，是否执行取决于语言配置：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">php 中，TXT 文件的 php 代码默认会被解析执行（只要文件内容符合 php 语法）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">区别在于文件后缀不影响解析，核心是内容是否包含可执行代码。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">9. 网站只有本地包含可以怎么利用？</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">读取敏感文件</span></strong></span><span class="md-plain">：包含 </span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain">、数据库配置文件（</span><span class="md-pair-s" spellcheck="false"><code>config.php</code></span><span class="md-plain">）、SSH 密钥等。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">包含日志文件</span></strong></span><span class="md-plain">：向服务器日志（如访问日志）注入 php 代码（通过 User-Agent 或请求参数），再包含日志文件执行代码。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">利用临时文件</span></strong></span><span class="md-plain">：若目标有文件上传功能（即使不上传恶意文件），可包含上传产生的临时文件（如 php 临时文件 </span><span class="md-pair-s" spellcheck="false"><code>phpXXXXXX</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Session 文件包含</span></strong></span><span class="md-plain">：若 Session 内容可控（如用户输入存入 Session），注入代码后包含 Session 文件（默认路径 </span><span class="md-pair-s" spellcheck="false"><code>(/var/lib/php/sessions/sess_&lt;sessionid&gt;)</code></span><span class="md-plain">）。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">10. 若已知某网站仅存在本地文件包含漏洞，且没有文件上传漏洞及其他漏洞，如何 getshell？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">核心思路：</span><span class="md-pair-s "><strong><span class="md-plain">寻找可写入的文件并注入代码</span></strong></span><span class="md-plain">，再通过 LFI 包含执行：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">利用日志注入</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">向服务器日志（如 Apache 访问日志）注入 php 代码（如在 User-Agent 中添加 </span><span class="md-pair-s" spellcheck="false"><code>&lt;?php eval($_POST[&#039;cmd&#039;]);?&gt;</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">包含该日志文件，执行注入的代码，获取 Shell。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Session 注入</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若用户输入（如用户名）会存入 Session，在输入中注入 php 代码。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">找到 Session 文件路径（如通过报错获取），包含该文件执行代码。</span></p>
</li>
</ul>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">11. 文件包含防御措施有哪些？</span></h3>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">设置包含文件白名单：强制限制包含路径，仅允许包含指定目录下的文件，例如</span><span class="md-pair-s" spellcheck="false"><code>/var/www/include/</code></span><span class="md-plain">。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">路径过滤与校验</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">禁止目录穿越：使用</span><span class="md-pair-s" spellcheck="false"><code>realpath()</code></span><span class="md-plain">或</span><span class="md-pair-s" spellcheck="false"><code>basename()</code></span><span class="md-plain">过滤参数，防止</span><span class="md-pair-s" spellcheck="false"><code>../../</code></span><span class="md-plain">攻击。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">校验文件扩展名：确保包含文件为预期类型，如.php。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">设置 PHP 包含的文件目录：利用 open_basedir 安全机制，限制 PHP 只能访问特定目录。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">禁用危险配置：</span></p>



```
关闭远程包含:在php.ini中设置:
ini
allow_url_include = Off
allow_url_fopen = Off (非必需时关闭)
```


</li>
</ol>
