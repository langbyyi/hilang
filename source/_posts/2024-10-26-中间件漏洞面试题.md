<h3 class="md-end-block md-heading"><span class="md-plain">1. 中间件概念是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">中间件是位于操作系统和应用程序之间的软件层，用于提供通用服务（如通信、数据处理、安全等），简化分布式系统开发。它屏蔽底层复杂性，使应用程序更易集成、扩展和维护，常见于企业级应用、Web 服务和分布式系统。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">2. 常见的中间件有哪些？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Web 服务器</span></strong></span><span class="md-plain">：Apache HTTP Server、Nginx、IIS（Internet Information Services）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">应用服务器</span></strong></span><span class="md-plain">：WebLogic、JBoss、Tomcat、WebSphere</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">消息队列</span></strong></span><span class="md-plain">：RabbitMQ、Kafka、ActiveMQ</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">数据库中间件</span></strong></span><span class="md-plain">：MySQL Proxy、Oracle Data Integrator</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">缓存中间件</span></strong></span><span class="md-plain">：Redis、Memcached</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">3. 中间件漏洞有哪些？（大概介绍）</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">配置漏洞</span></strong></span><span class="md-plain">：错误的权限设置、不安全的默认配置（如开放不必要的端口）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">代码执行漏洞</span></strong></span><span class="md-plain">：通过输入注入恶意代码（如 SQL 注入、命令注入）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">反序列化漏洞</span></strong></span><span class="md-plain">：利用不安全的对象反序列化执行任意代码（如 Weblogic、Fastjson）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">解析漏洞</span></strong></span><span class="md-plain">：文件解析逻辑错误导致恶意文件被执行（如 Nginx、Apache 解析漏洞）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">协议漏洞</span></strong></span><span class="md-plain">：特定协议实现中的缺陷（如 Weblogic 的 T3 协议漏洞）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">4. 遇见过哪些常见中间件的漏洞？</span></h3>
<ul class="ul-list" data-mark="-">

</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Weblogic</span></strong></span><span class="md-plain">：T3 协议反序列化漏洞（CVE-2017-10271、CVE-2020-14882）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">JBoss</span></strong></span><span class="md-plain">：反序列化漏洞（CVE-2017-7504、CVE-2017-12149）。</span></p>

</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">5. Apache HTTPD 多后缀解析漏洞是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">Apache HTTPD 多后缀解析漏洞是指当服务器配置不当（如启用 </span><span class="md-pair-s" spellcheck="false"><code>AddHandler cgi-script .cgi</code></span><span class="md-plain"> 等类似规则）时，攻击者可通过构造包含多个后缀的文件名（如 </span><span class="md-pair-s" spellcheck="false"><code>test.php.aaa</code></span><span class="md-plain">），使 Apache 错误解析文件类型，将本不应被解析为脚本的文件当作脚本执行，从而可能导致远程代码执行的漏洞。</span></p>
<p class="md-end-block md-p"><span class="md-plain">例如，若服务器允许 </span><span class="md-pair-s" spellcheck="false"><code>.php</code></span><span class="md-plain"> 后缀文件被 PHP 解析，攻击者上传 </span><span class="md-pair-s" spellcheck="false"><code>shell.php.jpg</code></span><span class="md-plain"> 时，Apache 可能因错误识别最后一个有效后缀为 </span><span class="md-pair-s" spellcheck="false"><code>.php</code></span><span class="md-plain">，而将该文件作为 PHP 脚本执行，触发漏洞。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">6. Apache 多后缀解析漏洞原理是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">Apache 解析文件时，会从文件名的最后一个 </span><span class="md-pair-s" spellcheck="false"><code>.</code></span><span class="md-plain"> 开始向前识别，寻找第一个在配置中定义过的 “有效后缀”，并按该后缀对应的规则解析文件。</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">若服务器配置了类似 </span><span class="md-pair-s" spellcheck="false"><code>AddType application/x-httpd-php .php</code></span><span class="md-plain">（即 </span><span class="md-pair-s" spellcheck="false"><code>.php</code></span><span class="md-plain"> 后缀文件由 PHP 解析），攻击者构造 </span><span class="md-pair-s" spellcheck="false"><code>test.php.abc</code></span><span class="md-plain"> 时，Apache 会忽略 </span><span class="md-pair-s" spellcheck="false"><code>.abc</code></span><span class="md-plain">（未配置关联解析规则），识别 </span><span class="md-pair-s" spellcheck="false"><code>.php</code></span><span class="md-plain"> 为有效后缀，从而将文件作为 PHP 脚本解析。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">核心原因是 Apache 的 “后缀优先级” 机制：优先匹配最左侧的有效后缀，而非最后一个后缀，导致多后缀文件名被错误解析。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">7. Apache HTTPD 换行解析漏洞是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">Apache HTTPD 换行解析漏洞（CVE-2017-15715）是存在于 Apache 2.4.0~2.4.29 版本中的漏洞。当服务器启用 </span><span class="md-pair-s" spellcheck="false"><code>mod CGI</code></span><span class="md-plain"> 或 </span><span class="md-pair-s" spellcheck="false"><code>mod cgid</code></span><span class="md-plain"> 模块时，攻击者可通过在 URL 中插入换行符（</span><span class="md-pair-s" spellcheck="false"><code>%0A</code></span><span class="md-plain">）构造特殊请求，绕过 Apache 的 CGI 路径验证规则，执行任意 CGI 脚本，进而可能导致远程命令执行。</span></p>
<p class="md-end-block md-p"><span class="md-plain">例如，攻击者访问 </span><span class="md-pair-s" spellcheck="false"><code>http://example.com/cgi-bin/test%0A.sh</code></span><span class="md-plain"> 时，换行符会干扰 Apache 对路径的解析，使服务器误将恶意脚本当作合法 CGI 程序执行。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">8. Apache SSI 远程命令执行漏洞是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">SSI（Server-Side Includes，服务器端包含）是 Apache 的一项功能，允许在 HTML 页面中嵌入指令（如 </span><span class="md-pair-s" spellcheck="false"><code>&lt;!--#exec cmd=&quot;ls&quot;--&gt;</code></span><span class="md-plain">），服务器解析页面时会执行这些指令并将结果嵌入页面。</span> <span class="md-plain">Apache SSI 远程命令执行漏洞是指当服务器启用 SSI 功能（如配置 </span><span class="md-pair-s" spellcheck="false"><code>AddType text/html .shtml</code></span><span class="md-plain"> 且允许 </span><span class="md-pair-s" spellcheck="false"><code>.shtml</code></span><span class="md-plain"> 等文件使用 SSI 指令），且攻击者能上传包含 SSI 指令的文件（如 </span><span class="md-pair-s" spellcheck="false"><code>.shtml</code></span><span class="md-plain"> 或被错误解析为 SSI 类型的文件）时，可通过嵌入 </span><span class="md-pair-s" spellcheck="false"><code>exec</code></span><span class="md-plain"> 等指令执行系统命令，实现远程命令执行。</span></p>
<p class="md-end-block md-p"><span class="md-plain">例如，攻击者上传 </span><span class="md-pair-s" spellcheck="false"><code>shell.shtml</code></span><span class="md-plain">，内容为 </span><span class="md-pair-s" spellcheck="false"><code>&lt;!--#exec cmd=&quot;id&quot;--&gt;</code></span><span class="md-plain">，服务器解析该文件时会执行 </span><span class="md-pair-s" spellcheck="false"><code>id</code></span><span class="md-plain"> 命令并返回结果，导致漏洞利用。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">9. Nginx 解析漏洞是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">Nginx 解析漏洞是指攻击者通过构造特殊文件名或请求路径，利用 Nginx 与后端 PHP-FPM 交互时的解析逻辑缺陷，使服务器将非 PHP 文件（如图片）当作 PHP 脚本执行，从而实现远程代码执行的漏洞。常见类型包括：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">路径截断漏洞</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞条件：Nginx 配置中存在 </span><span class="md-pair-s" spellcheck="false"><code>location ~ \.php$</code></span><span class="md-plain"> 规则，且使用 </span><span class="md-pair-s" spellcheck="false"><code>fastcgi_pass</code></span><span class="md-plain"> 转发请求到 PHP-FPM。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击方式：请求 </span><span class="md-pair-s" spellcheck="false"><code>http://example.com/test.jpg/.php</code></span><span class="md-plain">，Nginx 因正则匹配规则将路径截断为 </span><span class="md-pair-s" spellcheck="false"><code>test.jpg</code></span><span class="md-plain">，但 PHP-FPM 可能将其解析为 </span><span class="md-pair-s" spellcheck="false"><code>test.jpg.php</code></span><span class="md-plain">，导致图片文件被当作 PHP 执行。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">空字节漏洞（CVE-2009-2625）</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞条件：Nginx ≤ 0.8.37 版本。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击方式：请求 </span><span class="md-pair-s" spellcheck="false"><code>http://example.com/shell.php%00.jpg</code></span><span class="md-plain">，空字节（</span><span class="md-pair-s" spellcheck="false"><code>%00</code></span><span class="md-plain">）会截断 Nginx 的路径处理逻辑，使服务器将 </span><span class="md-pair-s" spellcheck="false"><code>shell.php</code></span><span class="md-plain"> 作为文件名解析，而忽略 </span><span class="md-pair-s" spellcheck="false"><code>.jpg</code></span><span class="md-plain"> 后缀，触发 PHP 解析。</span></p>
</li>
</ul>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">10. Nginx 文件名逻辑漏洞是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">Nginx 文件名逻辑漏洞是指攻击者通过构造特殊符号（如分号、点等），利用 Nginx 对文件名的处理逻辑缺陷，绕过前端过滤机制，使恶意文件被错误解析为可执行脚本。常见场景包括：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">分号截断漏洞</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击方式：上传 </span><span class="md-pair-s" spellcheck="false"><code>shell.php;.jpg</code></span><span class="md-plain">，Nginx 在处理分号时可能将其视为 URI 参数分隔符，导致实际传递给 PHP-FPM 的文件名为 </span><span class="md-pair-s" spellcheck="false"><code>shell.php</code></span><span class="md-plain">，而 </span><span class="md-pair-s" spellcheck="false"><code>.jpg</code></span><span class="md-plain"> 被忽略，触发 PHP 解析。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">点号处理漏洞</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击方式：上传 </span><span class="md-pair-s" spellcheck="false"><code>shell.php.</code></span><span class="md-plain">（末尾带点），某些配置下 Nginx 会将其规范化为 </span><span class="md-pair-s" spellcheck="false"><code>shell.php</code></span><span class="md-plain">，导致文件被错误解析为 PHP。</span></p>
</li>
</ul>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">11. IIS 7.5 解析漏洞是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">IIS 7.5 解析漏洞是指 Internet Information Services (IIS) 7.5 版本中存在的文件解析逻辑缺陷，攻击者可通过构造特殊文件名，使服务器将非 ASP 文件当作 ASP 脚本执行，导致任意代码注入。主要类型包括：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">分号截断漏洞（CVE-2011-1260）</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞条件：IIS 7.5 + FastCGI 模式。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击方式：请求 </span><span class="md-pair-s" spellcheck="false"><code>http://example.com/shell.asp;.jpg</code></span><span class="md-plain">，IIS 会忽略分号及后面的内容，将 </span><span class="md-pair-s" spellcheck="false"><code>shell.asp</code></span><span class="md-plain"> 作为文件名解析，导致恶意代码被执行。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">PHP 解析漏洞</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞条件：IIS 7.5 + PHP 配置错误。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击方式：上传 </span><span class="md-pair-s" spellcheck="false"><code>shell.php.jpg</code></span><span class="md-plain">，若 IIS 配置中 </span><span class="md-pair-s" spellcheck="false"><code>.php</code></span><span class="md-plain"> 处理规则优先级高于 </span><span class="md-pair-s" spellcheck="false"><code>.jpg</code></span><span class="md-plain">，则该文件可能被当作 PHP 执行。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">.user.ini 漏洞</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞条件：PHP 配置允许 </span><span class="md-pair-s" spellcheck="false"><code>user_ini.filename</code></span><span class="md-plain">（默认 </span><span class="md-pair-s" spellcheck="false"><code>.user.ini</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击方式：上传 </span><span class="md-pair-s" spellcheck="false"><code>.user.ini</code></span><span class="md-plain"> 文件，内容为 </span><span class="md-pair-s" spellcheck="false"><code>auto_prepend_file=shell.jpg</code></span><span class="md-plain">，然后上传包含 PHP 代码的 </span><span class="md-pair-s" spellcheck="false"><code>shell.jpg</code></span><span class="md-plain">，使服务器自动加载并执行该文件。</span></p>
</li>
</ul>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">12. Weblogic 里什么是 T3 协议？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">T3 是 Oracle WebLogic Server 内部使用的</span><span class="md-pair-s "><strong><span class="md-plain">二进制通信协议</span></strong></span><span class="md-plain">，用于实现服务器间的远程方法调用（RMI）和集群通信。它支持对象序列化和反序列化，允许 Java 对象在网络间传输，默认监听端口为 </span><span class="md-pair-s "><strong><span class="md-plain">7001</span></strong></span><span class="md-plain">。</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">核心功能</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">实现 WebLogic 集群节点间的数据同步和负载均衡；</span></p>
</li>
 	<li class="md-list-item">
