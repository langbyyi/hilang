<h3 class="md-end-block md-heading"><span class="md-plain">1. 安全加固都有哪些点需要加固？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">网络层</span></strong></span><span class="md-plain">：防火墙配置（白名单策略）、入侵检测 / 防御系统（IDS/IPS）部署、端口封禁（关闭非必要端口）、VPN 加密、DDoS 防护。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">系统层</span></strong></span><span class="md-plain">：账号安全（密码策略、禁用空密码、删除冗余账号）、权限控制（最小权限原则）、补丁更新（高危漏洞修复）、日志审计开启、恶意代码防护（杀毒软件）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">应用层</span></strong></span><span class="md-plain">：Web 漏洞修复（SQL 注入、XSS 等）、API 接口校验（签名 / Token）、文件上传限制、会话管理（超时机制、Cookie 安全属性）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">数据层</span></strong></span><span class="md-plain">：数据加密（传输加密 TLS/SSL、存储加密）、数据备份（定期备份 + 异地存储）、敏感数据脱敏。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">物理层</span></strong></span><span class="md-plain">：设备物理访问控制（门禁、监控）、硬件冗余（防单点故障）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">2. Linux 系统安全加固需要注意的内容有哪些？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">账号安全</span></strong></span><span class="md-plain">：禁用 root 直接 SSH 登录（用 sudo 提权）、设置强密码策略（</span><span class="md-pair-s" spellcheck="false"><code>/etc/pam.d/system-auth</code></span><span class="md-plain">配置）、删除无用账号（如 nobody 权限收紧）、限制 su 命令使用（</span><span class="md-pair-s" spellcheck="false"><code>/etc/pam.d/su</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">服务与端口</span></strong></span><span class="md-plain">：关闭不必要服务（</span><span class="md-pair-s" spellcheck="false"><code>systemctl disable 服务名</code></span><span class="md-plain">）、用</span><span class="md-pair-s" spellcheck="false"><code>netstat/lsof</code></span><span class="md-plain">检查开放端口并封禁（</span><span class="md-pair-s" spellcheck="false"><code>iptables</code></span><span class="md-plain">规则）、隐藏系统版本信息（如修改 Apache/Nginx 配置）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">文件权限</span></strong></span><span class="md-plain">：关键文件权限锁定（</span><span class="md-pair-s" spellcheck="false"><code>chmod 600 /etc/passwd</code></span><span class="md-plain">）、禁用 SUID/SGID 权限（</span><span class="md-pair-s" spellcheck="false"><code>find / -perm +6000</code></span><span class="md-plain">排查）、</span><span class="md-pair-s" spellcheck="false"><code>/tmp</code></span><span class="md-plain">目录挂载为不可执行（</span><span class="md-pair-s" spellcheck="false"><code>/etc/fstab</code></span><span class="md-plain">配置）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p md-focus"><span class="md-pair-s "><strong><span class="md-plain">日志与审计</span></strong></span><span class="md-plain">：开启</span><span class="md-pair-s" spellcheck="false"><code>rsyslog</code></span><span class="md-plain">日志集中管理、配置</span><span class="md-pair-s" spellcheck="false"><code>auditd</code></span><span class="md-plain">记录关键操作（如文件修改、账号登录）、定期备份日志。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">其他</span></strong></span><span class="md-plain">：启用 SELinux/AppArmor（强制访问控制）、定期更新系统补丁（</span><span class="md-pair-s" spellcheck="false"><code>yum update</code></span><span class="md-plain">/</span><span class="md-pair-s" spellcheck="false"><code>apt upgrade</code></span><span class="md-plain">）、部署主机入侵检测（HIDS）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">3. 安全加固怎么操作的？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">步骤 1：资产梳理与风险评估</span></strong></span> <span class="md-plain">清点所有资产（服务器、网络设备、应用），通过扫描（如 Nessus）识别漏洞与风险点，优先级排序。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">步骤 2：制定加固方案</span></strong></span> <span class="md-plain">结合业务需求，针对风险点制定可落地的加固措施（如 “关闭 UDP 137 端口”“修复 Apache Struts2 漏洞”）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">步骤 3：分层实施加固</span></strong></span> <span class="md-plain">按 “网络→系统→应用→数据” 顺序执行，例如：配置防火墙规则→修改 Linux 账号策略→修复 Web 漏洞→加密敏感数据。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">步骤 4：验证与复盘</span></strong></span> <span class="md-plain">加固后通过扫描工具复查漏洞是否修复，模拟攻击验证效果，记录未解决问题。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">步骤 5：持续监控与迭代</span></strong></span> <span class="md-plain">定期复查加固效果，跟进新漏洞（如 CVE 漏洞），更新加固策略。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">4. 如何防范 Webshell？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">预防阶段</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">限制文件上传：校验文件类型（白名单 + 后缀名，拒绝</span><span class="md-pair-s" spellcheck="false"><code>.php/.jsp</code></span><span class="md-plain">等可执行文件）、禁用上传目录执行权限（如 Nginx 配置</span><span class="md-pair-s" spellcheck="false"><code>location /upload { deny all; }</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">权限控制：Web 服务账号仅赋予最小权限（如禁止读写</span><span class="md-pair-s" spellcheck="false"><code>/etc/passwd</code></span><span class="md-plain">），分离 Web 目录与系统目录。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">代码防护：过滤文件上传接口的恶意参数（如</span><span class="md-pair-s" spellcheck="false"><code>../</code></span><span class="md-plain">路径穿越），使用 WAF 拦截 Webshell 特征（如</span><span class="md-pair-s" spellcheck="false"><code>eval</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>system</code></span><span class="md-plain">函数）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">检测阶段</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">日志监控：跟踪</span><span class="md-pair-s" spellcheck="false"><code>access.log</code></span><span class="md-plain">中异常请求（如访问</span><span class="md-pair-s" spellcheck="false"><code>/upload/shell.php</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">工具扫描：用 ClamAV、D 盾等扫描 Web 目录，检测特征码；定期对比文件哈希（基线比对）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">应急阶段</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">发现后立即隔离删除 Webshell，检查是否有衍生文件（如</span><span class="md-pair-s" spellcheck="false"><code>crontab</code></span><span class="md-plain">定时任务），溯源入侵路径（如通过日志找上传 IP）。</span></p>
</li>
</ul>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">5. 设备巡检的流程是什么？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">准备阶段</span></strong></span><span class="md-plain">：明确巡检范围（服务器、网络设备、安全设备）、工具（如 Zabbix 监控、Nessus 扫描）、指标（CPU 使用率≤80%、无高危漏洞）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">执行阶段</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">硬件检查：设备运行状态（温度、风扇）、接口连接（是否松动）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">系统检查：进程 / 服务（无异常进程）、日志（登录失败、错误日志）、磁盘占用（是否满容）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">安全检查：漏洞扫描（是否有未修复高危漏洞）、配置合规性（防火墙规则是否过期）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">记录与分析</span></strong></span><span class="md-plain">：汇总异常项（如 “服务器 1 CPU 持续 90%”“设备 2 存在 CVE-2023-XXX 漏洞”），评估风险等级。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">报告与整改</span></strong></span><span class="md-plain">：提交巡检报告（含问题清单、建议方案），跟踪开发 / 运维修复，验证修复效果。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">归档与优化</span></strong></span><span class="md-plain">：存档报告，更新巡检指标（如新增 “是否开启日志审计”）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">6. 应急响应标准流程是什么？每个阶段需要做哪些事情？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">标准流程为 “</span><span class="md-pair-s "><strong><span class="md-plain">准备→检测→遏制→根除→恢复→总结</span></strong></span><span class="md-plain">”：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">准备阶段</span></strong></span><span class="md-plain">：制定应急预案（明确角色分工、工具清单）、储备应急工具（如取证工具 FTK、病毒库）、团队培训（模拟演练）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">检测阶段</span></strong></span><span class="md-plain">：确认事件真实性（如 “服务器被植入 Webshell”）、初步分析影响范围（涉及哪些业务、数据）、判断攻击类型（勒索 / 挖矿 / 数据泄露）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">遏制阶段</span></strong></span><span class="md-plain">：阻止事态扩大（如断网隔离受影响服务器、封禁攻击 IP、暂停漏洞接口）、保存现场证据（内存快照、进程列表）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">根除阶段</span></strong></span><span class="md-plain">：清除恶意代码（删除木马、 Webshell）、修复漏洞（如打补丁、改配置）、移除攻击入口（如删除新增恶意账号）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">恢复阶段</span></strong></span><span class="md-plain">：验证系统干净（扫描无残留）、逐步恢复服务（先测试环境，再生产）、监控是否复现异常。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">总结阶段</span></strong></span><span class="md-plain">：复盘攻击路径（如 “通过 SQL 注入上传 Webshell”）、优化预案（补充检测指标）、输出报告（含时间线、整改建议）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">7. 应急响应遏制阶段需要做哪些事情？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">隔离受影响资产</span></strong></span><span class="md-plain">：断开被入侵服务器的网络（物理拔线或防火墙阻断），避免攻击扩散到内网。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">限制恶意行为</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">终止异常进程（如挖矿进程</span><span class="md-pair-s" spellcheck="false"><code>minerd</code></span><span class="md-plain">）、kill 恶意连接（</span><span class="md-pair-s" spellcheck="false"><code>netstat -anp</code></span><span class="md-plain">找可疑 IP，</span><span class="md-pair-s" spellcheck="false"><code>iptables -A INPUT -s 攻击IP -j DROP</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">冻结可疑账号（如新增的</span><span class="md-pair-s" spellcheck="false"><code>admin123</code></span><span class="md-plain">账号），禁用漏洞相关服务（如临时关闭文件上传接口）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">保存关键证据</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">记录当前状态（进程列表</span><span class="md-pair-s" spellcheck="false"><code>ps -ef</code></span><span class="md-plain">、网络连接</span><span class="md-pair-s" spellcheck="false"><code>netstat</code></span><span class="md-plain">、登录日志）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">内存快照（用</span><span class="md-pair-s" spellcheck="false"><code>dd</code></span><span class="md-plain">命令 dump 内存：</span><span class="md-pair-s" spellcheck="false"><code>dd if=/dev/mem of=/tmp/memdump bs=1M</code></span><span class="md-plain">），避免重启丢失证据。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">临时止损</span></strong></span><span class="md-plain">：如 “关闭服务器的 80 端口” 阻止 Web 攻击，“暂停数据库写入” 防止数据被篡改。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">8. Apache/Nginx/IIS 日志在什么位置？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Apache</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">CentOS：</span><span class="md-pair-s" spellcheck="false"><code>/etc/httpd/logs/</code></span><span class="md-plain">（含</span><span class="md-pair-s" spellcheck="false"><code>access_log</code></span><span class="md-plain">访问日志、</span><span class="md-pair-s" spellcheck="false"><code>error_log</code></span><span class="md-plain">错误日志）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Ubuntu：</span><span class="md-pair-s" spellcheck="false"><code>/var/log/apache2/</code></span><span class="md-plain">。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Nginx</span></strong></span><span class="md-plain">：默认</span><span class="md-pair-s" spellcheck="false"><code>/var/log/nginx/</code></span><span class="md-plain">（</span><span class="md-pair-s" spellcheck="false"><code>access.log</code></span><span class="md-plain">访问日志、</span><span class="md-pair-s" spellcheck="false"><code>error.log</code></span><span class="md-plain">错误日志，可在</span><span class="md-pair-s" spellcheck="false"><code>nginx.conf</code></span><span class="md-plain">中自定义路径）。</span></p>
</li>
 	<li class="md-list-item">

<h3 class="md-end-block md-heading"><span class="md-plain">15. 如何检测内存马？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">内存马是驻留内存的恶意代码（不落地磁盘），检测方法：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">内存 dump 分析</span></strong></span><span class="md-plain">：用 Volatility 工具（</span><span class="md-pair-s" spellcheck="false"><code>volatility -f memdump.raw --profile=XXX pslist</code></span><span class="md-plain">）扫描可疑内存模块、未签名代码。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">进程行为监控</span></strong></span><span class="md-plain">：通过</span><span class="md-pair-s" spellcheck="false"><code>strace</code></span><span class="md-plain">（Linux）跟踪进程系统调用（如异常</span><span class="md-pair-s" spellcheck="false"><code>execve</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>connect</code></span><span class="md-plain">），或 Windows 的 Process Explorer 看进程加载的 DLL（是否有未知 DLL）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Web 容器检查</span></strong></span><span class="md-plain">：Java 内存马（如 Filter 马）可通过</span><span class="md-pair-s" spellcheck="false"><code>arthas</code></span><span class="md-plain">查看类加载器（</span><span class="md-pair-s" spellcheck="false"><code>sc -d *Filter</code></span><span class="md-plain">），检查是否有异常 Filter；Tomcat 查</span><span class="md-pair-s" spellcheck="false"><code>Catalina</code></span><span class="md-plain">日志的异常 Servlet 注册。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">动态调试</span></strong></span><span class="md-plain">：用 GDB（Linux）/x64dbg（Windows）附加进程，查看内存中是否有</span><span class="md-pair-s" spellcheck="false"><code>shellcode</code></span><span class="md-plain">（如</span><span class="md-pair-s" spellcheck="false"><code>\x90\x90\x31\xc0</code></span><span class="md-plain">）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">16. 什么是内存马？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">内存马是</span><span class="md-pair-s "><strong><span class="md-plain">驻留于目标系统内存中、不写入磁盘</span></strong></span><span class="md-plain">的恶意代码，通过注入进程内存（如 Web 容器、系统进程）实现隐蔽控制。</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">特点：传统文件扫描无法检测，隐蔽性极强。</span></p>
</li>
 	<li class="md-list-item">
