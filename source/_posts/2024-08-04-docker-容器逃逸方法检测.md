<h2 class="md-end-block md-heading"><span class="md-plain">一、判断是否为容器环境</span></h2>
<p class="md-end-block md-p"><span class="md-plain">通过检查 </span><span class="md-pair-s" spellcheck="false"><code>/proc/1/cgroup</code></span><span class="md-plain"> 中是否包含 </span><span class="md-pair-s" spellcheck="false"><code>docker</code></span><span class="md-plain"> 相关标识，判断是否处于 Docker 容器环境：</span></p>



```bash
cat /proc/1/cgroup | grep -qi docker && echo "Is Docker" || echo "Not Docker"
```


<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Is Docker</code></span><span class="md-plain"> ：当前为 Docker 容器环境</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Not Docker</code></span><span class="md-plain"> ：当前非 Docker 容器环境</span></p>
</li>
</ul>
<h2 class="md-end-block md-heading"><span class="md-plain">二、容器逃逸检测分类与方法</span></h2>
<p class="md-end-block md-p"><span class="md-plain">容器逃逸风险主要源于 </span><span class="md-pair-s "><strong><span class="md-plain">不安全配置、程序漏洞、内核漏洞</span></strong></span><span class="md-plain"> ，对应检测方式如下：</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">（一）不安全配置检测</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">1. 特权模式检测</span></h4>
<p class="md-end-block md-p"><span class="md-plain">检查容器是否以特权模式运行（特权模式下容器权限过高，易引发逃逸）：</span></p>



```bash
cat /proc/self/status | grep -qi "0000003fffffffff" && echo "Is privileged mode" || echo "Not privileged mode"
```


<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Is privileged mode</code></span><span class="md-plain"> ：容器为特权模式</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Not privileged mode</code></span><span class="md-plain"> ：容器非特权模式</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">2. Docker Socket 挂载检测</span></h4>
<p class="md-end-block md-p"><span class="md-plain">Docker Socket（</span><span class="md-pair-s" spellcheck="false"><code>docker.sock</code></span><span class="md-plain"> ）是 Docker 守护进程的通信接口，若被挂载到容器内，攻击者可借此控制宿主机 Docker ：</span></p>



```bash
ls /var/run/ | grep -qi docker.sock && echo "Docker Socket is mounted." || echo "Docker Socket is not mounted."
```


<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Docker Socket is mounted.</code></span><span class="md-plain"> ：容器内挂载了 Docker Socket</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Docker Socket is not mounted.</code></span><span class="md-plain"> ：容器内未挂载 Docker Socket</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">3. procfs 挂载检测</span></h4>
<p class="md-end-block md-p"><span class="md-pair-s" spellcheck="false"><code>procfs</code></span><span class="md-plain"> 是 Linux 伪文件系统，若容器内挂载宿主机 </span><span class="md-pair-s" spellcheck="false"><code>procfs</code></span><span class="md-plain"> ，可能被利用实现逃逸（如篡改内核参数 ）：</span></p>



```bash
find / -name core_pattern 2>/dev/null | wc -l | grep -q 2 && echo "Procfs is mounted." || echo "Procfs is not mounted."
```


<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Procfs is mounted.</code></span><span class="md-plain"> ：容器内挂载了宿主机 </span><span class="md-pair-s" spellcheck="false"><code>procfs</code></span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Procfs is not mounted.</code></span><span class="md-plain"> ：容器内未挂载宿主机 </span><span class="md-pair-s" spellcheck="false"><code>procfs</code></span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">4. 宿主机根目录挂载检测</span></h4>
<p class="md-end-block md-p"><span class="md-plain">若容器内挂载宿主机根目录，攻击者可直接操作宿主机文件系统：</span></p>



```bash
find / -name passwd 2>/dev/null | grep /etc/passwd | wc -l | grep -q 7 && echo "Root directory is mounted." || echo "Root directory is not mounted."
```


<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Root directory is mounted.</code></span><span class="md-plain"> ：容器内挂载了宿主机根目录</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Root directory is not mounted.</code></span><span class="md-plain"> ：容器内未挂载宿主机根目录</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">5. Docker Remote API 未授权访问检测</span></h4>
<p class="md-end-block md-p"><span class="md-plain">Docker Remote API 若未授权开放，攻击者可远程控制 Docker 服务（需结合宿主机 IP 检测 ）：</span></p>



```bash
IP=`hostname -i | awk -F. '{print $1 "." $2 "." $3 ".1"}'` && timeout 3 bash -c "echo >/dev/tcp/$IP/2375" > /dev/null 2>&1 && echo "Docker Remote API Is Enabled." || echo "Docker Remote API is Closed."
```


<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Docker Remote API Is Enabled.</code></span><span class="md-plain"> ：存在未授权访问风险</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">输出 </span><span class="md-pair-s" spellcheck="false"><code>Docker Remote API is Closed.</code></span><span class="md-plain"> ：未开放未授权访问</span></p>
</li>
</ul>

<h2 class="md-end-block md-heading"><span class="md-plain">三、容器逃逸检测脚本</span></h2>
<p class="md-end-block md-p"><span class="md-plain">可通过现成脚本批量检测容器逃逸风险，使用方式如下：</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">1. 项目地址</span></h3>
<p class="md-end-block md-p"><span class="md-link md-pair-s" spellcheck="false"><a href="https://github.com/teamssix/container-escape-check">https://github.com/teamssix/container-escape-check</a></span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">2. 容器内直接执行（需 </span><span class="md-pair-s" spellcheck="false"><code>wget</code></span><span class="md-plain"> ）</span></h3>


```bash
wget https://raw.githubusercontent.com/teamssix/container-escape-check/main/container-escape-check.sh -O -| bash
```


<h3 class="md-end-block md-heading"><span class="md-plain">3. 无 </span><span class="md-pair-s" spellcheck="false"><code>wget</code></span><span class="md-plain"> 时的替代方案</span></h3>
<p class="md-end-block md-p"><span class="md-plain">本地克隆仓库后，将脚本上传至容器执行：</span></p>



```bash
git clone https://github.com/teamssix/container-escape-check.git  
cd container-escape-check  
chmod +x container-escape-check.sh  
./container-escape-check.sh
```
