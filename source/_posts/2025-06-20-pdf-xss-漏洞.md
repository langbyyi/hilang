<h2 class="md-end-block md-heading"><span class="md-plain">一、PDF XSS 漏洞核心原理</span></h2>
<p class="md-end-block md-p"><span class="md-plain">PDF XSS（PDF 跨站脚本）漏洞是指攻击者通过在 PDF 文件中植入恶意 JavaScript 代码，利用 PDF 阅读器或在线预览工具的安全缺陷，实现代码执行的一类漏洞。与传统网页 XSS 不同，PDF XSS 的载体是 PDF 文件，攻击场景更隐蔽，且依赖于 PDF 处理工具的解析逻辑。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">1. PDF 文件的动态特性基础</span></h3>
<p class="md-end-block md-p"><span class="md-plain">PDF 格式并非纯静态文件，其规范支持多种动态交互功能，主要包括：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">嵌入式 JavaScript</span></strong></span><span class="md-plain">：通过</span><span class="md-pair-s" spellcheck="false"><code>/JS</code></span><span class="md-plain">或</span><span class="md-pair-s" spellcheck="false"><code>/JavaScript</code></span><span class="md-plain">字典定义文档级脚本，可在打开 PDF 时自动执行</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">表单事件响应</span></strong></span><span class="md-plain">：支持</span><span class="md-pair-s" spellcheck="false"><code>OnFocus</code></span><span class="md-plain">、</span><span class="md-pair-s" spellcheck="false"><code>OnMouseUp</code></span><span class="md-plain">等事件绑定脚本</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">注释与链接</span></strong></span><span class="md-plain">：可在注释或链接中嵌入脚本，点击时触发执行</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">字体与渲染逻辑</span></strong></span><span class="md-plain">：部分阅读器对字体的特殊处理可能成为代码注入点</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">2. 漏洞触发的核心条件</span></h3>
<p class="md-end-block md-p"><span class="md-plain">PDF XSS 漏洞的利用需同时满足三个条件：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">存在恶意代码载体</span></strong></span><span class="md-plain">：PDF 文件中包含经过精心构造的恶意 JavaScript 代码</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">解析器安全缺陷</span></strong></span><span class="md-plain">：PDF 阅读器 / 预览工具存在沙箱绕过、权限管控不严等问题</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">执行环境允许</span></strong></span><span class="md-plain">：目标工具未禁用 JavaScript 执行功能（多数阅读器默认开启）</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading md-focus"><span class="md-plain">3. 攻击链形成过程</span></h3>
<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击者利用 PDF 编辑工具或脚本生成含恶意代码的 PDF 文件</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">通过钓鱼邮件、恶意链接等方式诱导用户下载打开</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">用户使用存在漏洞的 PDF 工具打开文件</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">恶意代码突破安全限制执行，完成信息窃取或权限提升</span></p>
</li>
</ol>
<h2 class="md-end-block md-heading"><span class="md-plain">二、PDF XSS 漏洞分类与技术特点</span></h2>
<h3 class="md-end-block md-heading"><span class="md-plain">1. 基于嵌入式 JavaScript 的传统漏洞</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">技术特点</span></strong></span><span class="md-plain">：直接利用 PDF 规范中的 JavaScript 支持特性</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">典型利用方式</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">在文档级</span><span class="md-pair-s" spellcheck="false"><code>/JS</code></span><span class="md-plain">字典中写入</span><span class="md-pair-s" spellcheck="false"><code>app.launchURL(&quot;恶意地址&quot;)</code></span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">在表单按钮的</span><span class="md-pair-s" spellcheck="false"><code>OnClick</code></span><span class="md-plain">事件中嵌入</span><span class="md-pair-s" spellcheck="false"><code>this.exportDataObject</code></span><span class="md-plain">下载恶意文件</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">代表案例</span></strong></span><span class="md-plain">：Adobe Acrobat CVE-2021-28550（JS API 权限绕过）</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">2. 基于渲染引擎缺陷的新型漏洞</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">技术特点</span></strong></span><span class="md-plain">：利用 PDF 解析器在处理字体、图像等资源时的逻辑漏洞</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">典型利用方式</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">通过特殊字体指令注入代码（如 PDF.js CVE-2024-4367）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">利用注释内容 DOM 注入（如 PDF.js CVE-2020-16042）</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">代表案例</span></strong></span><span class="md-plain">：PDF.js 系列漏洞（无需依赖 PDF 内置 JS 功能）</span></p>
</li>
</ul>
<h2 class="md-end-block md-heading"><span class="md-plain">三、PDF.js CVE-2024-4367 漏洞复现</span></h2>
<h3 class="md-end-block md-heading"><span class="md-plain">1. 漏洞环境准备</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">（1）基础环境</span></h4>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">操作系统：Ubuntu 22.04</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">Docker 及 Docker Compose：用于部署漏洞环境</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">漏洞靶场：Vulhub 中的 PDF.js 环境</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击工具：LOURC0D3 的 CVE-2024-4367 PoC 脚本</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">（2）环境搭建步骤</span></h4>


```
# 安装Docker
sudo apt update && sudo apt install -y docker.io docker-compose
sudo systemctl start docker && sudo systemctl enable docker
​
# 获取Vulhub
git clone https://github.com/vulhub/vulhub.git
cd vulhub/pdfjs/CVE-2024-4367
​
# 启动漏洞环境
sudo docker-compose up -d
​
# 查看环境状态（确认8080端口映射）
sudo docker-compose ps
```


<h3 class="md-end-block md-heading"><span class="md-plain">2. 恶意 PDF 生成（利用 LOURC0D3 的 PoC）</span></h3>
<h4 class="md-end-block md-heading"><span class="md-plain">（1）获取 PoC 工具</span></h4>


```
git clone https://github.com/LOURC0D3/CVE-2024-4367-PoC.git
cd CVE-2024-4367-PoC
```


<h4 class="md-end-block md-heading"><span class="md-plain">（2）生成含 XSS payload 的 PDF</span></h4>


```
# 生成弹出 domain 的恶意PDF
python3 CVE-2024-4367.py "alert(document.domain)"
​
# 生成的文件为poc.pdf，查看文件信息
file poc.pdf
```


<p class="md-end-block md-p"><span class="md-plain">PoC 核心原理：通过修改 PDF 的</span><span class="md-pair-s" spellcheck="false"><code>FontMatrix</code></span><span class="md-plain">字段，将恶意 JavaScript 代码伪装为字体配置参数，利用 PDF.js 字体渲染时的代码编译机制执行。</span></p>
