<h2 class="header-vfC6AV auto-hide-last-sibling-br">一、漏洞概述</h2>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. 漏洞描述</h3>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">Struts2 是 Apache 推出的基于 MVC 模式的轻量级 Web 框架，在处理标签属性（如<code>id</code>）值时，会将用户可控参数直接作为<strong>OGNL 表达式</strong>解析。攻击者可构造恶意 OGNL 表达式注入标签属性，框架解析时执行表达式中的恶意代码，导致远程代码执行（RCE）。</div>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">2. OGNL 表达式基础</h3>
<ul class="auto-hide-last-sibling-br">
 	<li><strong>定义</strong>：Object-Graph Navigation Language（对象导航图语言），支持存取对象属性、调用方法、遍历对象结构及类型转换，功能强大但风险较高。</li>
 	<li><strong>风险示例</strong>：在 Struts2 标签中，OGNL 可直接调用 Java 方法，如：
<div class="relative w-fit custom-code-block-canvas-wrapper w-full">
<div class="code-block-element-YDlfvc light custom-code-block-container--html hide-indicator disable-theme-style custom-code-block-container">
<div class="code-area-tZtDgG code-area" dir="ltr">
<div class="header-wrapper-UnnlXS"></div>
<div class="content-EGRozu code-content light-scrollbar-TbsSwi">


```html
<code class="language-html">&lt;s:property value=&quot;@java.lang.Runtime@getRuntime().exec(&#039;calc&#039;)&quot; /&gt;  
</code>
```


</div>
</div>
</div>
</div>
上述代码会直接执行本地计算器程序。若用户输入被直接拼接到 OGNL 表达式中，攻击者可注入恶意命令。</li>
</ul>

<h2 class="header-vfC6AV auto-hide-last-sibling-br">四、漏洞核心原理</h2>
<ol class="auto-hide-last-sibling-br">
 	<li><strong>OGNL 解析缺陷</strong>：Struts2 将标签属性（如<code>id</code>）的用户输入直接作为 OGNL 表达式解析，未过滤恶意内容；</li>
 	<li><strong>安全限制绕过</strong>：<br class="container-utlnW2 wrapper-d0Cc1k undefined" />POC1 通过获取框架上下文对象（<code>context</code>、<code>container</code>），修改<code>OgnlUtil</code>的<code>excludedClasses</code>和<code>excludedPackageNames</code>为空，解除对<code>Runtime</code>等危险类的限制；</li>
 	<li><strong>命令执行链</strong>：<br class="container-utlnW2 wrapper-d0Cc1k undefined" />POC2 利用<code>@java.lang.Runtime@getRuntime().exec()</code>调用系统命令，结合反弹 Shell 实现远程控制。</li>
</ol>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">五、漏洞识别方法</h2>
<ol class="auto-hide-last-sibling-br">
 	<li><strong>表达式验证</strong>：<br class="container-utlnW2 wrapper-d0Cc1k undefined" />向参数（如<code>id</code>、<code>name</code>）传入简单 OGNL 表达式（如<code>%{4*5}</code>或 URL 编码后的<code>%25{4*5}</code>），若返回计算结果，说明存在 OGNL 解析漏洞；</li>
 	<li><strong>错误信息分析</strong>：<br class="container-utlnW2 wrapper-d0Cc1k undefined" />构造畸形 OGNL 表达式（如<code>%{invalid}</code>），若响应中出现<code>ognl.OgnlException</code>等错误，表明框架正在解析 OGNL 表达式；</li>
 	<li><strong>版本核查</strong>：<br class="container-utlnW2 wrapper-d0Cc1k undefined" />确认目标 Struts2 版本，若为 2.5.16 及以下，结合上述特征可判断存在风险；</li>
 	<li><strong>工具扫描</strong>：<br class="container-utlnW2 wrapper-d0Cc1k undefined" />使用 Burp Suite 的 Struts2 插件、Nessus 等工具，自动检测是否存在 S2-059 等远程代码执行漏洞。</li>
</ol>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">六、防御措施</h2>
<ol class="auto-hide-last-sibling-br">
 	<li><strong>版本升级</strong>：升级至 Struts2 2.5.17 及以上安全版本，官方已修复 OGNL 解析漏洞；</li>
 	<li><strong>禁用静态方法调用</strong>：在<code>struts.xml</code>中配置：
