<h2 class="header-vfC6AV auto-hide-last-sibling-br">一、漏洞概述</h2>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">WebLogic 是 Oracle 推出的 Java 应用服务器，其专有通信协议<strong>T3 协议</strong>因存在反序列化缺陷，成为远程代码执行漏洞的重灾区。攻击者可构造恶意序列化数据，通过 T3 协议发送给 WebLogic 服务器，在服务器反序列化过程中执行任意代码，实现远程控制。</div>
<div></div>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">历史上基于 T3 协议的高危漏洞包括 CVE-2015-4582、CVE-2016-0638、CVE-2018-2628、CVE-2020-2555 等，覆盖多个 WebLogic 版本，对企业级应用威胁极大。</div>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">二、T3 协议核心解析</h2>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. 协议用途与特点</h3>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">T3 协议是 WebLogic 的专有协议，主要用于：</div>
<ul class="auto-hide-last-sibling-br">
 	<li>服务器与客户端之间的通信；</li>
 	<li>服务器节点之间的内部数据传输；</li>
 	<li>承载 RMI（远程方法调用）通信，传输过程中会对 Java 对象进行序列化 / 反序列化处理。</li>
</ul>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">2. 协议协商流程</h3>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">T3 协议通信始于 “握手” 阶段，客户端发送包含协议版本等信息的包头，服务器返回响应确认协议支持。</div>

<h3 class="header-vfC6AV auto-hide-last-sibling-br">3. 协议数据结构</h3>
<img class="alignnone wp-image-387 " src="/wp-content/uploads/2025/img-06fa6123d585.png" alt="" width="941" height="141" />
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">T3 协议数据包由 7 部分组成，其中第 2-7 部分为序列化数据（以<code>ac ed 00 05</code>为标识，Java 序列化数据的魔数），这是漏洞利用的关键区域。</div>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">三、漏洞利用原理</h2>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">1. 核心逻辑</h3>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space">T3 协议在传输 Java 对象时需经过 “序列化→传输→反序列化” 流程。攻击者通过构造恶意序列化对象，替换 T3 协议中第 2-7 部分的合法序列化数据，或拼接在第一部分后，使服务器在反序列化时执行恶意代码。</div>
<h3 class="header-vfC6AV auto-hide-last-sibling-br">2. 利用流程</h3>
<ol class="auto-hide-last-sibling-br">
 	<li>向 WebLogic 服务器发送 T3 协议握手包，确认协议支持；</li>
 	<li>生成恶意序列化数据（通过<code>ysoserial</code>等工具）；</li>
 	<li>将恶意数据按 T3 协议格式封装，作为第二个数据包发送；</li>
 	<li>服务器反序列化恶意数据，触发代码执行（如反弹 shell、文件操作）。</li>
</ol>

<h2 class="header-vfC6AV auto-hide-last-sibling-br">六、漏洞识别方法</h2>
<ol class="auto-hide-last-sibling-br">
 	<li>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space"><strong>T3 协议探测</strong>：<br class="container-utlnW2 wrapper-d0Cc1k undefined" />向目标 7001 端口发送 T3 握手包，若收到含<code>HELO</code>的响应，说明 T3 协议启用，存在潜在风险。</div></li>
 	<li>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space"><strong>版本核查</strong>：<br class="container-utlnW2 wrapper-d0Cc1k undefined" />从响应的<code>HELO</code>信息提取版本（如<code>HELO:12.2.1.3.false</code>），若版本在漏洞影响范围内（如 10.3.6.0、12.1.3.0），则可能存在漏洞。</div></li>
 	<li>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space"><strong>工具检测</strong>：<br class="container-utlnW2 wrapper-d0Cc1k undefined" />使用 Nessus、AWVS 等扫描工具，或专用脚本（如<code>weblogic_scan</code>）检测 7001 端口，自动识别 T3 协议状态及漏洞版本。</div></li>
 	<li>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space"><strong>漏洞验证</strong>：<br class="container-utlnW2 wrapper-d0Cc1k undefined" />发送含无害序列化数据的 T3 包，若服务器返回<code>ClassNotFoundException</code>等反序列化错误，说明协议解析逻辑存在，可进一步测试恶意 Payload。</div></li>
</ol>
<h2 class="header-vfC6AV auto-hide-last-sibling-br">七、防御建议</h2>
<ol class="auto-hide-last-sibling-br">
 	<li>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space"><strong>关闭 T3 协议</strong>：</div>
<ul class="auto-hide-last-sibling-br">
 	<li>登录 WebLogic 控制台，进入 “服务器→配置→协议”，取消勾选 “启用 T3 协议”；</li>
 	<li>通过防火墙阻断 7001 端口的外部访问（非必要不开放）。</li>
</ul>
</li>
 	<li>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space"><strong>更新补丁与 JDK</strong>：</div>
<ul class="auto-hide-last-sibling-br">
 	<li>应用 Oracle 官方对应漏洞补丁（如 CPU 季度补丁）；</li>
 	<li>升级 JDK 至 1.7.0.21 及以上，修复底层序列化漏洞。</li>
</ul>
</li>
 	<li>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space"><strong>代码与配置防护</strong>：</div>
<ul class="auto-hide-last-sibling-br">
 	<li>在 WebLogic 黑名单中添加危险类（如<code>org.apache.commons.collections.Transformer</code>）；</li>
 	<li>部署 WAF 拦截 T3 协议中含<code>ac ed 00 05</code>（序列化魔数）+ 危险类名的数据包。</li>
</ul>
</li>
 	<li>
<div class="auto-hide-last-sibling-br paragraph-JOTKXA paragraph-element br-paragraph-space"><strong>监控审计</strong>：</div>
<ul class="auto-hide-last-sibling-br">
 	<li>启用 WebLogic 日志，监控异常 T3 连接和反序列化错误（如<code>InvalidClassException</code>）；</li>
 	<li>定期扫描端口，确认 T3 协议是否已禁用或限制访问。</li>
</ul>
</li>
</ol>
